## 3. 总览图说明

## 3.1 图 0：比赛系统 IPO 分解总图

该图的作用是先把系统拆成 3 个主层：

1. `1 输入`
2. `2 处理`
3. `3 输出`

### 图中含义

#### `1 输入`

表示系统运行之前必须给定的所有输入对象：

1. 控制参数
2. 配置文件
3. 股票清单
4. 原始业务数据或参考特征文件

#### `2 处理`

表示系统的核心处理链路：

1. 样本装载
2. 特征提取
3. Task1 模式识别
4. Task2 资金识别
5. 市场 PID 聚合

#### `3 输出`

表示系统最终向比赛、分析、复盘交付的文件：

1. 标准比赛提交文件
2. 市场状态辅助文件
3. 诊断与打包文件

### 为什么总览图先拆成 1、2、3

因为 IPO 分析的核心就是：

1. 输入是什么
2. 如何处理
3. 产出什么

所以总图先给出最稳定的主框架，再继续下钻到细节图。

## 4.2 图 1-1：控制输入

控制输入对应 `main.py` 命令行参数，决定本次批处理任务如何执行。

### 主要内容

1. `mode`
2. `date`
3. `input_dir`
4. `output_dir`
5. `stock_limit`
6. `stock_offset`
7. `stock_list_file`
8. `build_zip`

### IPO 意义

这是系统最外层输入边界，决定：

1. 分析哪一天
2. 读哪一批数据
3. 只跑哪些股票
4. 输出到哪里

也就是说，控制输入决定系统“做什么任务”。

---

## 4.3 图 1-2：配置输入

配置输入由 `dev.yaml` 和 `label_dict.yaml` 提供。

### 主要内容

1. Task1 阈值
2. Task2 阈值
3. 标签压缩模式
4. 市场快照开关
5. 提交压缩包开关
6. 标签字典全集

### IPO 意义

配置输入不直接提供业务数据，但提供“处理规则”。

它决定：

1. 低置信样本怎么回退
2. 资金意图输出用细粒度还是压缩标签
3. 哪些附加文件要不要导出

也就是说，配置输入决定系统“按什么规则处理”。

## 4.4 图 1-3：数据输入

数据输入是系统最关键的业务输入。

### 主要内容

1. `逐笔成交.csv`
2. `逐笔委托.csv`
3. `行情.csv`
4. `reference_features.csv / features.csv / 参考特征.csv`

### IPO 意义

这部分数据是所有后续特征和输出的来源。

系统支持两条装载路径：

1. 原始逐笔路径
2. 参考特征路径

也就是说，数据输入决定系统“拿什么样本做分析”。

## 5. 处理分解图说明

处理层是系统的核心，因此拆成 5 张分图。

---

## 5.1 图 2-1：样本装载与构建图

该图解释原始目录如何变成统一的 `DailySample`。

### 处理步骤

1. 扫描输入目录
2. 判断是否存在参考特征文件
3. 若无参考特征，则按股票目录装载
4. 按交易日和股票清单筛选
5. 为每个股票-交易日生成一个 `DailySample`

### IPO 意义

它解决的问题是：如何把杂乱的文件系统输入，整理成后续算法能稳定消费的统一样本对象。

---

## 5.2 图 2-2：日级摘要特征分解图

该图解释 `DailySample.feature_summary` 的构成。

### 分成四类特征

1. 价格方向类
2. 成交活跃类
3. 委托盘口类
4. 衍生解释类

### 代表公式

1. `close_return = (close_price - prev_close) / prev_close`
2. `intraday_range = (high_price - low_price) / prev_close`
3. `tail_ratio = tail_trade_amount / total_trade_amount`
4. `order_buy_ratio = buy_orders / (buy_orders + sell_orders)`
5. `directional_efficiency = min(abs(close_return - open_return) / intraday_range, 1)`

### IPO 意义

这一步实现了从“原始市场记录”到“可解释结构化特征”的转换，是整个处理层的基础。

---

## 5.2.1 图 2-2-1：价格方向特征

对应文件：

`2-2-1_price_direction_features.png`

### 说明

这张图聚焦价格本身的运动轨迹，包括：

1. 收盘收益方向
2. 开盘跳空
3. 日内振幅
4. 收盘位置强弱

### 代表公式

1. `close_return = (close_price - prev_close) / prev_close`
2. `open_return = (open_price - prev_close) / prev_close`
3. `intraday_range = (high_price - low_price) / prev_close`
4. `close_strength = (close_price - low_price) / (high_price - low_price)`

### IPO 意义

它告诉我们价格是怎样走出来的，是所有行为判断的最直接结果层。

---

## 5.2.2 图 2-2-2：成交活跃特征

对应文件：

`2-2-2_trading_activity_features.png`

### 说明

这张图聚焦成交规模、尾盘活跃和阶段性爆发。

### 代表公式

1. `deal_amount = sum(price * volume)`
2. `avg_trade_size = total_trade_amount / trade_count`
3. `tail_ratio = tail_trade_amount / total_trade_amount`
4. `burst_ratio = max(bucket_amount) / total_bucket_amount`

### IPO 意义

它告诉我们市场参与是均匀展开的，还是在某个时段集中爆发的。

---

## 5.2.3 图 2-2-3：委托盘口特征

对应文件：

`2-2-3_order_book_features.png`

### 说明

这张图聚焦买卖委托结构、撤单近似比例以及盘口支撑和抛压。

### 代表公式

1. `order_buy_ratio = buy_orders / (buy_orders + sell_orders)`
2. `cancel_ratio`：委托类型字段近似撤单比
3. `bid_support = bid_vol / (bid_vol + ask_vol)`
4. `ask_pressure = ask_vol / (bid_vol + ask_vol)`

### IPO 意义

它告诉我们资金在盘口上的姿态，是主动承接、主动抛压，还是高扰动试探。

---

## 5.2.4 图 2-2-4：衍生解释特征

对应文件：

`2-2-4_derived_explanatory_features.png`

### 说明

这张图聚焦对价格、成交和盘口进行二次组合后得到的解释性特征。

### 代表公式

1. `directional_efficiency = min(abs(close_return - open_return) / intraday_range, 1)`
2. `reversal_strength = close_return - open_return`
3. `price_impact = abs(close_price - prev_close) / prev_close`
4. `up_count_market / down_count_market`

### IPO 意义

它把多个基础指标组合成更适合解释行为风格和市场状态的上层特征。

---

## 5.3 图 2-3：Task1 交易模式识别图

该图解释交易模式识别的三段式逻辑。

### 三段式结构

1. 强规则识别
2. 候选模式评分
3. 低置信回退

### 为什么这么设计

因为当前比赛系统不是训练好的聚类原型在线推理系统，而是基线规则系统，所以必须：

1. 先抓明显模式
2. 再对复杂模式打分比较
3. 对边界样本做回退，保证稳定性

### IPO 意义

Task1 是把摘要特征转换成 `pattern_type` 和 `pattern_explanation` 的关键环节。

## 5.4 图 2-4：Task2 资金识别图

该图解释资金类型与意图识别过程。

### 主要步骤

1. 计算 `retail_score`
2. 计算 `hot_money_score`
3. 计算 `quant_score`
4. 选择最高分为 `capital_type`
5. 再按类型分支判断资金意图
6. 压缩为提交标签

### IPO 意义

Task2 不是单纯分类，而是两层映射：

1. 先从特征映射到资金类型
2. 再从资金类型映射到资金意图

所以它比 Task1 更像一个“分层决策器”。

## 5.5 图 2-5：市场 PID 聚合图

该图解释如何从单股票特征上升到市场横截面状态。

### 主要步骤

1. 对每只股票计算 `P / I / D`
2. 对所有股票聚合均值、中位数、标准差
3. 计算 `breadth_ratio` 和 `breadth_balance`
4. 判定市场状态 `market_regime`
5. 生成个股相对市场指标 `p_rel / i_rel / d_rel`

### IPO 意义

这一层把个股分析提升成市场分析，使系统不仅能判断单只股票，还能判断它在市场中的相对位置。

## 7. 从整体到细节的 IPO 分析路径

如果要从整体到细节分析系统，可按下面顺序：

1. 先看图 0，理解系统的三层主结构
2. 看图 1，再分别看 `1-1 / 1-2 / 1-3`，理解输入来源
3. 看图 2-1，理解样本如何进入系统
4. 看图 2-2，再分别看 `2-2-1 / 2-2-2 / 2-2-3 / 2-2-4`，理解原始数据如何转成特征
5. 看图 2-3，再分别看 `2-3-1 / 2-3-2 / 2-3-3`，理解特征如何转成交易模式
6. 看图 2-4，再分别看 `2-4-1 / 2-4-2 / 2-4-3`，理解特征如何转成资金类型与意图
7. 看图 2-5，理解个股如何进一步汇聚成市场状态
8. 看图 3，再分别看 `3-1 / 3-2 / 3-3`，理解系统如何输出比赛文件、市场状态文件和诊断交付文件

这条路径正好对应 IPO 分析最典型的逻辑：

`输入 -> 数据整理 -> 特征提取 -> 决策识别 -> 聚合分析 -> 输出交付`

## 8. 结论

通过这套“总图 + 分图 + 编号说明”的结构，系统的 IPO 过程被拆成了一个可以逐层追踪的体系：

1. 输入层回答“系统拿到了什么”
2. 处理层回答“系统如何把数据变成结果”
3. 输出层回答“系统最终交付了什么”

这使得比赛系统既能从宏观上被理解，也能在细节层面被复核、答辩和扩展。
