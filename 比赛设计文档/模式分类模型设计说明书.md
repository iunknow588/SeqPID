## 1. 文档目的

本文件定义比赛系统中的“资金运行模式分类模型”。

这里的 `pattern_type` 不应被理解为简单的价格形态标签，而应被理解为**资金运行模式识别结果**。  
模型目标不是扩充标签数，而是把日内成交、盘口、收盘位置和 PID 贡献统一解释成少量稳定模式。

当前官方提交目标更接近以下模式集合：

- `量化T0`
- `散户博弈`
- `尾盘突袭`
- `日内套利`
- `大单吸筹`

因此，模式分类模型需要完成两件事：

1. 识别资金运行模式
2. 给出可审计的证据与解释

## 2. 资金运行模式分类模型重构计划

### 阶段 1：统一目标口径

1. `pattern_type` 最终提交只输出五类资金运行模式：
   - `量化T0`
   - `散户博弈`
   - `尾盘突袭`
   - `日内套利`
   - `大单吸筹`
2. 当前内部标签不再直接作为提交标签。
3. 在 `label_dict.yaml` 中增加 `pattern_labels_submit`。

### 阶段 2：重构 Python 检测器

1. 将 `比赛系统/src/pattern_model.py` 从单体候选打分改成五个模式检测器。
2. 每个检测器输出 `score / evidence / confidence`。
3. 删除或替换当前 `del label_dict`，让模型读取提交标签配置。
4. 保留旧标签作为证据字段，不作为最终 `pattern_type`。

### 阶段 3：重构 Rust 检测器

1. 将 `src-rust/src/pattern_model.rs` 与 Python 规则对齐。
2. 抽取相同的五类模式评分逻辑。
3. 保证 Rust/Python 对同一样本输出一致。

### 阶段 4：引入 PID 二次修正

1. 使用 `dominant_type / dominant_intention` 辅助区分：
   - `量化T0`
   - `散户博弈`
   - `大单吸筹`
2. PID 只做修正，不直接替代原始模式检测。
3. 修正结果必须写入解释和诊断字段。

### 阶段 5：回归验证

1. 使用官方样例检查标签全集是否一致。
2. 使用 20260715 的 100 只样本重跑 Rust/Python。
3. 验证：
   - 输出标签全集只包含五类目标模式
   - Rust/Python `pattern_reco.csv` 一致
   - 典型样本解释能对应资金运行模式

## 3. 现状问题

当前实现存在三个问题：

1. `pattern_model` 主要是价格形态分类，不是资金运行模式分类。
2. Python 版本直接忽略 `label_dict`，Rust 版本也硬编码了内部标签池。
3. 现有标签更多是中间态证据，如 `盘中诱多 / 对倒拉升 / 压单吸货`，并非最终提交目标。

因此，现有模型的核心缺陷不是“少几个标签”，而是：

- 检测对象不对
- 证据聚合方式不对
- 最终输出口径不对

## 4. 目标定义

### 3.1 输出目标

最终输出仍保持原始字段：

```text
stock_code, transaction_date, pattern_type, pattern_explanation
```

但 `pattern_type` 必须是资金运行模式，而不是内部形态名称。

### 3.2 建议目标模式

| 模式 | 定义 |
| --- | --- |
| 量化T0 | 高频小单、快速回转、日内来回做差价 |
| 散户博弈 | 主导资金不强，小单混乱，噪声高、方向弱 |
| 尾盘突袭 | 尾盘集中放量，收盘段显著改变价格路径 |
| 日内套利 | 日内振幅大，但收盘不极端，反复高抛低吸 |
| 大单吸筹 | 大单持续承接，价格不一定暴涨，但筹码被吸收 |

## 5. 检测框架

模型采用“多检测器打分 + 规则裁决 + PID 二次修正”的三段式结构。

### 4.1 第一层：模式检测器

每个模式对应一个独立检测器，输出：

- `score`
- `evidence`
- `confidence`

检测器不是单一阈值，而是多个信号加权。

### 4.2 第二层：裁决器

若多个模式得分接近：

- 优先尾盘/收盘强特征
- 再看量化/散户主导性
- 再看日内振幅与回转特征

### 4.3 第三层：PID 修正

PID 不负责定义模式，只负责二次校验：

- `dominant_type`
- `dominant_intention`
- `hot_money_ratio / quant_ratio / retail_ratio`
- `tail_ratio / inertia / damping`

PID 只能修正模式倾向，不能直接替代模式检测。

## 6. 五类模式检测信号

### 5.1 量化T0

核心信号：

- `avg_trade_size` 偏小
- 买卖方向频繁切换
- `gross_flow / abs(net_flow)` 偏高
- 收盘不极端
- 日内振幅存在，但尾部不强

辅助证据：

- `quant_ratio` 高
- `burst_ratio` 中高
- `noise_ratio` 较高

### 5.2 散户博弈

核心信号：

- `retail_ratio` 高，或三类资金都不强
- 小单为主
- 盘口噪声大
- 方向性弱
- 收盘位置中性

辅助证据：

- `order_buy_ratio` 接近 0.5
- `explain_ratio` 低
- `noise_ratio` 高

### 5.3 尾盘突袭

核心信号：

- `tail_ratio` 高
- `last15_return` 显著
- 收盘前成交集中
- 收盘位置明显偏高或偏低

辅助证据：

- `close_strength` 极端
- `burst_ratio` 尾部放大

### 5.4 日内套利

核心信号：

- `intraday_range` 高
- `abs(close_return)` 不大
- 收盘靠近中性区
- 日内多次反转

辅助证据：

- `neutral_close_score` 高
- `mid_close_score` 高

### 5.5 大单吸筹

核心信号：

- 大单持续承接
- `close_strength` 偏高
- `bid_support` 强
- `ask_pressure` 不足以压垮价格

辅助证据：

- `hot_money_ratio` 高
- `large_order_score` 高
- `buy_bias_score` 高

### 8.3 先检测，再压缩

## 12. 基于官方样本的盘面预判类型归纳

本节根据以下两个官方样本目录归纳：

- `背景知识介绍/官方标准数据资料/submit`
- `背景知识介绍/官方标准数据资料/submit 2/submit`

两个目录中的 `pattern_reco.csv` 与 `predict_result.csv` 样本内容一致，均为 20 条记录。  
样本不是完整训练集，但足以反推出官方提交口径的基本盘面预判类型。

### 12.1 样本标签分布

`pattern_reco.csv` 的五类模式分布如下：

| 模式 | 样本数 | 样本含义 |
| --- | ---: | --- |
| 散户博弈 | 6 | 低成交、弱主导、小单混乱、情绪换手 |
| 尾盘突袭 | 5 | 收盘前突然放量，主动影响收盘价或制造情绪 |
| 量化T0 | 5 | 高频拆单、反复买卖、围绕盘口价差做日内回转 |
| 日内套利 | 3 | 围绕 VWAP 或价格区间做高抛低吸 |
| 大单吸筹 | 1 | 大额资金集中承接或扫货，筹码被吸收 |

`predict_result.csv` 的参与者与方向分布如下：

| 维度 | 标签 | 样本数 | 样本含义 |
| --- | --- | ---: | --- |
| 参与者 | 散户 | 9 | 小单、跟随、情绪化、缺少稳定主导 |
| 参与者 | 游资 | 7 | 大单、突击、方向性强、尾盘或关键价位影响明显 |
| 参与者 | 量化 | 4 | 高频、拆单、价差回转、持仓周期短 |
| 方向 | 卖出 | 12 | 样本中卖出/出货/砸盘语义略多 |
| 方向 | 买入 | 8 | 样本中买入/建仓/吸筹语义次之 |

因此，官方样本实际传达的是一个三层判断结构：

1. `pattern_type`：盘面资金运行模式
2. `capital_type`：主要参与者类型
3. `capital_intention`：主方向，即买入或卖出

### 12.2 五类盘面基本确证预判类型

| 预判类型 | 盘面确证信号 | 常见参与者 | 常见方向 | 解释关键词 |
| --- | --- | --- | --- | --- |
| 量化T0 | 小单高频、买卖切换快、价格围绕盘口或均线反复回转 | 量化 | 买入/卖出均可 | 高频、拆单、价差、回转、数分钟持仓 |
| 散户博弈 | 成交低迷、小单为主、缺少大单支撑、方向混乱 | 散户 | 买入/卖出均可 | 小单、情绪、换手、缺主力、噪声 |
| 尾盘突袭 | 最后 10~30 分钟放量，收盘价被明显改变 | 游资 | 买入或卖出 | 尾盘、突击、拉升、砸盘、封板、恐慌 |
| 日内套利 | 日内振幅存在，但收盘不极端，围绕 VWAP 或区间反复交易 | 量化/游资 | 买入/卖出均可 | VWAP、区间、高抛低吸、均值回复 |
| 大单吸筹 | 大额挂单或扫货，价格冲击可控，筹码被持续承接 | 游资 | 买入 | 大单、扫货、承接、吸筹、建仓 |

这五类不是价格形态标签，而是盘面行为的预判类型。  
同一类模式内部可以有买入或卖出方向，例如 `尾盘突袭` 既可以是尾盘拉升，也可以是尾盘砸盘；最终方向由 `predict_result.csv` 中的 `capital_intention` 表达。

### 12.3 样本对模型设计的约束

1. `pattern_type` 必须稳定收敛到五类模式，不应输出 `对倒拉升 / 盘中诱多 / 分时脉冲` 等内部证据标签。
2. `pattern_explanation` 应优先使用样本中体现出的盘口语言：尾盘、放量、拉升、砸盘、小单、大单、VWAP、高抛低吸、T0 回转。
3. `capital_type` 与 `pattern_type` 不是一一映射关系，只能作为 PID 修正与解释证据。
4. `capital_intention` 只表达方向，不应反向覆盖 `pattern_type`。
5. 样本中的 `散户博弈 / 尾盘突袭 / 量化T0` 占比较高，说明模型应特别重视小单噪声、尾盘集中度和高频回转三个信号。

### 12.4 解释文本与资金类型的一致性

模式解释不应只解释价格形态，还应给出最终资金类型判断的近似原因。  
也就是说，读者看到 `pattern_explanation` 后，应能大致理解为什么 `predict_result.csv` 会输出 `散户 / 游资 / 量化` 中的某一类。

| 最终资金类型 | 解释中应体现的证据 | 典型描述 |
| --- | --- | --- |
| 散户 | 小单、分散、跟风、低成交、噪声高、方向混乱 | 小单换手、散户情绪、缺少主导资金、盘口支撑弱 |
| 游资 | 大单、突击、扫货、拉升、砸盘、尾盘控制、封板/开板 | 大单买入、尾盘突袭、快速拉升、集中砸盘、强势收盘 |
| 量化 | 高频、拆单、价差、VWAP、均值回复、T0 回转、买卖均衡 | 程序化拆单、盘口价差、短周期回转、VWAP 附近套利 |

解释文本可以同时描述多种资金行为，例如“游资尾盘拉升后散户跟风接力”。  
但最终 `capital_type` 只能选择主导者，因此解释中必须能看出谁是主要驱动力：

1. 影响价格最明显者优先
2. 持续时间最长者优先
3. 成交金额贡献最大者优先
4. 与 PID `dominant_type` 一致者优先
5. 若证据冲突，则在解释中写明“主导资金不强 / 多方混合”，并降低置信度

### 12.5 解释摘要与提交句

官方样本里的 `pattern_explanation` 本质上是“摘要化后的提交句”。  
完整生成过程应拆成三步：

1. 概要
   - 先描述主导资金和关键时点
2. 详细解释
   - 再展开成交方式、订单规模、持续方式和尾盘行为
3. 结论压缩
   - 最后压成一条适合提交的短句

例如：

- 概要：尾盘出现集中资金拉升
- 详细解释：在下午 2 点半之后成交显著放大，大单持续扫货，快速推高收盘价
- 结论压缩：`尾盘突袭, 在下午2点半之后集中拉升，制造假象和典型技术形态`

因此，提交文件中的 `pattern_explanation` 不是原始分析全文，而是“从详细解释压缩而来的结论句”。  
压缩时应优先保留：

1. 主导资金
2. 关键时点
3. 主要动作
4. 模式结论

删除：

- 重复词
- 过长背景
- 没有区分度的修饰语
- 对最终判断没有帮助的旁支描述

这样既能保留可读性，也能保证 `pattern_explanation` 足够短、足够像官方样例。

### 12.6 提交句模板表

`pattern_explanation` 应由模板生成。模板的输入至少包含：

```text
pattern_type
capital_type
capital_intention
time_bucket
flow_style
order_size_style
price_effect
```

推荐模板如下：

| 模式 | 适用条件 | 提交句模板 |
| --- | --- | --- |
| 量化T0 | 高频、小单、买卖回转 | 程序化拆单后快速买入卖出，围绕盘口价差完成T0回转 |
| 量化T0 | VWAP 附近反复交易 | 量化模型围绕VWAP反复挂单，利用短周期偏离完成日内回转 |
| 散户博弈 | 低成交、小单、方向混乱 | 成交以小单为主且方向混乱，呈现散户之间的博弈换手 |
| 散户博弈 | 缺少大单支撑 | 盘口缺乏大单支撑，价格随散户情绪小幅波动 |
| 尾盘突袭 | 尾盘拉升 | 在下午2点半之后集中拉升，制造强势收盘和典型技术形态 |
| 尾盘突袭 | 尾盘砸盘 | 收盘前集中砸盘，制造恐慌情绪并显著影响收盘价 |
| 日内套利 | 区间反复 | 资金在一定价格区间来回高抛低吸，获取日内价差收益 |
| 日内套利 | VWAP 回转 | 资金围绕VWAP上下反复交易，赚取均值回复利润 |
| 大单吸筹 | 大单集中买入 | 资金大笔挂单买入，短时间内集中扫货并吸收筹码 |
| 大单吸筹 | 持续承接 | 盘中持续出现大额买单，逐步抬高股价重心 |

模板选择优先级：

1. 先按 `pattern_type` 选模板组
2. 再按 `capital_intention` 选择买入或卖出语义
3. 再按 `time_bucket` 选择早盘、盘中、尾盘、竞价表达
4. 再按 `order_size_style` 选择大单、小单或拆单表达
5. 最后按 `price_effect` 选择拉升、砸盘、回转、承接、换手等动作词

### 12.7 提交句字段拼接规则

为保证解释统一，建议先生成结构化摘要，再渲染为中文短句：

```text
主语 = 主导资金或资金
时间 = 早盘 / 盘中 / 下午2点半之后 / 尾盘 / 集合竞价
动作 = 集中拉升 / 集中砸盘 / 持续承接 / 高频回转 / 小单换手
结果 = 影响收盘价 / 制造强势形态 / 赚取价差 / 吸收筹码 / 形成散户博弈
```

示例：

| 结构化摘要 | 提交句 |
| --- | --- |

## 13. 结论

`pattern_type` 的核心不是补标签，而是重建模式识别逻辑。  
当前模型应从“形态分类器”升级为“资金运行模式检测器”。
