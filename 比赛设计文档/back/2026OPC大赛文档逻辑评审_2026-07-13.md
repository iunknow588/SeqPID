## 二、《金融领域的物理学概念模型.md》逻辑缺陷

### 缺陷1：符号体系存在歧义（第6节理论主方程）

**问题描述：**
文档第6节给出理论主方程：
```
v_q,t = (P_state(q_t) - P_state(q_t-1)) / Delta q_t
P_state,t = P_state,t-1 + v_q,t * Delta q_t
```

但紧接着又写：
```
v_t = v_q,t
a_t = v_t - v_t-1
```

这里 `a_t` 被定义为 `v_t - v_t-1`，即**交易时间域内的一阶差分**，但物理类比中加速度应该是 `d²P/dq²`，即 `v_q` 对 `q` 的导数。当 `Delta q ≠ 1` 时，`a_t = v_t - v_t-1` 不等于 `Delta_q² P_state`，量纲不一致。

**影响：** 导致后续 "齐次特征方程" `r² - (phi + theta)r + theta = 0` 的推导基础不严谨。

**建议：** 明确定义 `a_t = (v_q,t - v_q,t-1) / Delta q_t`，并在等步长 `Delta q = 1` 时注明简化关系。

---

### 缺陷2：稳定性条件的推导前提未声明（第6节）

**问题描述：**
文档给出齐次特征方程：
```
r² - (phi + theta) * r + theta = 0
```

并给出 Jury 稳定性条件：
```
abs(theta) < 1
1 - phi > 0
1 + phi + 2 * theta > 0
```

但**未说明该特征方程的推导前提**。该方程对应的是二阶差分方程：
```
v_t - phi * v_t-1 - theta * (v_t-1 - v_t-2) = 0
=> v_t - (phi + theta) * v_t-1 + theta * v_t-2 = 0
```

这与主方程中的 `v_t = ... + phi * v_t-1 + theta * (v_t-1 - v_t-2)` 一致，但文档中先定义 `a_t = v_t - v_t-1`，然后直接跳到特征方程，中间缺少显式推导。

**影响：** 读者可能误以为特征方程来自 `P_state` 的二阶差分，而非 `v_q` 的递推关系。

**建议：** 补充显式推导步骤，明确特征方程是关于 `v_q` 的齐次递推，而非 `P_state`。

---

### 缺陷3：双域映射的链式法则在离散场景下的适用性未充分讨论（第5节）

**问题描述：**
文档给出连续形式的链式法则：
```
dP/dt = (dP/dV) * (dV/dt)
```

以及二阶展开：
```
d²P/dt² = (d²P/dV²) * (dV/dt)² + (dP/dV) * (d²V/dt²)
```

但在离散窗口中，`V(t)` 是阶梯函数，`dV/dt` 在数学上不存在（只有脉冲）。文档虽然提到 "阶梯性带来的差分偏差"，但**未给出离散场景下的严格替代公式**。

**影响：** 工程实现中若直接套用连续公式，会在零成交窗口产生除以零错误。

**建议：** 给出离散窗口的严格差分公式，例如：
```
Delta_t P / Delta_t ≈ (Delta_V P / Delta_V) * (Delta_t V / Delta_t)
```
并明确说明这是近似，仅在 `Delta_t V > 0` 时有效。

---

### 缺陷4：候选稳定量的检验标准过于模糊（第7.3节）

**问题描述：**
文档列出 `m_eff` 使用前必须满足的条件：
1. `beta_norm` 不接近 0
2. `beta_norm` 符号或方向经过稳定性处理
3. 行为代理流 `U_*` 可辨识
4. 残差诊断通过
5. 跨日波动幅度、变异系数和分位稳定性通过检验

但**未给出具体阈值或检验方法**，例如 "跨日波动幅度" 是指 7 日标准差？还是最大最小值比率？"分位稳定性" 如何量化？

**影响：** 工程实现中无法操作化执行，不同实现者可能得出完全不同的结论。

**建议：** 参照 PID 文档第10节的稳定性诊断表格，补充具体阈值建议。

---

### 缺陷5：慢变质量提取方法的优先级说明存在逻辑矛盾（第7.4节）

**问题描述：**
文档说：
> "RTS 平滑因使用未来观测，只适合离线复盘和稳定性画像"

但随后又说：
> "实时预测只能使用滤波或递推估计；所有方法均应记录 `m_slow_method`"

然后推荐优先级为：
> `ewma_realtime` > `rolling_median_realtime` > `kalman_trend_realtime`

这里的问题是：**`kalman_trend_realtime` 是否属于 "滤波或递推估计"？** 卡尔曼平滑（smoother）使用未来信息，但卡尔曼滤波（filter）是递推的。文档未区分 "滤波" 与 "平滑"，可能导致实现者误用 RTS 平滑（属于 smoother）进行实时预测。

**影响：** 前瞻性偏差风险。

**建议：** 明确区分 `kalman_filter_realtime`（递推，可用）与 `kalman_smoother_offline`（使用未来信息，仅离线），并禁止后者进入实时预测。

## 三、《PID算法与实现.md》逻辑缺陷

### 缺陷1：主状态方程的符号体系与概念文档不一致（第4节）

**问题描述：**
PID 文档第4节主状态方程：
```
v_q,t = beta_ch,t * U_ch,t-1 + ... + phi_t * v_q,t-1 + theta_t * (v_q,t-1 - v_q,t-2) + epsilon_t
```

但概念文档第6节写的是：
```
v_q,t = beta_ch * U_ch,t-1 + ... + phi * v_q,t-1 + theta * (v_q,t-1 - v_q,t-2) + epsilon_t
```

PID 文档中 `beta`、`phi`、`theta` 都带时间下标 `t`，表示 "当前窗口响应参数"；但概念文档中不带时间下标，表示常数参数。两份文档**未明确说明参数是时变还是时不变**。

更严重的是，PID 文档说：
> "`psi_t` 默认指卡尔曼滤波的一步先验 `psi_t|t-1`"

但又在参数估计章节（第6节）说：
> "`psi_t = psi_t-1 + eta_t`"（状态随机游走）

如果 `psi_t` 是状态向量，那么 `beta_t`、`phi_t`、`theta_t` 都是时变的；但概念文档中的稳定性分析（Jury 条件）假设参数是常数。两份文档的**参数时变/时不变假设不一致**。

**影响：** 时变参数下的稳定性分析需要 Lyapunov 理论或滚动窗口稳定性检验，不能直接使用常数参数的 Jury 条件。若实现者混用，可能导致稳定性误判。

**建议：** 
- 明确区分 "参数估计值 `psi_t|t-1`"（时变）与 "稳定性检验窗口内的平均参数 `bar_psi`"（近似常数）
- 稳定性诊断应基于滚动窗口内的参数统计，而非单点参数

---

### 缺陷2：观测向量的定义存在索引混淆（第4节）

**问题描述：**
PID 文档定义观测向量：
```
x_t+1|t = [v_q,t, U_ch,t-1, U_q,t-1, U_retail,t-1, D_driver_t]
```

但主状态方程是：
```
v_q,t = beta_ch,t * U_ch,t-1 + ... + phi_t * v_q,t-1 + theta_t * D_driver_t + epsilon_t
```

这里存在**索引不一致**：
- 主方程中，`v_q,t` 的右侧包含 `v_q,t-1`（即 `t-1` 时刻的速度）
- 但观测向量 `x_t+1|t` 的第一项是 `v_q,t`（即 `t` 时刻的速度），用于预测 `v_q,t+1`

如果 `x_t+1|t` 用于预测 `v_q,t+1`，那么第一项应该是 `v_q,t`（已知），这没问题。但主方程中 `v_q,t` 的预测使用的是 `v_q,t-1`，说明**主方程和预测方程的索引体系不一致**。

更混乱的是，文档又说：
> "`x_t` 由截至 `t-1` 的已知输入和历史状态构成"

但 `x_t+1|t` 中包含 `v_q,t`，而 `v_q,t` 是 `t` 时刻的观测，不是 "截至 `t-1`" 的已知输入。

**影响：** 实现者容易混淆 `x_t`、`x_t+1|t` 和主方程的索引，导致前瞻性偏差。

**建议：** 统一索引体系，明确区分：
- `x_t`：用于估计 `psi_t`（包含 `t-1` 及之前的信息）
- `x_t+1|t`：用于预测 `v_q,t+1`（包含 `t` 时刻的最新观测 `v_q,t`）
- 并在文档中给出清晰的时序图

---

### 缺陷3：残差记忆扩展的启用条件存在循环论证风险（第4.2节）

**问题描述：**
文档规定残差记忆扩展的启用条件：
1. 滚动残差一阶自相关 `abs(rho_eps_1)` 持续高于阈值（0.3）
2. Ljung-Box 检验未通过（p < 0.05）
3. 离线诊断显示有稳定改善
4. 改善不是由输入共线性或异常窗口驱动

但条件3 "离线诊断显示有稳定改善" 需要比较 "有残差记忆项" 和 "无残差记忆项" 两种模型，这本身就需要**在训练阶段使用未来信息或交叉验证**。如果实现者在比赛提交阶段动态判断是否启用，就构成了前瞻性偏差。

**影响：** 文档虽然规定 "在线阶段不自动切换模型结构"，但条件3的 "离线诊断" 与 "在线阶段" 的界限模糊，实现者可能误用。

**建议：** 明确区分：
- **训练前固定配置**：基于历史样本的离线实验，在训练前决定是否启用残差记忆扩展
- **在线监控**：只产生告警，不切换结构
- 比赛提交时必须使用训练前固定的配置，不得根据在线监控结果动态切换

---

### 缺陷4：模式切换的滞回规则缺乏理论依据（第5.3节）

**问题描述：**
文档给出模式切换滞回规则：
- 至少连续 `k=5` 个窗口通过可辨识性检验才允许从 4 维切换到 5 维
- 至少连续 `k=3` 个窗口失败才回退到 4 维
- 单个交易日内默认最多切换 2 次

并给出三组敏感性参数（A/B/C），要求 "最终在训练前固定一组配置"。

但**未说明为什么需要滞回规则**，以及为什么阈值是 5/3/2 而不是其他值。更关键的是，**模式切换本身会改变模型结构，导致参数序列不连续**，这在统计上是有问题的。

**影响：** 频繁的模式切换会导致参数估计的非平稳性，影响后续稳定性诊断和 `m_eff` 计算的有效性。

**建议：** 
- 补充说明滞回规则的目的是防止噪声导致的频繁切换
- 建议增加 "模式切换惩罚项" 或 "结构稳定性评分"，而非单纯依赖滞回计数
- 或者，建议实现者考虑 "软切换"（如贝叶斯模型平均）而非 "硬切换"

### 缺陷5：`closure_error` 的阈值设定缺乏量纲分析（第7.1节）

**问题描述：**
文档规定：
```
closure_error = v_q,t - (c_p + c_i + c_d + eps)
```

并建议：
```
closure_error_ratio = abs(closure_error) / max(1e-8, abs(v_q,t))
```

但 `closure_error` 的量纲是 "价格速度"（元/股/窗口），而 `1e-4` 作为阈值**未考虑不同股票的价格量级差异**。对于高价股（如 1000 元/股），`1e-4` 的绝对阈值可能过严；对于低价股（如 1 元/股），`1e-4` 可能过松。

**影响：** 不同价格量级的股票可能面临不一致的校验标准。

**建议：** 采用相对阈值或分组阈值，例如：
```
closure_error_ratio < 1e-6  # 相对于价格速度
```
或按价格分位分组设定阈值。

### 缺陷6：`beta_norm` 的归一化公式存在量纲问题（第6.1节）

**问题描述：**
文档定义：
```
beta_norm_* = (capital_* / P_state) / U_*_mv_ratio
```

其中：
- `capital_* = beta_* * U_*`，量纲取决于 `U_*` 的量纲
- `P_state` 的量纲是 元/股
- `U_*_mv_ratio` 的量纲是无量纲（市值占比）

如果 `U_*` 是 `U_*_mv_ratio`（市值占比），那么 `capital_*` 的量纲是 "价格速度 / 市值占比"？这与 `capital_*` 作为 "价格贡献" 的直觉不符。

更关键的是，文档说：
> "`capital_* / P_state` 必须明确表示归一化价格贡献"

但 `capital_*` 的量纲与 `P_state` 相同（都是价格速度），所以 `capital_* / P_state` 是无量纲的。那么 `beta_norm_* = (capital_* / P_state) / U_*_mv_ratio` 的量纲是 "1 / 市值占比"，这与 "归一化响应系数" 的直觉也不一致。

**影响：** 量纲混乱导致 `m_eff = 1 / beta_norm` 的物理/金融含义难以解释。

**建议：** 
- 明确 `capital_*` 的量纲定义（是 "价格变化贡献" 还是 "价格速度贡献"？）
- 重新定义 `beta_norm` 的量纲，使其具有清晰的金融含义，例如：
  ```
  beta_norm_* = (Delta_q P / P) / U_*_mv_ratio  # 收益率 / 市值占比流
  ```
- 或者，放弃 "归一化" 的物理类比，直接采用统计口径（如标准化回归系数）

---

### 缺陷7：工程校验清单第15条与文档内容矛盾

**问题描述：**
工程校验清单第15条：
> "`theta` 的联合稳定性需通过特征根或 Jury 条件检查；`abs(phi + theta) < 1` 只能作为快速筛查"

但在概念文档第6节中，Jury 条件是：
```
abs(theta) < 1
1 - phi > 0
1 + phi + 2 * theta > 0
```

注意：**概念文档中并未出现 `abs(phi + theta) < 1` 作为快速筛查的条件**。PID 文档在此处引入了一个新概念，但未解释其来源。

实际上，`abs(phi + theta) < 1` 是 Jury 条件的**必要条件但不是充分条件**。文档说 "只能作为快速筛查" 是对的，但应该说明为什么（即它不能保证 `abs(theta) < 1` 和 `1 + phi + 2*theta > 0`）。

**影响：** 实现者可能误以为 `abs(phi + theta) < 1` 是充分条件，导致稳定性误判。

**建议：** 明确说明 `abs(phi + theta) < 1` 是必要条件，并给出反例（例如 `phi = 0.5, theta = -0.9`，满足 `abs(phi + theta) = 0.4 < 1`，但 `1 + phi + 2*theta = 1 + 0.5 - 1.8 = -0.3 < 0`，不满足 Jury 条件）。

## 四、跨文档一致性问题

### 问题1：`m_slow` 的提取方法优先级不一致

- **概念文档第7.4节**：`ewma_realtime` > `rolling_median_realtime` > `kalman_trend_realtime`
- **PID 文档第6.1节**：`ewma_realtime` > `rolling_median_realtime` > `kalman_trend_realtime`

表面一致，但 PID 文档未明确区分 "滤波" 与 "平滑"，而概念文档提到了 "RTS 平滑" 和 "HP 平滑"。建议统一术语。

### 问题2：`beta_norm` 的正式口径不一致

- **概念文档第7.2节**：`beta_norm_* = (capital_* / P_state) / U_*_mv_ratio`
- **PID 文档第6.1节**：`beta_norm_* = (capital_* / P) / (U_* / market_value)`，并补充说明 `capital_* / P_state` 必须标记为 `price_contribution`

PID 文档更详细，但两份文档的公式形式不一致（`P_state` vs `P`，`U_*_mv_ratio` vs `U_* / market_value`）。建议统一符号。

### 问题3：稳定性诊断阈值不一致

- **概念文档第7.3节**：未给出具体阈值
- **PID 文档第10节**：给出详细阈值表格（如 `param_cv_7d < 1.0`，`m_eff_iqr / abs(m_eff_median) < 0.5`）

建议概念文档引用 PID 文档的阈值，或统一合并到一份文档中。
