## 二、V1.1 更新亮点

### 2.1 一级 PID 贡献显式定义（✅ 重大改进）

V1.1 新增第 5.3 节"一级 PID 贡献计算"，明确定义：

```
C_P,t = β_ch,t U_ch,t-1 + β_q,t U_q,t-1 + β_retail,t U_retail,t-1
C_I,t = φ_t ΔP_t-1 + κ_I ε_t-1
C_D,t = θ_t D_driver_t
```

**改进价值：**
- 解决了前序评审中"P/I/D 贡献在结构层未显式输出"的问题
- 新增 `c_p / c_i / c_d / eps` 四个输出字段，便于调试和验证
- 明确闭合关系 `ΔP_t = C_P,t + C_I,t + C_D,t + ε_t`，数学上更严谨

### 2.2 D 驱动项有效性判定（✅ 重大改进）

V1.1 新增 4.7.1 节，提供 D_driver 有效性的量化门槛：

| 检查项 | 建议门槛 |
|--------|----------|
| 非零覆盖率 | abs(D_driver) > eps 的窗口占比 >= 30% |
| 与惯性项相关性 | abs(corr(D_driver, delta_P_lag1)) < 0.85 |
| 与代理流相关性 | 与任一代理流输入的绝对相关系数 < 0.85 |
| 方差占比 | std(D_driver) / std(delta_P_lag1) 处于 0.1 到 10 之间 |
| 多起点稳定性 | theta 多起点估计方向一致 |

**改进价值：**
- 将"D_driver 是否有效"从定性判断转为定量判定
- 明确未通过时的处理路径：引入盘口增强特征或回退 4 维
- 提供增强驱动项公式：`D_driver_t^enhanced = a1·Δ²P_t + a2·book_recovery_t + ...`

### 2.3 基线版交付策略（✅ 重大改进）

V1.1 新增 12.0.1 节，定义三层版本：

| 版本 | 启用条件 | 能力 |
|------|----------|------|
| baseline_rule | 阶段A完成 | 阈值分类 + 展示层规则兜底，不启用状态空间模型 |
| baseline_4d | 阶段A通过，阶段B中5维未通过 | 4维 ARX/KF/RTS + I 残差记忆，不启用 D 结构拆分 |
| enhanced_5d | 阶段A/B/D均通过 | 5维状态模型 + D 驱动拆分 + λ 敏感性通过 |

**改进价值：**
- 解决了前序评审中"如何优先保证提交能力"的问题
- 阶段A结束即可产出 baseline_rule，确保比赛有输出
- 避免因追求完整 PID 结构而影响整体交付

### 2.4 输出层级约定（✅ 重要改进）

V1.1 新增 11.0 节"输出层级约定"：

```
一级 PID 贡献（c_p / c_i / c_d / eps）→ 判断价格变化由推进/惯性/阻尼/未解释扰动主导
二级资金代理贡献（delta_ch_struct / delta_q_struct / delta_retail_struct）→ 判断三类行为代理对结构贡献的分摊
```

**改进价值：**
- 明确资金解释必须先检查一级 PID，再检查二级资金代理
- 若 abs(eps) 或 noise_ratio 过高，即使某类资金代理贡献最大，也不得给出高置信度强解释
- 避免"噪声窗口强行归因"的业务误读

### 2.5 伪代码完善（✅ 细节改进）

- 已增加闭合性断言（结构层 + 展示层）
- λ 参数已改为 config 读取
- 增加 `c_p / c_i / c_d / eps` 输出字段

### 建议1：增强驱动项的 a_i 权重标定（建议补充）

**问题：** 4.7.1 节提出增强驱动项公式：
```
D_driver_t^enhanced = a1·Δ²P_t + a2·book_recovery_t + a3·spread_converge_t + a4·cancel_replenish_t
```

但仅说明"优先采用标准化后等权或少量网格搜索"，未给出具体标定方法。

**建议：** 补充增强驱动项的权重标定策略：
```markdown
1. 首版默认采用等权：a1 = a2 = a3 = a4 = 0.25
2. 若盘口特征可得，通过网格搜索在 {0.1, 0.2, 0.3, 0.4} 范围内优化
3. 优化目标：最大化 D_driver 与 ΔP_t-1 的独立性（最小化相关性）
4. 约束条件：a1 + a2 + a3 + a4 = 1，且 ai >= 0
```

**理由：**
- 明确首版默认值，避免实现时犹豫
- 网格搜索范围小，计算量可控
- 优化目标明确（独立性最大化），而非模糊地"提升有效性"

### 建议2：kappa_i 的敏感性验证计划（建议补充）

**问题：** 12.1 节将 `kappa_i` 列为 P2 风险，但未给出具体验证计划。

**建议：** 补充 kappa_i 验证计划：
```markdown
### kappa_i 验证计划
1. 固定其他参数，在 [0, 0.25, 0.5, 0.75, 1.0] 范围内扫描 kappa_i
2. 评估指标：
   - 残差收敛性（|ε̄_20| 是否 < 0.005）
   - 展示层与结构层偏离（display_shift < 0.2 的窗口占比）
```

**理由：**
- 明确 kappa_i 不是"残差最小化"参数，而是"解释稳定性"参数
- 避免编码阶段将 kappa_i 当作自由拟合参数调优

### 建议5：伪代码中 y_adjusted 的注释澄清（建议补充）

**问题：** 伪代码中：
```python
y_adjusted = delta_P[t] - KAPPA_I * eps_prev_for_filter
innovation = y_adjusted - H_t @ psi_pred
```

此处 `y_adjusted` 的调整逻辑需要更明确注释。

**建议：** 补充注释：
```python
# 将当期观测值减去 I 环节残差记忆的已知贡献
# 使 innovation 仅反映 P 冲击和 D 阻尼的未解释部分
# 注意：这不会使 eps 被重复扣除，因为 eps 本身是通过 H_t @ psi_update 估计的
y_adjusted = delta_P[t] - KAPPA_I * eps_prev_for_filter
```

**理由：**
- 避免实现者误解为"双重扣除残差"
- 明确 y_adjusted 的目的是"分离已知贡献，使 innovation 更纯净"
