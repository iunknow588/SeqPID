# 统一资金类型判断参数配置表

文档版本：V1.0  
文档状态：工程配置版  
前置文档：`统一资金类型判断方法.md`

## 1. 文档目标

本文档用于将“统一资金类型判断方法”中的规则口径转化为可配置、可回放、可标定的工程参数。

目标是避免：

1. 规则散落在代码中硬编码
2. 买入与卖出口径不一致
3. 不同股票、不同阶段无法统一回放

## 2. 配置总原则

### 2.1 行为优先，金额辅助

参数配置应优先服务以下三类行为识别：

1. 快速影响价格：游资
2. 缓慢改变价格：量化
3. 适应既有走势：散户

因此：

1. 主动性、跨价深度、价格冲击强度参数优先级最高
2. 金额阈值仅作为增强项
3. 不能仅凭金额大小直接决定资金类型

### 2.2 固定阈值与相对阈值并存

建议所有规模类参数都同时保留：

1. 固定金额阈值
2. 历史分位数阈值
3. 相对流通市值阈值

最终由配置指定采用：

- `max`
- `min`
- `or`
- `and`

中的哪种组合方式。

### 2.3 参数必须可回放

所有参数都必须：

1. 可在历史样本上扫描
2. 可在报告中记录
3. 可与模式切换日志关联

## 3. 推荐配置层级

建议配置文件结构如下：

```yaml
capital_classifier:
  general: {}
  continuous_auction: {}
  call_auction: {}
  thresholds: {}
  confidence: {}
  fallback: {}
  output: {}
```

## 4. 总体参数表

## 4.1 通用参数

| 参数名 | 默认值 | 含义 | 建议说明 |
| --- | --- | --- | --- |
| `tick_mode` | `auto` | 最小价位获取方式 | 优先按证券真实 tick 规则 |
| `window_minutes` | `5` | 窗口聚合长度 | 与主 PID 保持一致 |
| `signed_amount_mode` | `net` | 有符号金额统计方式 | 买入为正、卖出为负 |
| `normalize_before_pid` | `true` | 进入 PID 前是否标准化 | 建议开启 |
| `winsorize_pct` | `0.01` | 缩尾比例 | 建议 1% |
| `missing_fill_value` | `0.0` | 缺失值填充值 | 建议补零 |

## 4.2 连续竞价行为参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `buy_aggressive_ticks` | `3` | 主动强攻击买入最小跨价档数 |
| `sell_aggressive_ticks` | `3` | 主动强攻击卖出最小砸盘档数 |
| `buy_normal_ticks_min` | `1` | 正常主动买入最小跨价档数 |
| `sell_normal_ticks_min` | `1` | 正常主动卖出最小砸盘档数 |
| `price_impact_high_pct` | `0.002` | 强价格冲击阈值 |
| `price_impact_mid_pct` | `0.0008` | 中等价格冲击阈值 |
| `follow_lag_seconds` | `20` | 判断“跟随走势”的参考滞后窗口 |
| `sustain_trade_count_min` | `3` | 判断“持续缓慢改变价格”的最小连续笔数 |
| `post_impact_recovery_seconds` | `30` | 盘口回补观察窗口 |

说明：

1. `buy_aggressive_ticks` / `sell_aggressive_ticks` 主要服务游资识别
2. `sustain_trade_count_min` 主要服务量化识别
3. `follow_lag_seconds` 主要服务散户“适应走势”识别

## 4.3 金额规模参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `large_amount_fixed` | `500000` | 大额固定阈值 |
| `small_amount_fixed` | `100000` | 小额固定阈值 |
| `large_float_mv_ratio` | `0.0001` | 大额相对流通市值比例，默认 0.01% |
| `small_float_mv_ratio` | `0.00002` | 小额相对流通市值比例，默认 0.002% |
| `large_amount_percentile` | `0.95` | 历史金额分位数大额阈值 |
| `mid_amount_percentile` | `0.70` | 历史金额分位数中额阈值 |
| `small_amount_percentile` | `0.30` | 历史金额分位数小额阈值 |
| `large_amount_rule` | `or` | 固定阈值、相对阈值、分位阈值的组合方式 |
| `small_amount_rule` | `or` | 小额阈值组合方式 |

建议定义：

\[
V_{large} = \max(large\_amount\_fixed,\ large\_float\_mv\_ratio \times float\_mv)
\]

\[
V_{small} = \min(small\_amount\_fixed,\ small\_float\_mv\_ratio \times float\_mv)
\]

若历史分位数可用，再叠加分位条件校验。

## 4.4 集合竞价参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `auction_open_cancelable_start` | `09:15:00` | 开盘集合竞价可撤单阶段开始 |
| `auction_open_cancelable_end` | `09:20:00` | 开盘集合竞价可撤单阶段结束 |
| `auction_open_locked_start` | `09:20:00` | 开盘集合竞价不可撤单阶段开始 |
| `auction_open_locked_end` | `09:25:00` | 开盘集合竞价不可撤单阶段结束 |
| `auction_close_locked_start` | `14:57:00` | 收盘集合竞价开始 |
| `auction_close_locked_end` | `15:00:00` | 收盘集合竞价结束 |
| `virtual_deviation_high` | `0.03` | 虚拟成交价大偏离阈值 |
| `virtual_deviation_mid` | `0.01` | 虚拟成交价中偏离阈值 |
| `locked_net_ratio_min` | `0.35` | 不可撤单阶段净新增占总委托比例最小门槛 |
| `auction_large_amount_fixed` | `500000` | 集合竞价大额阈值 |
| `auction_small_amount_fixed` | `100000` | 集合竞价小额阈值 |

说明：

1. `virtual_deviation_high` 高于该值且集中发生，偏向游资
2. `virtual_deviation_mid` 到 `virtual_deviation_high` 之间，偏向量化
3. 小额且跟随已有高开/低开预期，偏向散户

## 4.5 试盘/虚假挂单参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `fake_order_large_amount` | `500000` | 判定试盘的大额阈值 |
| `fake_order_cancel_ratio_min` | `0.8` | 可撤单阶段大额挂单撤销比例门槛 |
| `fake_order_price_push_min` | `0.01` | 对虚拟成交价的最小推动幅度 |
| `fake_order_keep_in_locked_min` | `0.2` | 不可撤单阶段最低保留比例 |

若满足：

1. 大额挂单
2. 明显推动虚拟成交价
3. 可撤单末期大量撤销
4. 不可撤单阶段保留比例低

则记为：

- `trial_probe`
- `fake_order_signal`

但不进入主净额。

## 4.6 涨跌停附近修正参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `near_limit_up_ratio` | `0.98` | 接近涨停判定比例 |
| `near_limit_down_ratio` | `1.02` | 接近跌停判定比例 |
| `limit_zone_aggressive_tick_relax` | `1` | 极端价格区放宽档位阈值 |
| `limit_zone_confidence_decay` | `0.15` | 涨跌停附近置信度衰减 |

修正规则：

1. 接近涨停的大额主动买入：游资置信度可上调
2. 接近跌停的大额主动卖出：游资置信度可上调
3. 小额追涨/恐慌单：散户置信度上调
4. 无法产生真实深跨价时：强规则结果整体降权

## 4.7 量化“缓慢改变走势”参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `quant_sustain_windows_min` | `2` | 最少持续窗口数 |
| `quant_signed_flow_stability_min` | `0.6` | 同方向流量稳定性门槛 |
| `quant_avg_impact_max` | `0.0015` | 平均单笔冲击上限 |
| `quant_recovery_speed_max` | `60` | 回补完成最大秒数 |
| `quant_split_order_count_min` | `3` | 分批执行最小笔数 |

满足以下越多，越偏向量化：

1. 连续多个窗口同方向
2. 每笔冲击不大
3. 分批执行明显
4. 盘口后效应平滑

## 4.8 散户“适应走势”参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `retail_small_amount_max` | `100000` | 散户小额上限 |
| `retail_follow_return_min` | `0.003` | 跟随已有价格变化的最小阈值 |
| `retail_follow_time_lag_max` | `60` | 追涨杀跌最大延后时间 |
| `retail_fragment_count_min` | `2` | 零散成交最小笔数 |
| `retail_price_chase_ticks_min` | `1` | 小额追价/砸盘最小档数 |

满足以下越多，越偏向散户：

1. 小额
2. 价格已先动
3. 后续才跟进
4. 行为零散

## 4.9 游资“快速影响走势”参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `hot_money_large_amount_min` | `500000` | 游资强锚点金额下限 |
| `hot_money_ticks_min` | `3` | 游资强攻击最小档位 |
| `hot_money_price_shift_min` | `0.0015` | 触发快速影响走势的最小价格位移 |
| `hot_money_time_cluster_seconds` | `15` | 事件聚簇时间窗口 |
| `hot_money_follow_through_min` | `0.3` | 冲击后持续成交最低比例 |

## 5. 置信度参数

## 5.1 评分项权重

建议配置：

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `score_weight_aggressive` | `0.35` | 主动攻击性权重 |
| `score_weight_amount` | `0.20` | 金额规模权重 |
| `score_weight_sustain` | `0.20` | 持续性权重 |
| `score_weight_follow` | `0.15` | 跟随/适应性权重 |
| `score_weight_reliability` | `0.10` | 方向恢复可靠性权重 |

要求：

\[
\sum w_i = 1
\]

## 5.2 置信度分层

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `confidence_high_min` | `0.80` | 高置信阈值 |
| `confidence_medium_min` | `0.60` | 中置信阈值 |
| `confidence_low_min` | `0.40` | 低置信阈值 |
| `confidence_unknown_below` | `0.40` | 低于该值不做强分类 |

## 5.3 降权项

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `degrade_if_direction_unreliable` | `0.20` | 主动方向恢复不稳时降权 |
| `degrade_if_limit_zone` | `0.15` | 涨跌停附近降权 |
| `degrade_if_single_trade_only` | `0.10` | 单笔极端但无持续时降权 |
| `degrade_if_auction_cancel_noise` | `0.20` | 集合竞价撤单扰动高时降权 |

## 6. 回退与混合池参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `force_mix_if_conflict` | `true` | 规则冲突时是否进入混合池 |
| `force_mix_if_confidence_below` | `0.55` | 低于该值优先放入混合池 |
| `force_unknown_if_direction_reliability_below` | `0.50` | 方向恢复太差时不强判 |
| `mix_bucket_keep_signed` | `true` | 混合池是否保留方向符号 |

建议：

1. 冲突规则不直接强判
2. 低置信样本优先进入 `M_qr`
3. 高置信样本才进入强锚点或种子流

## 7. 输出控制参数

| 参数名 | 默认值 | 含义 |
| --- | --- | --- |
| `emit_rule_reason` | `true` | 是否输出命中规则说明 |
| `emit_confidence_score` | `true` | 是否输出置信度分数 |
| `emit_confidence_level` | `true` | 是否输出高/中/低标签 |
| `emit_trace_fields` | `true` | 是否输出调试字段 |

建议输出调试字段包括：

1. 命中的主要规则编号
2. 金额等级
3. 主动性等级
4. 价格冲击等级
5. 最终分类前得分
6. 降权原因

## 8. 推荐 YAML 样例

```yaml
capital_classifier:
  general:
    window_minutes: 5
    normalize_before_pid: true
    winsorize_pct: 0.01
  continuous_auction:
    buy_aggressive_ticks: 3
    sell_aggressive_ticks: 3
    buy_normal_ticks_min: 1
    sell_normal_ticks_min: 1
    price_impact_high_pct: 0.002
    price_impact_mid_pct: 0.0008
    follow_lag_seconds: 20
    sustain_trade_count_min: 3
  thresholds:
    large_amount_fixed: 500000
    small_amount_fixed: 100000
    large_float_mv_ratio: 0.0001
    small_float_mv_ratio: 0.00002
    large_amount_percentile: 0.95
    mid_amount_percentile: 0.70
    small_amount_percentile: 0.30
  call_auction:
    virtual_deviation_high: 0.03
    virtual_deviation_mid: 0.01
    locked_net_ratio_min: 0.35
  confidence:
    confidence_high_min: 0.80
    confidence_medium_min: 0.60
    confidence_low_min: 0.40
    degrade_if_direction_unreliable: 0.20
    degrade_if_limit_zone: 0.15
  fallback:
    force_mix_if_conflict: true
    force_mix_if_confidence_below: 0.55
```

## 9. 标定建议

参数标定顺序建议如下：

1. 先标定行为参数：主动性、跨价、价格冲击
2. 再标定金额参数
3. 再标定集合竞价参数
4. 最后标定置信度和回退阈值

推荐最先扫描的参数：

1. `buy_aggressive_ticks / sell_aggressive_ticks`
2. `large_amount_fixed`
3. `virtual_deviation_high`
4. `force_mix_if_confidence_below`
5. `hot_money_time_cluster_seconds`

## 10. 版本要求

后续若进入代码实现，要求：

1. 所有参数有默认值
2. 所有参数能被配置覆盖
3. 所有参数修改能记录到回放报告
4. 所有参数修改不改变字段定义

也就是说：

> 可调的是阈值，不可漂移的是口径。
