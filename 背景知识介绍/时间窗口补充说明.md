# 时间窗口补充说明

## 时间窗口的核心作用

**所有 L2 指标都必须在明确的时间窗口内统计才有意义。**

以 `oss_mega_amount_pct`（超大单成交金额占比）为例：

```
错误理解："今天超大单占比 30%"
正确理解："今天 09:30-10:00 时段，超大单成交金额占该时段总成交金额的 30%"
```

## 窗口类型详解

### 1. 固定时段窗口

| 时段 | 用途 |
|------|------|
| 09:30-10:00 | 开盘后流动性分析 |
| 10:00-11:30 | 上午盘中分析 |
| 13:00-14:30 | 下午盘中分析 |
| 14:30-15:00 | 收盘前效应分析 |

**特点**：便于横向对比（同一股票不同时段、不同股票同一时段）

### 2. 事件窗口

```python
# 信号触发前 5 分钟
event_time = '2026-07-05 10:15:00'
window_start = event_time - 5分钟  # 10:10:00
window_end = event_time             # 10:15:00

# 信号触发后 5 分钟
window_start = event_time           # 10:15:00
window_end = event_time + 5分钟     # 10:20:00
```

**特点**：分析事件前后的市场结构变化

### 3. 滚动窗口

```python
# 每 1 分钟滚动一次，窗口长度 5 分钟
for t in ['09:30', '09:31', '09:32', ...]:
    window_start = t - 5分钟
    window_end = t
    # 计算该窗口的 oss_mega_amount_pct
```

**特点**：实时监控，捕捉动态变化

### 4. 全天窗口

```python
window_start = '09:30:00'
window_end = '15:00:00'
```

**特点**：日度汇总，适合盘后分析

## 窗口选择的影响

同一股票在不同窗口下的超大单占比可能截然不同：

| 窗口 | 超大单占比 | 解读 |
|------|-----------|------|
| 09:30-10:00 | 45% | 开盘机构集中交易 |
| 10:00-11:30 | 15% | 盘中散户主导 |
| 14:30-15:00 | 30% | 收盘前机构调仓 |

**结论**：脱离时间窗口谈"超大单占比"是没有意义的。

## 实际应用建议

1. **策略回测**：使用固定时段窗口，确保可比性
2. **实时监控**：使用滚动窗口，捕捉最新变化
3. **事件分析**：使用事件窗口，分析因果关系
4. **日度报告**：使用全天窗口，汇总当日特征

## 字段关联

| 字段 | 窗口依赖 | 说明 |
|------|----------|------|
| `oss_mega_amount_pct` | 是 | 窗口内超大单金额/窗口内总金额 |
| `rs_interval_mean_ms` | 是 | 窗口内订单间隔均值 |
| `cb_cancel_order_ratio` | 是 | 窗口内撤单率 |
| `ap_active_buy_pct` | 是 | 窗口内主动买入占比 |
| `obp_at_best_bid_ratio` | 是 | 窗口末订单簿状态 |
| `pd_Q1_ratio` | 是 | 窗口内价格发现过程 |
| `pi_price_std_pct` | 是 | 窗口内价格波动 |

**基础字段**（不依赖窗口）：
| 字段 | 说明 |
|------|------|
| `date` | 交易日期 |
| `symbol` | 股票代码 |
| `window_start` / `window_end` | 窗口边界定义 |
