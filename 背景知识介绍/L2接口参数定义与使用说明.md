# L2 接口参数定义与使用说明

## 概述

本文档定义了 L2 行情接口的参考特征集（Reference Feature Set），包含 85 个字段，涵盖订单流、成交结构、撤单行为、主动买卖、挂单分布、价格发现、价格冲击等七大维度。所有字段均基于 L2 逐笔委托与逐笔成交数据计算，适用于自动化交易中的实时风控、策略信号生成与交易执行优化。

## 字段分类

### 1. 基础信息（4 字段）

| 字段名 | 类型 | 说明 | 使用场景 |
|--------|------|------|----------|
| `date` | Date | 交易日期 | 数据分区、回测对齐 |
| `symbol` | String | 股票代码 | 标的筛选、组合管理 |
| `window_start` | Time | 窗口开始时间 | 时间切片、事件对齐 |
| `window_end` | Time | 窗口结束时间 | 时间切片、事件对齐 |
| `window_start_dt` | DateTime | 窗口起始日期时间（精确） | 微秒级时序分析 |
| `window_end_dt` | DateTime | 窗口结束日期时间（精确） | 微秒级时序分析 |

**使用示例**：
```python
# 筛选特定日期和时段的数据
df = df[(df['date'] == '2026-07-05') & 
        (df['window_start'] >= '09:30:00') &
        (df['window_end'] <= '10:00:00')]
```

### 2. 订单与成交统计（9 字段）

| 字段名 | 类型 | 说明 | 计算逻辑 | 使用场景 |
|--------|------|------|----------|----------|
| `order_count` | Int | 订单数量（委托笔数） | 窗口内有效委托笔数 | 流动性评估 |
| `order_count_all` | Int | 所有订单数量（含撤单） | 全部委托笔数 | 市场参与度 |
| `cancel_count_all` | Int | 撤单总笔数 | 全部撤单笔数 | 撤单压力 |
| `deal_count` | Int | 成交笔数 | 逐笔成交计数 | 活跃度 |
| `deal_amount` | Float | 成交金额 | 逐笔成交金额累加 | 成交额分析 |
| `total_deal_amount_all` | Float | 全部成交金额（总计） | 含所有成交类型 | 总成交额基准 |
| `signal_deal_buy_amount` | Float | 信号成交买入金额 | 策略信号触发的买入 | 策略执行质量 |
| `signal_deal_sell_amount` | Float | 信号成交卖出金额 | 策略信号触发的卖出 | 策略执行质量 |

**使用示例**：
```python
# 计算有效委托率
df['effective_order_ratio'] = df['order_count'] / df['order_count_all']

# 计算策略成交占比
df['signal_deal_ratio'] = (df['signal_deal_buy_amount'] + df['signal_deal_sell_amount']) / df['total_deal_amount_all']
```

### 3. 订单规模分布（OSS：Order Size Spectrum，12 字段）

**时间窗口说明**：

`window_start` 和 `window_end` 定义了统计超大单、大单、中单、小单的**时间窗口**。所有 OSS 字段均基于该窗口内的逐笔成交数据计算。

**典型窗口类型**：

| 窗口类型 | 示例 | 用途 |
|----------|------|------|
| 固定时段 | 09:30:00 - 10:00:00 | 开盘后时段分析 |
| 事件窗口 | 信号触发前 5 分钟 | 信号前序分析 |
| 滚动窗口 | 过去 5 分钟 | 实时滚动监控 |
| 全天窗口 | 09:30:00 - 15:00:00 | 日度汇总 |

**超大单定义**：通常 > 100 万股或 > 100 万元（具体阈值因交易所/券商系统而异）
**大单定义**：通常 20-100 万股或 20-100 万元
**中单定义**：通常 5-20 万股或 5-20 万元
**小单定义**：通常 < 5 万股或 < 5 万元

**计算逻辑**：

```
oss_mega_amount_pct = 窗口内超大单成交金额 / 窗口内总成交金额
oss_mega_count_pct = 窗口内超大单成交笔数 / 窗口内总成交笔数
```

**使用示例**：

```python
# 筛选开盘后 30 分钟的超大单占比
df = df[(df['window_start'] == '09:30:00') & 
        (df['window_end'] == '10:00:00')]

# 识别机构主导时段（超大单+大单占比 > 60%）
df['institutional_dominant'] = (df['oss_mega_amount_pct'] + df['oss_large_amount_pct']) > 0.6

# 滚动窗口计算（假设已有逐笔数据）
def rolling_oss(df, window_minutes=5):
    df['window_start'] = df['trade_time'].dt.floor(f'{window_minutes}min')
    df['window_end'] = df['window_start'] + pd.Timedelta(minutes=window_minutes)
    
    grouped = df.groupby(['symbol', 'window_start', 'window_end'])
    
    result = pd.DataFrame({
        'oss_mega_amount_pct': grouped.apply(
            lambda x: x[x['amount'] > 1_000_000]['amount'].sum() / x['amount'].sum()
        ),
        'oss_large_amount_pct': grouped.apply(
            lambda x: x[(x['amount'] > 200_000) & (x['amount'] <= 1_000_000)]['amount'].sum() / x['amount'].sum()
        )
    })
    
    return result.reset_index()
```

### 4. 订单节奏与拆分（RS：Rhythm & Split，7 字段）

| 字段名 | 类型 | 说明 | 计算逻辑 | 使用场景 |
|--------|------|------|----------|----------|
| `rs_interval_mean_ms` | Float | 订单/成交间隔均值（毫秒） | 时间间隔平均 | 交易节奏 |
| `rs_interval_median_ms` | Float | 订单/成交间隔中位数（毫秒） | 时间间隔中位数 | 稳健节奏度量 |
| `rs_interval_cv` | Float | 订单/成交间隔变异系数 | 标准差/均值 | 节奏稳定性 |
| `rs_burst_ratio` | Float | 订单/成交爆发比率 | 峰值速率/平均速率 | 突发交易检测 |
| `rs_buy_interval_cv` | Float | 买入订单间隔变异系数 | 买入间隔标准差/均值 | 买入节奏稳定性 |
| `rs_sell_interval_cv` | Float | 卖出订单间隔变异系数 | 卖出间隔标准差/均值 | 卖出节奏稳定性 |
| `rs_split_similarity` | Float | 订单拆分相似度 | 订单大小分布相似性 | 拆单检测 |
| `rs_split_run_ratio` | Float | 订单拆分连续运行比率 | 连续相似订单占比 | 算法交易识别 |

**使用示例**：
```python
# 检测算法交易
df['algo_trading_signal'] = (df['rs_split_similarity'] > 0.8) & (df['rs_split_run_ratio'] > 0.5)

# 识别突发交易

```

## 综合使用场景

### 场景1：实时风控

```python
def risk_alert(row):
    alerts = []
    if row['cb_cancel_order_ratio'] > 0.8:
        alerts.append('高撤单率')
    if row['rs_burst_ratio'] > 5.0:
        alerts.append('交易爆发')
    if row['pi_max_price_impact_pct'] > 0.02:
        alerts.append('价格冲击过大')
    if row['ap_unilateral_intensity'] > 0.7:
        alerts.append('单边交易')
    return alerts
```

### 场景2：策略信号生成

```python
def generate_signal(row):
    # 多头信号
    if (row['ap_active_net_direction'] > 0.3 and 
        row['oss_mega_buy_pct'] > 0.6 and
        row['pd_Q2_ratio'] > 0.5):
        return 'BUY'
    
    # 空头信号
    if (row['ap_active_net_direction'] < -0.3 and 
        row['oss_sell_amount_pct'] > 0.6):
        return 'SELL'
    
    return 'HOLD'
```

### 场景3：执行优化

```python
def execution_quality(row):
    # 计算滑点
    slippage = row['pi_vwap_deviation']
    
    # 评估市场冲击
    market_impact = row['pi_max_price_impact_pct']
    
    # 判断时机
    good_timing = (row['obp_buy_pressure'] > row['obp_sell_pressure'] and
                   row['ap_active_buy_run_max'] >= 3)
    
    return {
        'slippage': slippage,
        'market_impact': market_impact,
        'good_timing': good_timing
    }
```

## 数据质量检查

```python
def data_quality_check(df):
    checks = {
        'missing_values': df.isnull().sum(),
        'negative_amounts': (df['deal_amount'] < 0).sum(),
        'invalid_ratios': ((df['oss_mega_amount_pct'] < 0) | (df['oss_mega_amount_pct'] > 1)).sum(),
        'future_timestamps': (df['window_end_dt'] > datetime.now()).sum()
    }
    return checks
```

## 注意事项

1. **时间窗口定义**：所有指标（尤其是 OSS、RS、CB、AP、OBP、PD、PI）均基于 `window_start` 和 `window_end` 定义的**时间窗口**计算。窗口长度和位置直接影响指标含义：
   - **固定时段窗口**（如 09:30-10:00）：适合时段对比分析
   - **事件窗口**（如信号前 5 分钟）：适合前序分析
   - **滚动窗口**（如过去 5 分钟）：适合实时监控
   - **全天窗口**（09:30-15:00）：适合日度汇总
   
   窗口选择应根据具体策略需求确定，不同窗口的指标不可直接比较。

2. **比率字段**：所有 `_pct` 和 `_ratio` 字段为 0-1 之间的小数，使用时注意与百分比区分
3. **变异系数**：`_cv` 字段为无量纲指标，值越大表示波动性越强
4. **阈值参数**：`mega_threshold`、`large_threshold` 等阈值可能因股票类型（主板/科创板/创业板）而异
5. **实时计算**：L2 数据为逐笔级别，窗口指标需要流式计算或微批处理
