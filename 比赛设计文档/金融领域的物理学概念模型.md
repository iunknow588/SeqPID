# 金融领域的物理学概念模型

文档定位：股票价格预测的基础概念模型  
适用范围：价格状态定义、成交量交易时间、金融-物理映射、候选稳定量构造  
整理来源：`股价质地参数说明.md`、`类比说明.md`、`说明2.md`、`统一.md`

---

## 1. 文档目的

本文用于统一股票价格预测系统中的基础物理类比口径，回答以下问题：

1. 股价、成交量、成交金额分别映射到什么物理对象
2. 成交量是否可以作为交易时间
3. 股价能否类比为速度
4. 时间域价格变化与成交量域价格变化之间如何转换
5. 哪些量可以作为候选稳定量，而哪些不能直接称为不变量

本文只定义概念模型，不描述具体 PID 状态估计算法。PID 过程控制模型见 [PID算法与实现.md](PID算法与实现.md)。

### 1.1 符号约定

为避免不同空间中同一符号造成误读，本文采用如下约定：

| 符号 | 含义 | 所在空间 |
| --- | --- | --- |
| `P_wv` | `W-V` 空间中的窗口边际价格，工程上取 `Delta W / Delta V` | 成交金额-成交量空间 |
| `P_state(q)` | 交易时间轴上的价格状态路径 | 价格状态-交易时间空间 |
| `Delta_q P_state` | 交易时间轴上的一阶差分 | 价格状态-交易时间空间 |
| `beta_*` | 行为代理流对价格变化的加载系数 | 状态方程 / 工程层 |

若上下文未特别区分，`P` 仅作为简写使用；实现、建模和接口命名建议显式采用 `P_wv` 与 `P_state`。

## 2. 总体结论

当前系统采用双层映射。两层中的价格观测原则上来自同一条窗口价格序列；`P_wv` 与 `P_state` 是不同分析空间下的表示，不应在代码中维护两套互相独立的价格数据。

工程口径进一步规定：实时状态、预测和提交统一使用 `P_state`；`P_wv_window` 只作为成交量域的窗口诊断价格。二者不应被无条件视为相等。

建议记录：

```text
price_basis_error = P_wv_window - P_state
price_basis_error_ratio = abs(price_basis_error) / max(1e-8, abs(P_state))
price_basis_warning = price_basis_error_ratio > config.price_basis_error_ratio_limit
```

当窗口内成交价分布高度不均匀、盘口发生断层或 `price_basis_warning = true` 时，禁止用 `P_wv_window` 替换实时 `P_state`。

1. **成交金额-成交量空间**
   - `W(V)` 表示成交金额关于成交量的累计函数
   - `P_wv = dW / dV` 是连续近似记号，工程上取 `Delta W / Delta V` 表示窗口边际价格
   - `dP_wv / dV = d^2W / dV^2` 表示单位成交量推进下的价格敏感度

2. **价格状态-交易时间空间**
   - 成交量窗口或累计成交量进度可以视为内部交易时间 `q`
   - `P_state(q)` 表示交易时间轴上的价格状态
   - `Delta_q P_state` 表示价格在交易时间上的一阶变化
   - `Delta_q^2 P_state` 表示价格在交易时间上的二阶变化

因此，准确表述应为：

- `P_wv_window` 是窗口成交量加权平均价格；它只能与物理速度建立启发式类比，不能称为严格数学同构
- 但在价格状态动力学中，`P_state(q)` 仍是系统状态量
- `Delta_q P_state` 才是价格状态空间中的速度
- `Delta_q^2 P_state` 才是价格状态空间中的加速度

不能把“`P_wv = dW / dV` 与速度同构”直接改写成“股价就是速度”，也不能进一步把系统主方程中的 `Delta_q P_state` 直接称为加速度。

### 2.1 对统一框架的吸收原则

`统一.md` 的核心精神可概括为：以 `Delta W / Delta V` 作为窗口价格的工程构造，以 `dV/dt` 连接成交量域与自然时间域，以 `beta` 的归一化倒数构造候选响应指标。

本系统吸收该思路时采用三条边界：

1. **类比不等于同构**：`P_wv_window` 可以在 `W-V` 表示中提供速度类比直觉，但不把 `W`、`V` 视为独立物理坐标，也不把该类比当作严格证明。
2. **统一不等于严格证明**：`F = ma`、PID 与状态空间方程可以共享变量解释，但必须经过闭合、残差和稳定性诊断。
3. **慢变不等于不变量**：`m_slow` 可以作为候选质地参数，只有通过跨日稳定性检验后才进入画像口径。

## 3. 成交金额-成交量空间

### 3.1 基础定义

设：

- `W`：累计成交金额
- `V`：累计成交量
- `P_wv`：成交价格或边际价格

则：

```text
P_wv = dW / dV                  # 连续近似记号
P_wv_window = Delta W / Delta V # 工程计算，Delta V > 0
```

这个公式表示价格可视为成交金额关于成交量的边际变化率；在真实离散成交数据中，正式计算应使用窗口成交金额与窗口成交量形成的加权平均成交价格。不能把阶梯函数上的解析导数直接当作可观测字段。

连续性假设说明：

- 在严格数学上，`P_wv = dW/dV` 依赖于 `W(V)` 的局部连续或可微近似
- 在真实市场中，`W`、`V` 是离散累积量，工程实现使用 `Delta W / Delta V`
- 因此 `P_wv` 是窗口边际价格或窗口成交量加权平均价格，不是阶梯函数在每一点的解析导数

它可以与物理速度公式建立形式同构：

```text
物理速度：v = dx / dt
边际价格：P_wv = dW / dV
```

对应关系为：

| 物理空间 | 金融空间 | 含义 |
| --- | --- | --- |
| `x` | `W` | 累计位移 / 累计成交金额 |
| `t` | `V` | 时间 / 累计成交量 |
| `v = dx / dt` | `P_wv = dW / dV` | 速度 / 边际成交价格 |
| `a = dv / dt` | `dP_wv / dV` | 加速度 / 价格对成交量的敏感度 |

这一层的含义是：`P_wv_window` 可作为 `W-V` 表示中的速度类比指标；它的工程定义是窗口 VWAP，而不是阶梯累计函数的严格导数。

### 3.2 重要边界

该同构只在成交金额-成交量空间中成立。它不能直接推出：

```text
价格状态动力学中的 P_state 就是速度
```

因为后续价格预测模型关心的是价格路径本身，即 `P_state(q)` 如何随交易时间推进而变化。在这个层面，`P_state(q)` 是状态函数，而不是速度函数。

## 4. 成交量作为交易时间

成交量具备作为市场内部时间轴的合理性：

1. 成交量单向累积
2. 成交量反映真实撮合推进
3. 高成交期表示市场内部时间推进更快
4. 低成交期表示市场内部时间推进更慢

因此，系统允许将累计成交量、成交量分位进度或成交量窗口序号记为**成交量进度指标** `q`。`q` 是建模坐标或重采样索引，不是独立、外生且均匀流逝的物理时间。

推荐的 `q` 构造方法如下：

1. **等成交量窗口**：每个窗口累计相同成交量，适合日内冲击比较
2. **成交量分位进度**：将当日累计成交量映射到 `[0,1]`，适合跨窗口归一化
3. **窗口序号近似**：在已按等成交量或等分位切窗后，用窗口序号近似 `q`

需要明确：

- 不同股票的 `q` 尺度一般不可直接比较
- `q` 的可比性依赖于相同切窗规则、相同最小成交量阈值和相近流动性分组
- 若使用固定自然时间窗口，则 `q` 只能作为诊断标签，不能自动等同于等步长交易时间

建议工程输出显式增加：

```text
q_type = volume_equal / volume_quantile / window_index
```

以避免不同 `q` 构造口径下的 `Delta_q P_state`、`phi`、`theta` 和 `beta_*` 被直接横向比较。

需要区分两种口径：

- 若 `q` 是等步长窗口序号或等成交量分位进度，则相邻窗口的 `Delta q` 可视为 1
- 若 `q` 是真实累计成交量，则一阶变化应除以 `Delta q`，不能直接把价格差称为单位交易时间速度

在该口径下：

```text
P_state = P_state(q)
Delta_q P_state,t = P_state(q_t) - P_state(q_{t-1})                      # 等步长 q 口径
v_q,t = (P_state(q_t) - P_state(q_{t-1})) / (q_t - q_{t-1})              # 非等步长 q 口径
Delta_q^2 P_state,t = Delta_q P_state,t - Delta_q P_state,t-1            # 等步长 q 口径
```

对应物理映射为：

| 物理概念 | 金融变量 | 含义 |
| --- | --- | --- |
| 位置 | `P_state(q)` | 交易时间轴上的价格状态 |
| 速度 | `Delta_q P_state` | 等步长交易时间口径下的价格变化 |
| 加速度 | `Delta_q^2 P_state` | 等步长交易时间口径下价格变化速度的变化 |
| 外力 | `U_ch / U_q / U_retail` | 三类行为代理流 |
| 惯性 | `phi` | 前一交易时间速度的留存 |
| 变化率修正 | `theta` | 前一交易时间速度变化的助推或抑制修正 |

`U_*` 在概念层表示有方向的行为代理流，但在工程层可以有不同量纲口径。只有 `U_*_mv_ratio` 口径可直接生成正式 `beta_norm` 和 `m_eff`；金额流需要先做统一单位标准化，规则分数只能用于状态估计和方向诊断。

不同 `q_type` 会改变速度、惯性和变化率参数的统计含义：等成交量窗口更强调单位成交推进下的响应，固定自然时间窗口更容易混入成交速度差异。因此 `phi`、`theta` 不得跨 `q_type` 直接比较，至少应在诊断中按 `q_type` 分组报告。

跨股票比较时，必须先完成同一 `q_type` 下的流动性分组和分位标准化。推荐流程为：

1. 在同一交易日或滚动历史期内按成交额、换手率和平均窗口成交量进行流动性分组
2. 仅在同组内比较 `phi`、`theta`、`m_eff_slow` 等指标
3. 若需要全市场排名，使用组内分位数或稳健 z-score，而不是直接比较原始值

流动性分组建议使用训练前固定的量化口径，例如：

```text
liquidity_score = rank_pct(turnover_amount_20d)
                + rank_pct(turnover_rate_20d)
                + rank_pct(avg_window_volume_20d)
```

按 `liquidity_score` 分为 `low / mid / high` 或 5 个分位组；只有当两只股票处于同一组，或相邻分位组且 `abs(liquidity_score_i - liquidity_score_j) <= 0.2` 时，才允许做弱可比诊断。若分组不匹配，应降级为组内分位展示、同组样本扩展后再比较，或仅输出 `cross_symbol_comparable = false` 的单股诊断，不得给出全市场原始值排名。

若未完成上述对齐，跨股比较不属于本文正式支持范围。

### 4.1 离散差分说明

本文中的导数在工程实现中默认用离散差分近似：

```text
dW/dV      -> Delta W / Delta V
dP/dV      -> Delta P / Delta V
dP/dt      -> Delta P / Delta t
d^2P/dt^2  -> Delta^2 P / Delta t^2
```

默认使用前向或后向一阶差分；若样本较稳定，可使用中心差分作为离线诊断。高阶差分会显著放大噪声，因此只推荐用于复盘、局部解释或窗口级诊断，不应直接作为高权重实时特征。

特别地，连续形式的 `d^2P/dt^2` 及其链式展开只用于离线解释和误差诊断，不直接进入比赛实时特征或提交接口。工程默认保留一阶差分；若需要使用二阶差分，必须标记为离线诊断字段，并经过噪声、窗口粒度和零成交边界检查。

## 5. 双域映射关系

成交量域与自然时间域中的价格变化不必相同。

设自然时间为 `t`，成交量为 `V(t)`，价格路径记为 `P(V(t))`，则可写出连续变量下的链式法则形式：

```text
dP / dt = (dP / dV) * (dV / dt)
```

离散实现先固定同一时间区间的定义：

```text
Delta_t P = P_t - P_t-1
Delta_t V = V_t - V_t-1
Delta_t = t - t-1
```

在 `Delta_t V > 0`、窗口内价格路径 `P(V)` 可以做局部线性近似、且没有跨越明显流动性断层时，成交量域斜率定义为：

```text
kappa_t = Delta_t P / Delta_t V
```

因此更稳妥的离散关系是：

```text
Delta_t P / Delta_t ≈ kappa_t * (Delta_t V / Delta_t)
```

这里的 `kappa_t` 只是同一区间内的一阶局部近似，不是对离散阶梯成交量函数的严格导数；
`Delta_t V = 0` 时它无定义，`Delta_t V` 很小但非零时也可能数值不稳定。零成交/薄成交降级策略属于工程数据处理规则，不是由链式法则推出的理论结论。

离散链式近似隐含局部线性条件：在同一自然时间窗口内，价格路径 `P(V)` 可由一阶斜率 `kappa_t` 代表，且窗口内成交量推进没有跨越明显流动性断层。若 `P(V)` 在窗口内二阶变化有界，前向差分的一阶近似误差可粗略记为 `O((Delta_t V)^2)` 量级；真实盘口为阶梯过程时不承诺解析误差上界，只通过历史分组误差校验给出经验有效性。未完成分组误差评估时，该映射只能用于离线解释，不能直接进入跨股排名、正式标签或高权重实时特征。工程上建议仅在以下条件同时满足时使用 `kappa_t`：

```text
Delta_t V >= min_volume_threshold
abs(Delta_t V) >= volume_epsilon
domain_mapping_error_std <= config.domain_mapping_error_limit
price_basis_warning == false
```

若 `Delta_t V` 很小但非零，也应视为数值不稳定窗口，优先标记 `thin_trade_window`，而不是机械放大 `Delta_t P / Delta_t V`。

其中：

- `dP / dV`：成交量域价格敏感度
- `dV / dt`：成交速度或订单流速率
- `dP / dt`：自然时间域价格变化速度

离散窗口中可写为：

```text
Delta_t P / Delta t ≈ (Delta_t P / Delta_t V) * (Delta_t V / Delta t)
kappa_t = Delta_t P / Delta_t V
```

其中 `Delta_t V > 0` 时才允许计算 `kappa_t`；当 `Delta_t V = 0` 时，右侧成交量域敏感度无定义，只能走 `no_trade_window` 降级流程。这是联合随机过程上的局部启发式近似，不是离散数据上的严格恒等式。

离散映射没有跨所有窗口通用的解析误差上界；工程上应按 `q_type`、流动性组和窗口粒度估计经验误差：

```text
domain_mapping_error = Delta_t P / Delta t
                     - (Delta_t P / Delta_t V) * (Delta_t V / Delta t)
```

其中 `Delta_t P / Delta_t V` 仅在同一局部区间且 `Delta_t V > 0` 时定义。
为统一记号，工程字段建议直接使用 `kappa_t`；不再把 `Delta_V P` 当作
一个独立于时间区间的观测字段。

并记录 `domain_mapping_error_mean`、`domain_mapping_error_std` 和 `domain_mapping_valid_flag`。未完成分组误差评估时，双域映射只能用于离线解释，不能作为跨股排名特征。

边界情况说明：

- 当 `Delta V = 0` 时，`kappa_t` 无定义，不能直接计算成交量域敏感度
- 对零成交窗口，推荐优先标记为 `no_trade_window`
- 若必须连续输出，可选择以下降级策略之一：跳过该窗口、沿用上一有效窗口的 `kappa`、或只保留自然时间诊断而不做成交量域解释
- 为避免回测与实盘口径漂移，降级优先级应在运行前固定；推荐默认顺序为 `skip > carry_forward > mark_only`
- `carry_forward` 只能沿用最近一个已确认的历史有效值，不得读取当前窗口之后的数据；比赛提交时应根据评分规则预先选择固定方案。若评分要求每个窗口都有记录，优先采用 `mark_only` 或仅使用历史值的 `carry_forward`，并保留 `no_trade_window` 标记。

比赛实现应在配置中显式声明：

```text
zero_trade_policy = skip / carry_forward / mark_only
submission_requires_complete_windows = true / false
carry_forward_lookback_limit = 3
```

若评分要求完整窗口覆盖，则不得使用 `skip`；此时 `carry_forward` 必须使用窗口开始前最近一个完整有效值，默认最多追溯上一交易日最后 `3` 个有效窗口，不得无标记跨日沿用更早数据。`mark_only` 则只输出标记和降级预测，不得伪造成交量域敏感度。

这说明：

```text
固定自然时间窗口内的价格变化 = 成交量域价格敏感度 × 该时间窗口内成交量
自然时间价格变化率 = 成交量域价格敏感度 × 成交量流速
```

因此，固定自然时间窗口与固定成交量窗口会得到不同的 `Delta P`。若工程上使用 5 分钟固定窗口，需要记录窗口成交量或成交量分位进度，以便区分：

- 自然时间变化：`Delta_t P`
- 交易时间变化：`Delta_q P`
- 成交量敏感度：`Delta_V P / Delta V`

### 5.1 高阶互译与资金吞吐率

`统一.md` 中的双域几何可以作为本系统的补充解释，但需要限定在可观测价格路径 `P(V(t))` 上。

一阶互译为：

```text
dP / dt = (dP / dV) * (dV / dt)
```

二阶互译为：

```text
d^2P / dt^2 =
    (d^2P / dV^2) * (dV / dt)^2
  + (dP / dV) * (d^2V / dt^2)
```

其中 `dP/dV = d^2W/dV^2` 是成交量域价格敏感度，`d^2P/dV^2 = d^3W/dV^3` 是更高阶的敏感度变化率。工程上若高阶项噪声过大，可只保留一阶互译和窗口级诊断。

含义是：自然时间上的价格加速度由两部分组成：

- 成交量域曲率 `d^2P / dV^2` 与成交速度平方共同放大
- 成交量域敏感度 `dP / dV` 与成交加速度共同作用

同时，资金吞吐率可作为双域桥接指标：

```text
dW / dt = (dW / dV) * (dV / dt) = P * dV/dt
```

该指标表示单位自然时间内的成交金额流速，可用于衡量市场热度、流动性推进速度和固定自然时间窗口内的价格变化背景。

离散近似误差主要来自：

1. `V(t)` 的阶梯性带来的差分偏差
2. 高频噪声在 `Delta P / Delta V` 中被放大
3. 零成交或极小成交窗口导致的数值不稳定

因此，双域映射更适合作为解释和诊断工具，而不是在所有窗口上机械套用的严格恒等关系。

## 6. 理论主方程

在价格状态-交易时间空间中，理论主方程采用价格状态与价格速度的二层形式：

```text
v_q,t = (P_state(q_t) - P_state(q_t-1)) / Delta q_t
P_state,t = P_state,t-1 + v_q,t * Delta q_t
```

这两行分别表示：

- `v_q,t` 是由价格状态观测差分得到的观测量定义；
- `P_state,t` 是根据速度观测进行的状态重构。

它们互为定义与恢复关系，不应被表述为包含不可观测价格状态演化的完整状态方程。本文真正需要估计的是参数状态向量 `psi_t`，其观测方程为 `y_t = x_t' * psi_t + epsilon_t`；具体先验、后验和更新规则见 PID 实现文档。

等步长交易时间且 `Delta q_t = 1` 时，工程字段 `Delta_q P_t` 可作为 `v_q,t` 的数值简写。一般步长下不能省略 `Delta q_t`。

其中价格速度 `v_q,t` 由下式刻画：

```text
v_q,t =
    beta_ch * U_ch,t-1
  + beta_q * U_q,t-1
  + beta_retail * U_retail,t-1
  + phi * v_q,t-1
  + theta * (v_q,t-1 - v_q,t-2)
  + epsilon_t
```

令：

```text
v_t = v_q,t
a_q,t = (v_t - v_t-1) / Delta q_t
D_driver_t = v_t - v_t-1
```

`a_q,t` 是按交易时间步长归一化的加速度；`D_driver_t` 是主状态方程使用的原始速度变化驱动项。只有在等步长 `Delta q_t = 1` 时，二者数值相同。后续特征方程针对 `v_t` 的递推关系推导，不把它直接解释为 `P_state` 的二阶差分。

则可写为：

```text
v_t =
    beta_ch * U_ch
  + beta_q * U_q
  + beta_retail * U_retail
  + phi * v_t-1
  + theta * D_driver_t-1
  + epsilon_t
```

解释：

- `beta_* U_*` 是行为代理流对价格速度的加载
- `phi * v_t-1` 是趋势惯性
- `theta * D_driver_t-1` 是速度变化的助推或抑制诊断；最终方向需结合 `theta * D_driver_t-1` 的符号判断
- `epsilon_t` 是未解释扰动

当 `phi`、`theta` 在一个检验窗口内近似固定，且使用等步长 `q` 时，去除外力和噪声后有：

```text
v_t = phi * v_t-1 + theta * (v_t-1 - v_t-2)
v_t - (phi + theta) * v_t-1 + theta * v_t-2 = 0
```

对应的齐次特征方程为：

```text
r^2 - (phi + theta) * r + theta = 0
```

令标准二阶多项式为 `r^2 + a1*r + a0 = 0`，则本模型有：

```text
a1 = -(phi + theta)
a0 = theta
```

标准二阶 Jury 条件可写成：

```text
abs(a0) < 1
1 + a1 + a0 > 0
1 - a1 + a0 > 0
1 - a0 > 0
```

代入 `a1`、`a0` 后，相关条件化简为 `1 - phi > 0` 和 `1 + phi + 2 * theta > 0`；其中 `1 - a0 > 0` 已由 `abs(theta) < 1` 包含。因此本模型的二阶 Jury 条件可化简为：

一些资料会把二阶条件写成额外的
`abs(a1) < 1 + a0`。在本模型中，`abs(theta) < 1` 先保证
`1 + a0 > 0`，而 `1 + a1 + a0 > 0` 与
`1 - a1 + a0 > 0` 合并后，恰好等价于
`abs(a1) < 1 + a0`。因此它不是当前特征方程下遗漏的第四个独立约束；为避免争议，工程实现仍应直接计算特征根并记录最大模。

```text
abs(theta) < 1
1 - phi > 0
1 + phi + 2 * theta > 0
```

因此，`phi` 与 `theta` 的边界必须联合判断，不能只分别限制在单变量区间内。

若 `phi_t`、`theta_t` 随时间变化，上述 Jury 条件只能作为“冻结窗口”或窗口平均参数的局部稳定性诊断，不能直接证明整个时变系统全局稳定。工程上应同时记录滚动窗口内的特征根最大模、参数越界次数和冲击响应衰减情况。

一阶公式可以作为工程降阶近似，但不能替代理论主方程。

## 7. 候选稳定量

### 7.1 等效冲击柔度

`beta` 可以解释为行为代理流对价格变化的加载强度，也可称为响应系数。

但系统中存在多路 `beta`：

- `beta_ch`
- `beta_q`
- `beta_retail`
- `beta_mix`

因此不能简单把单个 `beta` 当作股票固有属性。

若工程上使用 `beta_mix`，它表示 `Q_rule + R_seed` 的净效应响应系数，不保留量化与散户方向分解。当 `Q_rule` 与 `R_seed` 符号相反时，`beta_mix` 只能解释混合净冲击，不能拆成两个结构化身份贡献。

### 7.2 等效质量

可定义候选等效质量。为保持与既有接口兼容，继续使用 `m_eff` 字段；在金融语境中，它更准确地称为**响应阻抗指数**或**响应弹性倒数**，不表示具有物理质量量纲的真实质量：

```text
m_eff = 1 / |beta_norm|
```

其中正式 `beta_norm` 只使用市值占比流口径：

```text
beta_norm_* = (capital_* / P_state) / U_*_mv_ratio
```

这里要求 `capital_*` 必须由同一市值占比流口径重新估计：

```text
capital_* = beta_*_mv_ratio * U_*_mv_ratio
beta_norm_* = beta_*_mv_ratio / P_state
```

不得用规则分数或金额口径生成的 `capital_*` 再除以
`U_*_mv_ratio`。前者是同口径定义的展开式，后者会混合输入口径，
导致量纲和跨股比较含义不稳定。

该指标表示：单位市值占比行为流推动单位价格收益变化的响应强度倒数。它仍受窗口和 `q` 步长影响，只能在固定配置、固定 `q_type` 和固定输入口径下进行相对比较。

若使用 `U_*_score` 或 `U_*_amount`，必须输出对应的 `beta_norm_unit`；在完成统一单位标准化前，这两种口径不得生成正式 `m_eff`。

若暂时没有结构化贡献，也可用总价格变化构造粗略口径：

```text
beta_norm_rough = (Delta_q P / P) / (U / market_value)
```

但该口径会混入惯性、阻尼与残差，只能作为诊断近似，不应直接等同于估计后的 `beta_*`。

解释：

- `beta_norm` 高：价格对行为代理流敏感，等效质量低
- `beta_norm` 低：价格对行为代理流不敏感，等效质量高

### 7.3 使用边界

`m_eff` 只能称为候选稳定量，不能直接称为已验证不变量。使用前必须满足：

1. `beta_norm` 不接近 0
2. `beta_norm` 符号或方向经过稳定性处理
3. 行为代理流 `U_*` 可辨识
4. 残差诊断通过
5. 跨日波动幅度、变异系数和分位稳定性通过检验

工程默认可引用 PID 文档第 10 节的阈值：

- `param_cv_7d < 1.0`
- `param_sign_flip_count_7d <= 2`
- `m_eff_iqr / abs(m_eff_median) < 0.5`
- `abs(beta_norm_mean)` 高于分组 10% 分位或 `1e-6`

样本不足时不得硬判稳定，应输出 `insufficient_history_flag = true` 并降级为状态估计量。

推荐输出：

- `beta_norm_mean`
- `beta_norm_std`
- `beta_norm_cv`
- `m_eff_median`
- `m_eff_iqr`
- `m_eff_slow`
- `m_eff_fast_std`
- `m_eff_stable_flag`

### 7.4 慢变质量与快变扰动

`统一.md` 中关于质量分解的思路可以作为稳定性诊断的补充，但不能直接把单次估计结果称为不变量。

可采用如下候选分解：

```text
m_eff,t = m_slow,t + m_fast,t
```

其中：

- `m_slow,t`：跨日或更长窗口上的慢变分量，可能反映流通盘、筹码结构、机构持仓结构等较稳定因素
- `m_fast,t`：日内或短窗口扰动分量，可能来自短线冲击、挂撤单变化、流动性突变和噪声

工程上默认优先对 `m_eff` 序列做 EWMA 提取慢变分量：

```text
alpha = 2 / (N + 1)
m_slow,t = alpha * m_eff,t-1 + (1 - alpha) * m_slow,t-1
```

其中 `N` 为训练前固定的历史窗口长度。次选为固定窗口滚动中位数，卡尔曼趋势项需要单独定义状态方程；RTS 平滑因使用未来观测，只适合离线复盘和稳定性画像。默认优先级 `ewma_realtime > rolling_median_realtime > kalman_filter_realtime` 是工程稳健性排序，不是理论最优性排序：EWMA 参数少且易审计，滚动中位数更抗异常值，卡尔曼滤波表达力更强但依赖 `Q/R`、初始化和状态方程，需通过离线稳定性验证后再作为默认。

实时预测只能使用滤波或递推估计；所有方法均应记录 `m_slow_method`。

若使用固定历史窗口提取 `m_slow`，窗口长度应在训练前写入配置并保持不变，例如固定为 `20` 日或 `60` 日；不得在比赛提交阶段按“稳定性最好”动态回选，以免形成循环论证或隐性前瞻性偏差。

只有当 `m_slow,t` 在跨日稳定性检验中通过阈值要求时，才可称为候选个股质地参数；在此之前只能称为状态估计量或候选稳定量。

推荐区分两种来源：

- `m_slow_realtime`：只使用历史窗口或历史交易日递推得到，可用于实时预测和比赛提交
- `m_slow_offline`：使用全样本平滑或未来窗口信息得到，只能用于复盘、画像和方法评估

`kalman_filter_realtime` 是递推滤波，可用于实时；`kalman_smoother_offline`、RTS 和 HP 平滑使用未来观测，只能用于离线。实现和字段命名不得把 smoother 统称为 filter。

HP 平滑当前只作为离线诊断备选，不作为比赛默认方法；若启用，必须固定平滑参数、样本区间和版本，并标记 `m_slow_method = hp_offline`，不得将其结果用于实时预测。

若历史窗口不足，应输出历史不足标记，而不是强行生成稳定量。

## 8. 不可采用的解释

以下解释不作为本系统正式口径：

1. `P` 在价格状态动力学中等于速度
2. `Delta_q P` 在价格状态动力学中等于加速度
3. `C_I / C_D` 可以反解为某一类资金身份
4. 单日 `beta` 可以直接解释为股票长期固定基因
5. `m_eff = 1 / beta` 可以在未归一化、未稳定性检验时直接称为不变量
6. `F = ma` 与 PID 的类比可以作为解释框架，但不能在未验证高阶项、残差和稳定性条件时称为严格证明

### 9.1 量纲声明

为避免物理术语误导，本文中的“速度、加速度、质量、阻尼”均为类比术语，工程实现应优先关注其统计和建模角色。

推荐量纲表述如下：

| 量 | 推荐工程单位 |
| --- | --- |
| `W` | 元 |
| `V` | 股 |
| `P_wv` | 元/股 |
| `Delta_q P_state` | 元/股/窗口 或 价格点/窗口 |
| `U_*_score` | 无量纲 |
| `U_*_amount` | 元/窗口 |
| `U_*_mv_ratio` | 无量纲 |
| `beta_norm_*` | `[每交易时间步的价格收益率 / 市值占比流]`，固定 `q` 步长和口径下可作归一化响应系数 |
| `m_eff_*` | `[市值占比流 / 价格收益率]` 的响应阻抗诊断指数，不解释为严格物理质量 |
| `beta_norm_unit` | `score_response / amount_response / mv_ratio_response` |
| `m_slow_method` | `ewma_realtime / rolling_median_realtime / kalman_filter_realtime / kalman_smoother_offline / rts_offline / hp_offline` |

本系统采用如下统一定义：

```text
W(V) -> P_wv_window = Delta W / Delta V
      -> dP/dV  # 仅作连续近似或离线诊断记号
```

用于解释成交金额-成交量空间中的边际价格形成。

同时采用：

```text
P(q) -> Delta_q P -> Delta_q^2 P
```

用于解释价格状态在交易时间中的动力学变化。

两层之间通过：

```text
dP/dt = (dP/dV) * (dV/dt)
```

建立映射。

并用：

```text
dW/dt = P * dV/dt
beta_norm_* -> m_eff_* -> m_slow / m_fast
```

将成交活跃度、外力响应和候选稳定量连接到同一工程诊断链条。

这一定义保留了“股价变化可借助物理速度直觉解释”的启发式价值，但不宣称存在严格数学同构；同时避免把不同空间中的速度、加速度混为一层，并保留比赛工程实现所需的可估计、可闭合和可校验边界。
