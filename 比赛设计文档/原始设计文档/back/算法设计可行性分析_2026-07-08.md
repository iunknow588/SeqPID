# 算法设计可行性分析报告

**分析日期**: 2026-07-08  
**分析对象**: 基于状态空间辨识的三类行为代理因子贡献度拆解算法（v1.1）  
**分析结论**: **理论可行，但存在3个关键工程风险点，必须逐一闭环后方可进入编码阶段**

---

## 一、总体评估结论

| 维度 | 评估 | 说明 |
|------|------|------|
| 理论自洽性 | ✅ 可行 | 控制论PID分解框架逻辑自洽，工程展开式合理 |
| 数据可得性 | ⚠️ 待验证 | 依赖Level2逐笔数据schema探针结果 |
| 模型可辨识性 | ⚠️ 需预检 | 5维状态向量在48窗口上的可辨识性需模拟验证 |
| 计算复杂度 | ✅ 可控 | 单股KF/RTS计算量极小，全市场批处理可接受 |
| 业务解释性 | ✅ 良好 | 分层解耦设计，展示层与建模层分离，解释路径清晰 |

**核心风险点（按优先级排序）**:

1. **P0 - theta（阻尼系数）可辨识性**: 5维状态向量中theta无直接观测约束，仅靠Q矩阵约束可能不稳定
2. **P1 - Level2数据schema可用性**: 逐笔数据字段、格式、关联方式尚未验证
3. **P1 - lambda参数敏感性**: D项拆分权重对量化/散户判定结果的影响程度未知

## 二、逐层可行性分析

### 2.1 第1层：行为代理因子分类器

**可行性**: ✅ 条件可行，依赖数据验证

**设计要点**:
- 分类规则：大额≥50万/小额<10万/多档穿越≥2档
- 窗口聚合：5分钟窗口，全日48个窗口
- 预处理：1%缩尾、缺失值补零、EWMA自适应标准化

**待确认问题**:

**问题1: Level2数据schema支持度**
- 逐笔成交是否有明确的买卖方向标识？（用于计算主动买入/卖出）
- 逐笔委托与逐笔成交能否通过订单号关联？（用于计算撤单率、订单生命周期）
- 十档盘口快照的时间粒度是多少？能否与成交时间对齐？

**问题2: 挂单时间字段`order_lifetime_ms`可得性**
- 文档明确1.1版本不将其作为硬依赖，但如有该字段，分类器精度显著提升
- 需要确认该字段在原始数据中的覆盖率
- 若缺失，是否影响"量化代理因子"的识别？（量化策略通常挂撤单频繁，生命周期短）

**问题3: 阈值初值调整**
- 50万/10万/2档穿越是经验值，不同市值股票的"大额"标准差异很大
- 建议先按绝对值实现1.1版本，后续考虑用个股历史成交额分位数动态调整

### 2.2 第2层：结构系数识别（核心层）

**可行性**: ⚠️ 理论可行，但存在可辨识性风险

**设计要点**:
- 状态向量：5维 [φ, β_ch, β_q, β_retail, θ]
- 观测方程：ARX结构，不含滞后残差（避免ARMAX循环依赖）
- 状态转移：随机游走
- 滤波算法：KF + RTS平滑

**关键风险: theta的可辨识性**

**问题描述**:
- theta（阻尼系数）在主观测方程中的系数为0（文档4.2节明确：H_t = [x_main_t[0], x_main_t[1], x_main_t[2], x_main_t[3], 0.0]）
- theta没有直接观测约束，只能通过过程噪声协方差Q的约束维持稳定
- 本质上是"弱观测"状态

**风险分析**:
```
风险等级: 高

原因:
1. 5维状态向量在仅4维有效观测输入的情况下，theta估计可能对初值敏感
2. 仅靠Q_55约束（文档建议Q_55 = Q_11 × 0.01~0.1）可能不足以防止长期漂移
3. 若theta不稳定，直接影响第4层D项拆分的可靠性
4. theta的漂移会传导到量化/散户的判定结果

缓解措施（文档已提出）:
- 方案1: 约束Q_55（默认）
- 方案2: 弱观测约束
- 方案3: 残差回归二次校正
- 回退: 4维模型（不含theta）
```

**建议**:
1. **编码前必须先做模拟数据验证**：人工生成已知theta序列的数据，检验KF/RTS能否恢复
2. **必须对比4维vs5维模型**：若5维模型残差没有比4维显著改善，则没有理由承担theta风险
3. **检查状态协方差矩阵条件数**：若条件数过大（>1e6），说明参数间存在严重共线性，应降维

**问题4: theta稳定方案选择**
- 文档提出三种方案，1.1版本默认采用"约束Q_55"
- 建议编码前同时实现方案1和方案2的对比实验
- 若方案1在模拟数据上表现不佳，是否直接回退到4维模型，而非尝试方案2？

### 2.3 第3层：基础归因层

**可行性**: ✅ 可行

**设计合理性**:
- 基础贡献公式：Δ = β·U + φ·ΔP_prev·w
- 第一项β·U承接P类作用（冲击推进）
- 第二项φ·ΔP·w承接I类作用（惯性延续）
- 权重w基于流量占比归一化，结构清晰

**问题5: 惯性项实现**
- 文档说明"不显式叠加ε_{t-1}"，这是正确的
- ΔP_{t-1}已包含历史所有信息，再叠加ε_{t-1}会造成重复解释

### 2.4 第4层：D项拆分层

**可行性**: ⚠️ 条件可行，依赖lambda参数验证

**设计合理性**:
- D项拆分意图合理：量化偏阻尼、散户偏恐慌反转、游资偏冲击
- lambda参数：lambda_ch=0.05, lambda_q=1.00, lambda_retail=0.40

**关键风险: lambda参数敏感性**

**问题描述**:
- lambda参数直接决定D项在量化与散户之间的分配比例
- lambda是行为代理层面的"先验"，不是从数据中估计的
- 若标定不当，可能导致量化/散户判定系统性偏差

**风险分析**:
```
风险等级: 中

原因:
1. lambda是行为代理层面的先验，非数据估计
2. 若标定不当，可能导致量化/散户判定系统性偏差
3. 文档要求验证"lambda浮动20%时日终主导判定变化<10%"，此标准是否合理？

缓解措施:
- 按文档要求执行敏感性扫描
- 建立lambda与人工抽检标签的吻合度评估
- 若敏感性过高，考虑将lambda纳入状态估计
```

**问题6: lambda标定方式**
- 文档提到lambda标定数据来源：
  1. 历史样本的人工抽检标签
  2. 规则种子的高置信度样本作为代理真值
  3. 保守默认值（前两者不足时）
- 需要确认：目前是否有历史样本的人工标签？
- 若无，规则种子的置信度是否足够高？
- 是否考虑用EM算法或交叉验证自动标定？

### 2.5 第5层：展示层

**可行性**: ✅ 完全可行

- 残差分摊逻辑简单，闭合性可数值验证
- 展示层与建模层分离的设计原则正确

**问题7: 展示层与结构层偏离处理**
- 文档规定"若展示层与结构层结论明显偏离，优先排查噪声占比、窗口空值与D项拆分参数"
- 需要明确："明显偏离"的量化标准是什么？
- 排查后仍无法解释时，是否降低置信度或触发规则兜底？

### 2.6 第6层：诊断层

**可行性**: ✅ 可行，但阈值需标定

- Ljung-Box检验和稳态残差监控都是标准方法，实现简单

**问题8: theta诊断阈值标定**
- 文档给出默认阈值：theta<-0.3（强阻尼）、-0.3≤theta≤0.1（正常）、theta>0.1（正反馈加速）
- 这些阈值基于什么分布确定？
- 建议先用历史60日数据计算theta分位数分布，再按分位数重新标定（如P10/P50/P90）

## 五、实施建议

### 推荐实施顺序

```
阶段A: 数据验证（1-2天）
  A1. Level2 schema探针
  A2. 单股单日代理流生成验证
  A3. 阈值初值回放
  → 产出: schema_probe_report.md

阶段B: 模型可辨识性验证（2-3天）
  B1. 模拟数据参数恢复实验（重点验证theta）
  B2. 4维vs5维模型对照
  B3. 协方差矩阵条件数检查
  → 产出: model_identifiability_report.md
  【gates】若theta不可辨识，回退到4维模型

阶段C: 核心链路实现（3-5天）
  C1. ARX + KF/RTS主状态模型
  C2. 基础归因层 + 结构层闭合校验
  C3. 展示层 + 展示层闭合校验

阶段D: D项拆分验证（2-3天）
  D1. D项拆分层实现
  D2. lambda参数敏感性扫描
  → 产出: lambda_sensitivity_report.md
  【gates】若敏感性过高，调整lambda默认值

阶段E: 集成与验收（2-3天）
  E1. 主导判定与比赛字段映射
  E2. 午间热启动验证
  E3. 全链路压测
  → 产出: integration_acceptance_report.md
```

### 关键原则

1. **阶段A和阶段B是编码的前置条件，不可跳过**
2. **若theta不可辨识，果断回退到4维模型，不要强行保留5维**
3. **lambda敏感性扫描不可省略，它决定了量化/散户拆分的可信度**
4. **整个算法定位为"解释型辅助引擎"，不要试图让它独立承担Task 1/2的全部判定责任**

### 回退机制

| 风险场景 | 回退策略 | 交付模式 |
|---------|---------|---------|
| theta不可辨识 | 回退到4维模型 | 基础归因口径，D项仅作诊断 |
| lambda敏感性过高 | 缩小lambda扰动范围 | 保守默认值，降低D项拆分权重 |
| Level2 schema缺失关键字段 | 分类器降级为阈值简版 | 优先保证代理流连续性 |
| 展示层与结构层严重偏离 | 触发规则兜底 | 低置信度输出 |

## 六、最终结论

| 维度 | 结论 |
|------|------|
| **整体可行性** | ✅ 可行，但需完成3个关键风险闭环 |
| **理论自洽性** | ✅ 控制论框架逻辑自洽，工程展开合理 |
| **工程可实现性** | ⚠️ 依赖schema探针和可辨识性验证 |
| **业务价值** | ✅ 解释型特征对Task 1/2有明确辅助价值 |
| **建议下一步** | 先执行阶段A（数据验证）和阶段B（可辨识性验证），通过后再进入编码 |

**核心建议**:
1. 不要跳过`model_identifiability_report.md`，这是整个算法可信度的基石
2. 若theta在模拟数据上无法稳定恢复，果断回退到4维模型，不要强行保留5维
3. lambda敏感性扫描不要省略，它决定了量化/散户拆分的可信度
4. 整个算法定位为"解释型辅助引擎"，应与统计特征引擎、规则层和市场级聚合模块共同服务Task 1/2
