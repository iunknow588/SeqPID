# 评审意见：2026OPC大赛自动化交易比赛设计文档

**评审时间**：2026-07-13  
**评审对象**：
- `金融领域的物理学概念模型.md`
- `PID算法与实现.md`

**评审结论**：两份文档整体架构严谨，概念定义清晰，工程边界明确。但在**逻辑自洽性、符号一致性、时序因果、工程可实现性**四个维度存在若干需要修正的缺陷。

---

## 一、逻辑缺陷清单

### 1. 【严重】时序因果倒置：主状态方程中 `v_q,t` 的自引用

**位置**：`PID算法与实现.md` 第4节主状态方程

**原文**：
```
v_q,t = beta_ch,t * U_ch,t-1 + ... + phi_t * v_q,t-1 + theta_t * (v_q,t-1 - v_q,t-2) + epsilon_t
```

**问题**：该方程将 `v_q,t` 表达为自身历史项的函数，但右侧同时包含 `v_q,t-1` 和 `v_q,t-2`。这在数学上是**递推定义**，在工程实现中需要明确：
- `v_q,t` 是**观测值**（已知的 `Delta_q P_t`）还是**预测值**（`v_q,t+1|t`）？
- 若用于参数估计，应明确是**观测方程** `y_t = x_t · psi_t + epsilon_t`；
- 若用于预测，应明确是**预测方程** `v_q,t+1|t = x_t+1|t · psi_t|t-1`。

**当前文档的模糊之处**：在4.1节中同时出现两种口径，但没有明确区分"估计阶段"和"预测阶段"的方程形式。这会导致实现时混淆：
- 估计时：`y_t` 已知，`psi_t` 待估；
- 预测时：`psi_t|t-1` 已知，`y_t+1` 待预测。

**建议修正**：
将主状态方程拆分为两个明确区分的方程：
```
# 观测方程（用于参数估计）
Delta_q P_t = x_t · psi_t + epsilon_t

# 预测方程（用于实时预测）
v_q,t+1|t = x_t+1|t · psi_t|t-1
```
并在文档中明确标注：前者用于离线训练/参数更新，后者用于在线预测/比赛提交。

### 2. 【严重】`D_driver_t` 的定义与 `theta` 的符号解释存在循环依赖

**位置**：`PID算法与实现.md` 第4节、第8.3节

**原文**：
```
D_driver_t = v_q,t-1 - v_q,t-2
C_D,t = theta_t * D_driver_t
```

**问题**：文档8.3节定义：
- `theta > 0`：偏助推（强化最近一次速度变化的方向）
- `theta < 0`：偏抑制（抵消最近一次速度变化）

但这里存在**符号解释的歧义**：
- 若 `D_driver_t = v_t-1 - v_t-2 > 0`（速度在增加），且 `theta > 0`，则 `C_D,t > 0`，即**正向助推**——这与8.3节定义一致。
- 但若 `D_driver_t < 0`（速度在减小），且 `theta > 0`，则 `C_D,t < 0`，即**负向助推**——这实际上是"加速减速"，与直觉上的"助推"矛盾。

**核心问题**："助推/抑制"的解释依赖于 `D_driver_t` 的符号，但文档没有明确说明这种**条件解释**。读者可能误以为 `theta > 0` 总是"助推价格上涨"。

**建议修正**：
在8.3节增加条件解释：
```
theta_sign_interpretation 应联合 D_driver_t 符号判断：
- 若 theta * D_driver_t > 0：强化当前速度变化趋势
- 若 theta * D_driver_t < 0：抵消当前速度变化趋势
- 单独 theta 的符号不直接等同于助推/抑制
```

或者，重新定义 `D_driver_t` 的口径，使其符号解释更直观。

### 3. 【中等】`phi` 与 `theta` 的稳定性条件在文档中存在两处表述不一致

**位置**：
- `金融领域的物理学概念模型.md` 第6节
- `PID算法与实现.md` 第6.0节

**原文对比**：

| 文档 | 稳定性条件 |
|------|-----------|
| 物理概念模型 | `abs(theta) < 1`, `1 - phi > 0`, `1 + phi + 2*theta > 0` |
| PID算法 | 相同，但增加 `abs(phi + theta) < 1` 作为"启发式快速筛查" |

**问题**：
1. `abs(phi + theta) < 1` 被标注为"既不是充分条件，也不是所有稳定组合的必要条件"，但文档没有给出**反例**说明为什么这个条件会误导。
2. 实际上，`phi = 0.5, theta = 0.9` 满足Jury条件但 `abs(phi+theta)=1.4>1`，文档已给出。但反之，若 `phi = -0.9, theta = 0.5`，`abs(phi+theta)=0.4<1`，但 `1 - phi = 1.9 > 0`，`1 + phi + 2*theta = 1.1 > 0`，Jury条件满足。这说明 `abs(phi+theta)<1` 可能**过度排除**稳定组合，而非"误导性包含"。

**建议修正**：
明确说明 `abs(phi+theta)<1` 的问题在于**过度保守**（可能将稳定组合误判为不稳定），而非"误导性包含"。或者，如果目的是快速筛查，应说明其**敏感性高但特异性低**的特点。

### 4. 【中等】`beta_norm` 的定义在两种文档中存在量纲不一致

**位置**：
- `金融领域的物理学概念模型.md` 第7.2节
- `PID算法与实现.md` 第6.1节

**原文对比**：

| 文档 | 定义 |
|------|------|
| 物理概念模型 | `beta_norm_* = (capital_* / P_state) / U_*_mv_ratio` |
| PID算法 | `beta_norm_* = (capital_* / P_state) / U_*_mv_ratio` |

**表面一致，但深层问题**：
- `capital_* = beta_* * U_*`（来自PID算法7.2节）
- 代入得：`beta_norm = (beta_* * U_* / P_state) / U_*_mv_ratio`

若 `U_*` 与 `U_*_mv_ratio` 是同一口径（都是市值占比流），则：
```
beta_norm = beta_* / P_state
```
这意味着 `beta_norm` 的量纲是 `1/价格`，而 `m_eff = 1/beta_norm` 的量纲是 `价格`。

但文档又说 `m_eff` 是"响应阻抗指数，不解释为严格物理质量"，这暗示量纲是**无量纲**的。这里存在**量纲解释的隐性矛盾**。

**建议修正**：
明确 `beta_norm` 的量纲，并统一说明：
```
beta_norm 的量纲为 [价格变化率 / 市值占比流]，在固定 q 步长下可作归一化响应系数。
m_eff = 1/beta_norm 的量纲为 [市值占比流 / 价格变化率]，作为响应阻抗的相对度量。
```

### 5. 【中等】`U_*` 的三种口径与 `beta_*` 的对应关系未完全闭合

**位置**：`PID算法与实现.md` 第3.3节、第6.1节

**原文**：
文档定义了三种 `U_*` 口径：`score`、`amount`、`mv_ratio`，并规定只有 `mv_ratio` 可用于正式 `beta_norm`。

**问题**：
若 `U_*` 口径不同，`beta_*` 的量纲和数值范围必然不同。但文档没有明确：
1. `beta_*` 是否随 `U_*` 口径变化而重新估计？
2. 若从 `score` 切换到 `mv_ratio`，`beta_*` 是否需要重新训练？
3. 不同口径下的 `beta_*` 是否可比？

**建议修正**：
明确声明：
```
beta_* 的量纲依赖于 U_* 的口径选择。不同口径下的 beta_* 不可直接比较。
若切换 U_* 口径，必须重新估计 beta_*，并记录 u_source_type 和 beta_norm_unit。
```

### 6. 【轻微】`q_type` 的可比性限制与工程实现的矛盾

**位置**：`金融领域的物理学概念模型.md` 第4节

**原文**：
```
不同股票的 q 尺度一般不可直接比较
q 的可比性依赖于相同切窗规则、相同最小成交量阈值和相近流动性分组
```

**问题**：
比赛场景通常需要**跨股票比较**（如选股、排名）。若 `q` 不可比，则基于 `q` 的 `phi`、`theta`、`m_eff` 等参数也不可直接横向比较。但文档又在7.3节建议输出 `m_eff_slow` 用于"个股画像"，这暗示跨股比较是需求之一。

**建议修正**：
明确区分两种使用场景：
1. **单股时序分析**：`q` 只需内部一致，不同 `q_type` 下的参数不可比。
2. **跨股比较**：必须先通过流动性分组、标准化或分位映射，使 `q` 尺度对齐。文档应增加跨股标准化方法，或明确声明跨股比较不在当前文档范围内。

### 7. 【轻微】`mode_switch` 的滞回规则与实时性的矛盾

**位置**：`PID算法与实现.md` 第5.3节

**原文**：
```
至少连续 k=5 个窗口通过可辨识性检验才允许从4维切换到5维
至少连续 k=3 个窗口失败才回退到4维
单个交易日内默认最多切换2次
```

**问题**：
日内高频场景下，5个窗口可能对应较长时间（如5分钟窗口×5=25分钟）。若市场结构在1-2个窗口内发生变化，滞回规则可能导致**模式切换滞后**，错过结构变化信号。

**建议修正**：
增加说明：
```
滞回规则的 k=5/k=3 是保守默认值，适用于结构变化较慢的市场。
若比赛场景要求更高灵敏度，可在训练前降低 k 值，但需记录 mode_switch_frequency 和
样本外误差，以评估切换噪声的影响。
```

### 8. 【轻微】`data_leakage_check` 的检查项与工程可实现性

**位置**：`PID算法与实现.md` 第10.1节

**原文**：
建议检查项包括：
1. 预测时是否只使用截至 t 或 t-1 的输入
2. 是否使用 psi_t|t-1 而非 psi_t|t
3. m_slow 是否标记为 realtime_filter
4. lookback_days 是否固定
5. RTS/全样本平滑是否仅用于离线

**问题**：
这些检查项大多是**流程规范**而非**自动化检查**。例如，"是否使用 psi_t|t-1" 需要代码审查，难以自动化断言。

**建议修正**：
将检查项分为两类：
1. **自动化检查**：如 `max_feature_timestamp <= prediction_cutoff_timestamp`、`m_slow_method not in {offline_methods}`
2. **人工审查**：如代码逻辑审查、训练流程审计

并明确：比赛提交时，自动化检查必须全部通过；人工审查作为辅助验证。

## 二、文档间一致性问题

### 1. `m_slow` 提取方法的命名不一致

**位置**：
- `金融领域的物理学概念模型.md` 第7.4节：使用 `kalman_filter_realtime`、`kalman_smoother_offline` 等
- `PID算法与实现.md` 第6.1节：使用 `kalman_filter_realtime`、`kalman_smoother_offline` 等

**问题**：命名一致，但物理概念模型文档中出现了 `hp_offline`（HP滤波），而PID算法文档中也出现了。需要确认：HP滤波是否被正式支持？若支持，需要补充HP滤波的参数设置和边界条件。

### 2. `beta_norm_floor` 的默认值来源

**位置**：`PID算法与实现.md` 第6.1节

**原文**：
```
beta_norm_floor = max(1e-6, p10(abs(beta_norm_*)))
```

**问题**：`p10` 表示10%分位，但这是**历史数据的统计量**。若实时预测时历史样本不足，如何计算 `p10`？文档未说明。

**建议修正**：
明确：
```
beta_norm_floor 的默认值 1e-6 是固定下限。p10 口径需要至少 N 个历史样本才启用；
样本不足时，固定使用 1e-6，并标记 insufficient_history_flag。
```

## 四、总体评审结论

**结论：文档质量较高，但需在以下方面修正后方可作为比赛正式设计文档：**

1. **必须修正**：
   - 主状态方程的时序因果拆分（估计方程 vs 预测方程）
   - `theta` 的符号解释增加条件依赖说明
   - `beta_norm` 的量纲和跨口径一致性声明

2. **建议修正**：
   - `phi`/`theta` 稳定性条件的表述一致性
   - `q_type` 跨股可比性的明确边界
   - `mode_switch` 滞回规则的灵敏度说明
   - `data_leakage_check` 的自动化与人工审查分层

3. **补充完善**：
   - HP滤波的参数设置（若支持）
   - `beta_norm_floor` 的历史样本不足处理
   - 跨股标准化方法（若跨股比较是需求）

**风险等级**：中等。当前文档的缺陷主要集中在**概念解释的精确性**和**时序逻辑的严谨性**上，不直接破坏模型结构，但可能导致实现时的口径漂移和解释歧义。
