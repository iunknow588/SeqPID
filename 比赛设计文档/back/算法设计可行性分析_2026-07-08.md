# 算法设计可行性分析

## 任务信息
- **分析目标**: 基于状态空间辨识的三类行为代理因子贡献度拆解算法
- **文档版本**: 算法设计 1.1 版本
- **分析日期**: 2026-07-08
- **分析结论**: 整体可行，但存在关键风险点需闭环

---

## 一、总体可行性结论

**结论：算法设计在理论层面具备可实现性，但工程落地存在 3 个关键风险点，需逐一闭环后方可进入编码阶段。**

| 维度 | 评估 | 说明 |
|------|------|------|
| 理论自洽性 | ✅ 可行 | 控制论视角的 PID 分解框架逻辑自洽，工程展开式合理 |
| 数据可得性 | ⚠️ 待验证 | 依赖 Level2 逐笔数据 schema 探针结果 |
| 模型可辨识性 | ⚠️ 需预检 | 5 维状态向量在 48 窗口上的可辨识性需模拟验证 |
| 计算复杂度 | ✅ 可控 | 单股 KF/RTS 计算量极小，全市场批处理可接受 |
| 业务解释性 | ✅ 良好 | 分层解耦设计，展示层与建模层分离，解释路径清晰 |

## 二、六大层级可行性逐项分析

### 第1层：行为代理因子分类器

**可行性：条件可行，依赖数据 schema 探针**

| 评估项 | 状态 | 关键问题 |
|--------|------|----------|
| 分类规则 | ✅ 合理 | 大额/小额/多档穿越阈值规则简洁，工程易实现 |
| 阈值初值 | ⚠️ 需验证 | 50万/10万/2档穿越为经验值，需回放验证区分度 |
| 挂单时间字段 | ⚠️ 待确认 | 文档明确不作为 1.1 硬依赖，但需确认 schema 中是否可得 |
| 窗口聚合 | ✅ 可行 | 5分钟窗口净额计算简单，48窗口结构清晰 |

**关键待确认问题：**
1. Level2 数据是否包含逐笔成交的买卖方向标识？（用于计算主动买入/卖出）
2. 逐笔委托与逐笔成交的关联方式？（是否需要订单号匹配）
3. 十档盘口快照的时间粒度与对齐方式？

### 第2层：结构系数识别（核心层）

**可行性：理论可行，但存在可辨识性风险**

#### 2.1 状态空间模型设计

| 要素 | 设计 | 评估 |
|------|------|------|
| 状态向量 | 5维 [φ, β_ch, β_q, β_retail, θ] | 合理，但维度偏高 |
| 观测方程 | ARX 结构，不含滞后残差 | ✅ 明智，避免 ARMAX 的循环依赖风险 |
| 状态转移 | 随机游走 | 标准做法，但需关注 θ 的漂移约束 |
| 滤波算法 | KF + RTS 平滑 | ✅ 成熟方案，实现路径清晰 |

#### 2.2 关键风险点：θ_t 的可辨识性

**问题描述：**
- θ_t 在主观测方程中系数为 0（文档 4.2 节明确说明）
- 这意味着 θ_t 没有直接的观测约束，仅靠过程噪声协方差 Q 的约束维持稳定
- 文档提出三种保持 θ_t 稳定的方案，1.1 版本默认采用"约束 Q_55"

**风险分析：**

```
风险等级：中高风险

原因：
1. 5维状态向量在仅4维观测输入的情况下，θ_t 本质上是"弱观测"状态
2. 仅靠 Q_55 约束可能导致 θ_t 估计对初值敏感
3. 若 θ_t 不稳定，将直接影响第4层 D 项拆分的可靠性

缓解措施（文档已提出）：
- Q_55 = Q_11 × 0.01 ~ 0.1
- 若仍不稳定，回退到"弱观测约束"方案
- 模拟数据验证参数恢复能力
```

**建议补充的验证实验：**
1. 在模拟数据上，固定真实 θ 序列，检验 KF/RTS 能否有效恢复
2. 对比 4 维模型（不含 θ）与 5 维模型的预测残差差异
3. 检查 θ_t 估计序列的条件数与方差膨胀因子

### 第3层：基础归因层

**可行性：可行**

| 要素 | 评估 |
|------|------|
| 权重定义 | 基于流量占比的归一化权重，数学简洁 |
| 基础贡献公式 | β·U + φ·ΔP·w 结构清晰，计算简单 |
| 惯性项处理 | 不显式叠加 ε_{t-1}，避免重复解释，合理 |

---

### 第4层：D项拆分层

**可行性：条件可行，依赖 λ 参数标定**

#### 4.1 设计合理性

| 设计点 | 评估 |
|--------|------|
| D 项主要区分量化 vs 散户 | ✅ 业务逻辑合理，量化偏阻尼、散户偏恐慌反转 |
| 游资 λ 默认 0.05 | ✅ 保守合理，游资主要是冲击方而非阻尼方 |
| 显式归一化权重 s_ch, s_q, s_retail | ✅ 保证结构层闭合性 |

#### 4.2 关键风险点：λ 参数敏感性

**问题描述：**
- λ_ch=0.05, λ_q=1.00, λ_retail=0.40 为行为代理层面的拆分先验
- 文档要求验证"λ 浮动 20% 时日终主导判定变化 < 10%"

**风险分析：**

```
风险等级：中等风险

原因：
1. λ 参数直接影响量化/散户的 D 项分配比例
2. 若 λ 参数标定不当，可能导致量化/散户判定系统性偏差
3. 文档已提出敏感性扫描计划，但需历史样本支持

缓解措施：
- 按文档要求执行 λ 敏感性扫描
- 建立 λ 参数与人工抽检标签的吻合度评估
- 若敏感性过高，考虑将 λ 也纳入状态估计或分层先验
```

## 三、关键风险点汇总与闭环建议

### 风险1：θ_t 可辨识性（高风险）

| 项目 | 内容 |
|------|------|
| 风险描述 | 5维状态向量中 θ_t 无直接观测约束，仅靠 Q 约束，可能不稳定 |
| 影响范围 | 第4层 D 项拆分、量化/散户区分度、诊断层 |
| 闭环动作 | 1. 模拟数据参数恢复实验 2. 4维/5维模型对照 3. 条件数检查 |
| 交付物 | `model_identifiability_report.md` |
| 回退策略 | 若 θ_t 不稳定，回退到 4 维模型，D 项仅作为诊断指标不参与拆分 |

### 风险2：λ 参数敏感性（中风险）

| 项目 | 内容 |
|------|------|
| 风险描述 | D 项拆分权重 λ 对量化/散户判定结果敏感 |
| 影响范围 | 第4层拆分结果、日终主导判定 |
| 闭环动作 | 1. λ 敏感性扫描实验 2. 与人工抽检标签吻合度评估 |
| 交付物 | `lambda_sensitivity_report.md` |
| 回退策略 | 若敏感性过高，缩小 λ 扰动范围或使用更保守默认值 |

### 风险3：Level2 数据 schema 不确定性（中风险）

| 项目 | 内容 |
|------|------|
| 风险描述 | 逐笔数据字段、格式、关联方式尚未验证 |
| 影响范围 | 第1层分类器、整个算法输入 |
| 闭环动作 | 1. Schema 探针 2. 单股单日 48 窗口代理流生成验证 |
| 交付物 | `schema_probe_report.md` |
| 回退策略 | 若关键字段缺失，分类器降级为更简单的阈值版本 |

## 五、实施顺序建议（优化版）

基于可行性分析，建议对文档第12节的实施顺序做如下调整：

```
原顺序：1. schema探针 → 2. 代理流 → 3. ARX+KF/RTS → 4. 基础归因 → 5. D项拆分 → 6. 展示层 → 7. 验收

建议调整为：

阶段A（数据验证）
  A1. Level2 schema 探针
  A2. 单股单日代理流生成验证（确认48窗口可稳定产出）
  A3. 阈值初值回放（50万/10万/2档穿越的基本区分度）

阶段B（模型可辨识性验证）
  B1. 模拟数据参数恢复实验（重点验证 θ_t）
  B2. 4维 vs 5维模型对照
  B3. 输出 model_identifiability_report.md
  【 gates 】若 θ_t 不可辨识，回退到4维模型，D项仅作诊断

阶段C（核心链路实现）
  C1. ARX + KF/RTS 主状态模型
  C2. 基础归因层 + 结构层闭合校验
  C3. 展示层 + 展示层闭合校验

阶段D（D项拆分验证）
  D1. D项拆分层实现
  D2. λ 参数敏感性扫描
  D3. 输出 lambda_sensitivity_report.md
  【 gates 】若敏感性过高，调整 λ 默认值或范围

阶段E（集成与验收）
  E1. 主导判定与比赛字段映射
  E2. 午间热启动验证
  E3. 全链路压测
  E4. 输出 integration_acceptance_report.md
```

## 六、需要讨论的问题

### 问题1：θ_t 的观测约束方案选择

文档 4.5 节提出三种保持 θ_t 稳定的方案：
1. 约束 Q_55（默认方案）
2. 弱观测约束
3. 残差回归二次校正

**建议讨论：**
- 是否应在编码前先实现方案1和方案2的对比实验？
- 若方案1在模拟数据上表现不佳，是否直接回退到4维模型？

### 问题2：λ 参数的标定方式

文档 6.2.1 节提出 λ 敏感性扫描计划，但未明确标定数据来源：
- 是否使用历史样本的人工标注标签？
- 若无人工标注，是否使用规则种子的高置信度样本作为代理真值？

### 问题3：午间热启动的 k_opt 标定

文档 4.8 节建议 k_opt 默认 3.0，经验范围 2.0-5.0：
- 是否有历史数据支持"上午收盘到下午开盘"的预测误差回放？
- 若缺乏历史数据，是否先使用默认值，后续再优化？

### 问题4：与比赛详细设计中 Task 1 的衔接

算法设计主要服务于解释特征，但 Task 1 的模式聚类需要历史样本：
- 历史样本的获取方式？
- 聚类原型的训练数据量要求？

## 七、最终结论

| 维度 | 结论 |
|------|------|
| **整体可行性** | ✅ 可行，但需完成 3 个关键风险闭环 |
| **理论自洽性** | ✅ 控制论框架逻辑自洽，工程展开合理 |
| **工程可实现性** | ⚠️ 依赖 schema 探针和可辨识性验证 |
| **业务价值** | ✅ 解释型特征对 Task 1/2 有明确辅助价值 |
| **建议下一步** | 先执行阶段A（数据验证）和阶段B（可辨识性验证），通过后再进入编码 |

**核心建议：**
1. 不要跳过 `model_identifiability_report.md`，这是整个算法可信度的基石
2. 若 θ_t 在模拟数据上无法稳定恢复，果断回退到 4 维模型，不要强行保留 5 维
3. λ 敏感性扫描不要省略，它决定了量化/散户拆分的可信度
4. 整个算法定位为"解释型辅助引擎"，不要试图让它独立承担 Task 1/2 的全部判定责任
