## 1. 引言

### 1.1 文档目的

本详细设计说明书面向比赛落地实现，给出模块拆分、数据结构、接口契约、调度入口、导出文件与测试验收标准，确保项目可以：

- Task 1 交易模式识别
- Task 2 参与者类型与资金意图识别

### 1.3 统一字段与模型口径

本文与 `统一资金类型判断规范.md`、`PID原理与实现.md`、`算法设计.md` 保持完全一致，统一遵守以下边界：

1. 规则层字段：
   - `CH_rule / Q_rule / R_seed` 只表示行为锚点与种子流
   - 不表示最终模型外力贡献
2. PID 层字段：
   - `c_p / c_i / c_d / eps` 负责价格闭合
   - `phi / theta / beta_*` 是状态参数
3. 模型外力贡献字段：
   - `capital_ch / capital_q / capital_retail` 只能由 `beta_* U_*` 计算得到
   - 不得由 `CH_rule / Q_rule / R_seed` 直接覆写
   - 不得由 `c_i / c_d` 反解摊派得到
4. 模式边界：
   - `rule_base`：只允许规则近似展示，`is_structural_output = false`
   - `baseline_4d`：`U_ch = CH_rule`，`U_mix = Q_rule + R_seed`
   - `diag_5d / full_5d`：`U_ch / U_q / U_retail` 三路输入独立进入 PID
5. `phi / theta` 只负责市场惯性、阻尼与状态诊断，不参与三类资金身份拆分

- 三类规则流：`CH_rule / Q_rule / R_seed`
- 48 窗口状态建模

- 稳定性检测

#### 5.2.3 核心函数

补充要求：

6. `signed_large_active_amount / signed_mix_qr_amount` 仅作为历史兼容输入，进入规则层后必须统一转换为 `CH_rule / Q_rule / R_seed`

### 5.3 市场 PID 聚合

统一补充约束：

1. 市场 PID 聚合的主口径优先来自个股 `c_p / c_i / c_d` 的横截面统计。
2. `phi / theta / beta_*` 可进入 diagnostics、相对市场解释与后续研究分析，但不直接替代市场 P/I/D 主聚合值。
3. 若个股 PID 结果缺失，允许启用启发式 fallback，但必须在 diagnostics 中标记来源。

#### 5.3.1 设计定位

市场 PID 模块不替代单股模型，而是作为**市场先验层**，服务于：

1. 估计当日全市场状态
2. 生成市场 PID 分布快照
3. 给单股输出增加相对市场偏离指标

#### 5.3.2 输入

- 当日全市场 `DailySample`
- 单股 `PatternResult`
- 单股 `PredictResult`

#### 5.3.3 核心函数

```python
def estimate_market_pid(samples: list[DailySample], pattern_results: list[PatternResult],
                        predict_results: list[PredictResult], config: dict) -> MarketPidSnapshot:
    ...

def attach_market_relative_metrics(samples: list[DailySample], snapshot: MarketPidSnapshot) -> list[DailySample]:
    ...
```

#### 5.3.4 核心口径

市场广度：

```text
ADR = up_count / down_count
Breadth = (up_count - down_count) / (up_count + down_count)
```

市场 PID 横截面聚合：

```text
P_market_dist = dist(P_stock for all stocks)
I_market_dist = dist(I_stock for all stocks)
D_market_dist = dist(D_stock for all stocks)
```

相对市场偏离：

```text
P_rel = (P_stock - median(P_market)) / std(P_market)
I_rel = (I_stock - median(I_market)) / std(I_market)
D_rel = (D_stock - median(D_market)) / std(D_market)
```

#### 5.3.5 市场状态标签

首版可先支持以下标签：

- `强趋势上行`
- `弱趋势上行`
- `震荡中性`
- `弱趋势下行`
- `风险偏好退潮`

标签判定综合考虑：

- 上涨/下跌家数比值
- 净买方向均值
- 强势模式占比
- 高撤单占比
- 市场 PID 中位数及离散度

### 6.1 设计思路

## 7. Task 2 参与者识别模块详细设计

### 7.1 总体流程

Task 2 采用三阶段输出：

1. 规则层三类行为识别：`CH_rule / Q_rule / R_seed`
2. `capital_type` 三分类：`游资 / 量化 / 散户`
3. `capital_intention` 多分类

### 7.2 参与者类型识别

#### 7.2.1 输入特征

参与者类型模型重点使用：

- 订单规模分布
- 拆单连续性
- 订单节奏稳定性
- 撤单风格
- 盘口穿越行为
- 规则层特征：`CH_rule`、`Q_rule`、`R_seed`、`low_fallback` 占比
- 状态解释特征：`phi`、`theta`、`beta_ch`、`beta_q`、`beta_mix`、`beta_retail`、`noise_ratio`
- 外力贡献特征：`capital_ch`、`capital_q`、`capital_retail`

#### 7.2.2 建模策略

```python
def predict_capital_type(sample: DailySample, model_bundle: dict) -> tuple[str, float, dict]:
    ...
```

推荐实现：

- 首版主口径：`capital_ch / capital_q / capital_retail` 外力贡献主导项 + 规则一致性校验
- 辅助模型：`LightGBM` / `CatBoost`
- 若真实标签不足，先通过规则引擎生成高精度种子，再做弱监督扩展

#### 7.2.3 规则种子

示例高置信度种子：

- 游资种子：超大单主动买入高、尾盘冲击高、拆单一致性低、短时爆发强
- 量化种子：买卖节奏稳定、拆单相似度高、方向净值低、T0 风格明显
- 散户种子：被动成交为主、`order_age_minutes > 5`、跟随性强、跨窗口零散存活

#### 7.2.4 输出边界

1. `capital_type` 的正式主口径应优先使用模型外力贡献主导项
2. `Q_rule` 与 `R_seed` 不能直接覆写为最终 `capital_q / capital_retail`
3. `rule_base` 模式只允许输出 `capital_*_rule_approx`，不得把规则近似值写入模型贡献字段
4. `low_fallback` 事件必须参与窗口聚合，但应保留低置信诊断标记

### 7.3 资金意图识别

#### 7.3.1 标签空间

实际标签空间由 `label_dict.yaml` 统一管理，支持两种模式：

1. **细标签模式**
   - 买入、卖出、中性、T0交易、吸筹、试盘、拉升、出货、对倒、做市
2. **压缩标签模式**
   - 买入、卖出、中性、T0交易

#### 7.3.2 设计原则

- 资金意图必须依赖高频行为特征推断，禁止股票代码硬规则
- 当细标签置信度不足时，可回退到压缩标签

## 11. 异常与降级设计

### 11.1 数据异常

| 异常 | 处理策略 |
|---|---|
| 某股票无成交数据 | 该股输出空结果并记录错误，不阻塞全批 |
| 缺少撤单数据 | 撤单特征置缺失，降低相关模型置信度 |
| 缺少盘口快照 | 盘口压力与价格发现类特征置缺失 |
| 缺少 `order_lifetime_ms` 或等效委托存活时间 | 主动成交按 `100000` 金额阈值低置信回退；被动成交若无法恢复 `order_age_minutes`，不得输出高置信散户，需按最弱可用规则低置信归类并追加 `fallback_reason` |
| 四类文件 schema 不一致 | 直接终止批处理并输出 schema 告警 |

### 11.2 模型异常

| 异常 | 处理策略 |
|---|---|
| 原型库未加载成功 | 终止 Task 1，并返回明确异常 |
| 参与者模型未加载成功 | 若规则引擎可用则降级，否则终止 Task 2 |
| 状态解释引擎失败 | 不阻塞主流程，但将相关辅助特征置缺失 |
| HDBSCAN 聚类结果不可用 | 自动回退到 KMeans/KMedoids 基线方案 |
| 市场 PID 聚合失败 | 不阻塞主提交流程，但输出缺失告警并跳过市场状态文件 |

### 11.3 提交异常

| 异常 | 处理策略 |
|---|---|
| CSV 字段顺序错误 | 导出前校验器直接失败 |
| 行数不一致 | 禁止打包 `submit.zip` |
| 空标签 | 禁止输出提交文件 |
