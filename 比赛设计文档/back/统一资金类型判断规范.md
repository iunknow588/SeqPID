# 统一资金类型判断规范

文档版本：V1.1  
文档状态：正式版  
适用范围：连续竞价、集合竞价下的买入/卖出资金行为判断，以及到 PID 输入变量的统一映射  

## 1. 文档目标

本文档用于统一比赛系统中的三类资金行为判断口径，明确：

1. 三类资金的行为定义
2. 连续竞价与集合竞价的判断规则
3. 参数配置原则
4. 事件级与窗口级输出字段
5. 到 PID 输入变量的映射关系
6. 与模型层的边界

本文档强调：

> 本规范识别的是“行为作用类型”，不是“真实账户身份”。

> 若行为解释与工程可实现性冲突，以基于现有 Level2 数据可稳定实现、可复现、可验收的规则为准；相关输出均视为行为代理标签，而非真实主体识别。

## 2. 三类资金的统一定义

## 2.1 游资

定义：

> 游资是指在连续竞价或集合竞价中，试图通过主动成交或更激进的委托价格来直接塑造价格路径的行为类型。

行为特征：

1. 主动性强
2. 执行集中
3. 价格冲击明显
4. 追求短时间内推动或压制价格
5. 愿意承担较高冲击成本

典型表现：

1. 大额主动吃单
2. 大额主动砸盘
3. 多档跨价
4. 短时间内直接推升或打压成交价格

## 2.2 量化

定义：

> 量化是指在连续竞价中，既可能主动成交也可能被动成交，但整体表现为窗口内程序化、重复性较强、冲击相对可控的执行行为类型。

行为特征：

1. 节奏稳定
2. 分批执行
3. 可同时出现主动成交与被动成交
4. 单笔冲击通常低于游资
5. 更强调窗口内持续执行而非跨窗口等待

典型表现：

1. 中大额但不激进
2. 贴价或小幅跨价
3. 连续多笔、方向一致
4. 委托与成交主要集中在当前时间窗口内

说明：

本文档统一使用“量化”，不再使用“量化/机构”。

## 2.3 散户

定义：

> 散户是指在连续竞价中以被动成交为主、对既有价格变化做跟随反应，且委托时间跨度通常长于量化的行为类型。

行为特征：

1. 跟随性强
2. 被动适应
3. 常在价格已变化后再行动
4. 零散、小额、持续性弱
5. 更像顺势跟进，而不是主动塑造走势

典型表现：

1. 小额追价买入
2. 小额恐慌卖出
3. 零散挂单并在较长时间后成交
4. 更像跟随已有涨跌，而不是主动塑造价格

## 3. 总体判断原则

## 3.1 先看行为目标，再看行为强度

统一判断优先级：

1. 连续竞价中，主动成交优先进入游资/量化拆分
2. 连续竞价中，被动成交优先进入量化/散户拆分
3. 集合竞价单独处理，不沿用连续竞价主动/被动拆分

## 3.2 行为强度优先于金额规模

金额只作为增强项，不单独决定类型；是否主动塑价、是否在 5 分钟内成交、是否属于实际成交委托，优先级高于金额本身。

因此：

1. 大额但不主动、不急迫，不应直接判为游资
2. 小额但委托价格已体现主动抬价/压价意图时，仍可判为游资行为，但应降低置信度
3. 中大额、窗口内重复执行、单笔冲击较低，更应优先解释为量化

## 3.3 规则层输出的是锚点与种子流

规则层的职责不是给出最终模型贡献，而是输出：

1. `CH_rule`：游资规则锚点
2. `Q_rule`：量化规则锚点或量化种子流
3. `R_seed`：散户种子流

最终三类资金代理贡献由 PID 模型中的 `beta_* U_*` 外力加载项计算；`phi / theta` 仅用于市场系统响应诊断，不反解为资金身份。

补充统一理论口径：规则层向 PID 层输送的是交易时间 `q` 轴上的窗口级行为流。这里的 `q` 可由成交量窗口序号或累计成交量进度离散化得到；价格状态是 `P(q)`，而不是把股价本身解释成速度。

## 4. 核心判断维度

| 维度 | 含义 | 主要用途 |
| --- | --- | --- |
| 主动性 | 是否主动吃单/砸盘 | 区分游资与非游资 |
| 价格冲击 | 是否跨价、跨几档、是否显著改变虚拟成交价 | 区分游资与非游资 |
| 执行节奏 | 是否瞬时集中还是分批持续 | 区分游资与量化 |
| 金额规模 | 单笔或阶段累计金额 | 增强置信度 |
| 趋势相对位置 | 是领涨领跌还是跟随 | 区分主动塑造与顺势适应 |
| 委托存活时间 | 委托从挂出到成交的时间差是否超过 5 分钟 | 区分被动量化与散户 |
| 时间约束 | 是否处于不可撤单等强约束阶段 | 调整证据权重 |

建议统一定义以下可计算字段：

1. `order_age_minutes = (trade_time - order_time) / 60`
2. `is_active_order`：是否可判定为主动成交
3. `is_passive_order`：是否可判定为被动成交
4. `fallback_active_hot = (is_active_order = true) and (order_amount > active_fallback_hot_amount)`
5. `fallback_passive_retail = (is_passive_order = true) and (order_age_minutes > passive_survival_minutes)`

## 5. 判定执行顺序

为避免同一事件多重解释，规则层必须按固定顺序执行。

## 5.1 基础可用性检查

先检查：

1. 主动方向是否可恢复
2. 是否具备必要盘口价格字段
3. 时间戳是否可确定交易场景
4. 是否可归属到明确窗口

若关键字段不足，则：

1. 按最弱可用规则做低置信回退分类
2. 若只能确认是主动成交，但盘口快照缺失、无法确认是否达到卖二/买二等更强攻击性条件，则按委托金额回退：大于 `100000` 的优先判为 `hot_money`，否则判为 `quant`
3. 主动方向无法恢复时，优先回退为 `quant`
4. 缺少关键盘口字段但能确认被动成交时，按委托时间阈值回退为 `quant` 或 `retail`
5. 记录 `fallback_reason`

## 5.2 场景分流

1. 连续竞价：先按主动/被动成交分流，再分别判游资、量化、散户
2. 集合竞价：单独处理，按成交量统计，不沿用连续竞价主动/被动链路

## 5.3 先识别游资

若为主动成交，且高置信满足“主动塑造价格”条件，则：

1. 直接判为 `hot_money`
2. 不再参与量化/散户判断

## 5.4 再区分量化与散户

连续竞价中仅在未判为游资时：

1. 主动成交但未达到游资标准 -> `quant`
2. 被动成交且委托时间距成交时间不超过 5 分钟 -> `quant`
3. 被动成交且委托时间距成交时间超过 5 分钟 -> `retail`
4. 若证据不足或冲突，则主动成交按委托金额回退：大于 `100000` 判为 `hot_money`，否则判为 `quant`；被动成交按 5 分钟阈值回退为 `quant` 或 `retail`

## 5.5 最后做置信度修正

统一修正以下因素：

1. 涨跌停附近
2. 集合竞价撤单噪声
3. 单笔极端但无持续
4. 方向恢复可靠性不足

## 5.6 兜底判定矩阵

当主规则无法直接给出高置信结果时，统一按以下顺序回退：

| 可确认信息 | 回退条件 | 归类结果 |
| --- | --- | --- |
| 可确认主动成交 | `order_amount > 100000` | `hot_money` |
| 可确认主动成交 | `order_amount <= 100000` | `quant` |
| 可确认被动成交 | `order_age_minutes <= 5` | `quant` |
| 可确认被动成交 | `order_age_minutes > 5` | `retail` |
| 集合竞价平盘 | 无更强区分证据 | `quant` |
| 集合竞价且 `P_order = P_auction_trade` | 无更强区分证据 | `quant` |

## 6. 连续竞价规则

## 6.1 买入规则

定义：

1. `P_order`：委托价
2. `P_best_ask`：委托时卖一价
3. `tick_size`：最小变动价位
4. `N_tick_buy = floor((P_order - P_best_ask) / tick_size)`

含义：

1. `N_tick_buy >= 1`：主动向上跨价
2. `N_tick_buy >= 3`：强攻击性买入

### A. 游资买入

同时满足以下特征越多，越倾向游资：

1. 主动买入
2. 委托价格高于委托时卖一价，且达到卖二或更高价位
3. 价格体现出主动抬价意图
4. 时间上集中或同方向重复出现
5. 金额较大仅作为增强项

解释：

> 只要买入委托的主要作用是主动抬价或主动塑造上涨价格路径，即判为游资买入；金额大小影响置信度，但不作为唯一前提。

输出：

- `buy_ch_anchor += signed_amount`

### B. 量化买入

同时满足以下特征越多，越倾向量化：

1. 非游资
2. 主动成交但未达到游资标准，或被动成交且委托时间距成交时间不超过 5 分钟
3. 在窗口内多次重复出现
4. 更像程序化执行而非跨窗口等待
5. 单笔冲击通常弱于游资

解释：

> 连续竞价中，只要买入订单未达到游资的主动塑价标准，或其为被动成交且委托时间距成交时间不超过 5 分钟，则优先判为量化买入。

输出：

- `buy_q_anchor += signed_amount`

### C. 散户买入

同时满足以下特征越多，越倾向散户：

1. 非游资
2. 被动成交
3. 委托时间距成交时间超过 5 分钟
4. 发生在价格已上涨之后或更像跟随已有上涨
5. 行为零散、跨窗口存活

解释：

> 连续竞价中，若买入订单为被动成交，且委托时间距成交时间超过 5 分钟，则优先判为散户买入。

输出：

- `buy_retail_seed += signed_amount`

## 6.2 卖出规则

定义：

1. `P_order`：委托价
2. `P_best_bid`：委托时买一价
3. `N_tick_sell = floor((P_best_bid - P_order) / tick_size)`

含义：

1. `N_tick_sell >= 1`：主动向下砸盘
2. `N_tick_sell >= 3`：强攻击性卖出

### A. 游资卖出

同时满足以下特征越多，越倾向游资：

1. 主动卖出
2. 委托价格低于委托时买一价，且达到买二或更低价位
3. 价格体现出主动压价意图
4. 时间上集中或同方向重复出现
5. 金额较大仅作为增强项

解释：

> 只要卖出委托的主要作用是主动压价或主动塑造下跌价格路径，即判为游资卖出；金额大小影响置信度，但不作为唯一前提。

输出：

- `sell_ch_anchor += signed_amount`

其中卖出方向必须记为负值。

### B. 量化卖出

同时满足以下特征越多，越倾向量化：

1. 非游资
2. 主动成交但未达到游资标准，或被动成交且委托时间距成交时间不超过 5 分钟
3. 在窗口内多次重复出现
4. 更像程序化执行而非跨窗口等待
5. 单笔冲击通常弱于游资

解释：

> 连续竞价中，只要卖出订单未达到游资的主动塑价标准，或其为被动成交且委托时间距成交时间不超过 5 分钟，则优先判为量化卖出。

输出：

- `sell_q_anchor += signed_amount`

### C. 散户卖出

同时满足以下特征越多，越倾向散户：

1. 非游资
2. 被动成交
3. 委托时间距成交时间超过 5 分钟
4. 发生在价格已走弱之后
5. 行为零散、跨窗口存活

解释：

> 连续竞价中，若卖出订单为被动成交，且委托时间距成交时间超过 5 分钟，则优先判为散户卖出。

输出：

- `sell_retail_seed += signed_amount`

其中卖出方向必须记为负值。

## 6.3 连续竞价分类矩阵

| 行为特征 | 主要解释 | 资金类型 | 输出去向 |
| --- | --- | --- | --- |
| 主动成交 + 明显跨价/砸盘 + 直接改变价格路径 | 主动塑造走势 | 游资 | `CH_rule` |
| 主动成交但未达游资标准，或被动成交且委托时间距成交时间不超过 5 分钟 | 5 分钟内程序化执行 | 量化 | `Q_rule` |
| 被动成交 + 委托时间距成交时间超过 5 分钟 + 跟随已有涨跌 | 被动适应走势 | 散户 | `R_seed` |
| 主动成交但盘口攻击性细节缺失 | 按 `100000` 金额阈值低置信回退 | 游资或量化 | `CH_rule` / `Q_rule` |
| 被动成交但盘口细节缺失 | 按 `5` 分钟阈值低置信回退 | 量化或散户 | `Q_rule` / `R_seed` |

## 7. 集合竞价规则

集合竞价单独处理，不沿用连续竞价中的主动/被动拆分。集合竞价阶段仅统计实际成交的委托，不统计未成交委托；并根据竞价结果对应的价格方向，以及委托价相对成交价的位置关系，区分游资与量化。

## 7.1 关键变量

定义：

1. `P_auction_trade`：集合竞价成交价
2. `P_prev_close`：前收盘价
3. `delta_auction = (P_auction_trade - P_prev_close) / P_prev_close`
4. `P_order`：委托价
5. `V_trade`：集合竞价实际成交金额

方向定义：

1. `delta_auction > 0`：股价上涨
2. `delta_auction < 0`：股价下跌
3. `delta_auction = 0`：股价平盘，默认按量化低置信处理

说明：

1. 集合竞价阶段的平盘事件不再单独保留未定状态
2. 若无额外证据，平盘与成交价相等事件统一按量化低置信处理

## 7.2 集合竞价成交分类规则

### A. 上涨场景

若 `delta_auction > 0`，则仅对实际成交的委托按以下规则分类：

1. `P_order > P_auction_trade`：判为游资
2. `P_order < P_auction_trade`：判为量化
3. `P_order = P_auction_trade`：按量化低置信处理

解释：

> 在股价上涨的集合竞价场景中，委托价高于最终成交价的成交委托，视为更主动推动价格的游资行为；委托价低于最终成交价的成交委托，视为量化行为。

### B. 下跌场景

若 `delta_auction < 0`，则仅对实际成交的委托按以下规则分类：

1. `P_order < P_auction_trade`：判为游资
2. `P_order > P_auction_trade`：判为量化
3. `P_order = P_auction_trade`：按量化低置信处理

解释：

> 在股价下跌的集合竞价场景中，委托价低于最终成交价的成交委托，视为更主动压低价格的游资行为；委托价高于最终成交价的成交委托，视为量化行为。

### C. 统计边界

1. 仅统计集合竞价阶段实际成交的委托
2. 未成交委托不纳入游资、量化或散户统计
3. 集合竞价阶段默认不做散户单独识别
4. 集合竞价分类结果按成交金额累计

## 8. 试盘与虚假挂单处理

集合竞价阶段未成交挂单不进入资金类型主统计，但可作为诊断信号单独记录。

若出现：

1. 可撤单阶段大额挂单明显推动虚拟成交价
2. 临近不可撤单前快速撤销
3. 不可撤单阶段未保留对应净意图

则应解释为：

1. `trial_probe`
2. `fake_order_signal`
3. `auction_noise`

处理方式：

1. 只记诊断标签
2. 不计入主分类成交统计
3. 仅作低权重辅助证据

说明：

1. `trial_probe / fake_order_signal / auction_noise` 是诊断标签，不是第四类资金类型
2. 若相关委托最终成交，仍必须按游资、量化、散户三类主规则之一归类

## 9. 冲突裁决规则

## 9.1 游资与量化冲突

若事件同时具备：

1. 金额大
2. 有一定主动性
3. 又有持续执行特征

则：

1. 若核心是“速度与瞬时位移” -> 判游资
2. 若核心是“主动但未显著塑价，且更像窗口内程序化执行” -> 判量化

优先级：

1. 价格瞬时位移
2. 时间集中度
3. 跨价/砸盘深度
4. 执行持续性

## 9.2 量化与散户冲突

若事件：

1. 非游资
2. 为被动成交
3. 金额不构成游资强证据
4. 又像在跟随已有走势

则：

1. 若被动成交且委托时间距成交时间不超过 5 分钟 -> 判量化
2. 若被动成交且委托时间距成交时间超过 5 分钟 -> 判散户
3. 若无法明确区分，则按委托时间阈值低置信回退

## 9.3 游资与散户冲突

小额但明显主动抬价或主动压价的委托，可直接视为游资行为候选。

若同时满足以下因素越多，则游资置信度越高：

1. 委托价格达到对手二档或更高价位
2. 价格位移意图显著
3. 速度足够快
4. 后续仍有同向跟进

若仅能确认其为被动成交，且委托时间距成交时间超过 5 分钟，则优先解释为散户种子。

## 10. 特殊情形修正

## 10.1 接近涨跌停

修正规则：

1. 接近涨停的大额主动买入：游资权重可上调
2. 接近跌停的大额主动卖出：游资权重可上调
3. 接近涨停的小额追价买入：更偏散户
4. 接近跌停的小额恐慌卖出：更偏散户
5. 剩余价格空间不足时，整体应降权

## 10.2 方向恢复不可靠

若主动买卖方向恢复不稳定，则：

1. 主动成交按委托金额回退：大于 `100000` 判为游资，否则判为量化
2. 被动成交按 5 分钟阈值回退为量化或散户
3. 同时降低最终置信度

## 10.3 单笔极端但后续无持续

若只有单笔极端冲击，但后续无持续成交、无同向重复执行，则：

1. 仅视为弱游资候选
2. 不直接作为高置信游资强锚点

## 11. 置信度计算规范

规则层必须输出分类结果与置信度。

## 11.1 基础得分项

建议统一计算以下得分，范围均为 `[0,1]`：

1. `aggressive_score`
2. `amount_score`
3. `sustain_score`
4. `follow_score`
5. `reliability_score`

## 11.2 三类原始分数

建议分别计算：

\[
score_{hot}
\]

\[
score_{quant}
\]

\[
score_{retail}
\]

其中：

1. `score_hot` 更依赖攻击性、价格位移、时间集中度
2. `score_quant` 更依赖窗口内执行、分批性、低冲击
3. `score_retail` 更依赖被动成交、委托时间跨度、跟随性

## 11.3 最终置信度

建议：

\[
confidence = score_{winner} \times reliability\_score \times adjuster
\]

其中 `adjuster` 为特殊情形修正项。

## 11.4 置信度分层

| 区间 | 等级 |
| --- | --- |
| `>= 0.80` | `high` |
| `[0.60, 0.80)` | `medium` |
| `[0.40, 0.60)` | `low` |
| `< 0.40` | `low_fallback` |

若 `< 0.40`，仍需输出三类资金之一，但必须标记为低置信回退结果。

建议补充解释：

1. `high`：主规则直接命中，且证据充分
2. `medium`：主规则命中，但增强证据有限
3. `low`：主规则命中，但部分证据较弱
4. `low_fallback`：主规则不足，依赖兜底矩阵完成最终归类

## 14. 窗口级聚合规范

对每个窗口 `t` 聚合：

\[
buy\_ch\_anchor_t
\]

\[
sell\_ch\_anchor_t
\]

\[
buy\_q\_anchor_t
\]

\[
sell\_q\_anchor_t
\]

\[
buy\_retail\_seed_t
\]

\[
sell\_retail\_seed_t
\]

其中：

1. 买入为正
2. 卖出为负
3. 所有事件都必须归入游资、量化、散户三类之一
4. 集合竞价阶段若规则仅输出游资/量化，则散户相关字段记为 `0`
5. `low_fallback` 事件同样参与窗口级聚合，但应保留回退标记供后续诊断

进一步定义：

\[

\]

\[

\]

\[
R^{seed}_t = buy\_retail\_seed_t + sell\_retail\_seed_t
\]

## 15. 进入 PID 前的预处理

固定顺序如下：

1. 窗口内按主标签求有符号净额
2. 对净额做 1% 缩尾
3. 缺失补零
4. 执行 EWMA 自适应标准化
5. 输出给 PID 输入对象

不允许：

1. 先标准化再求和
2. 买卖方向各自标准化后再相减
3. 直接把事件级数据喂给 PID 主状态方程

## 16. 到 PID 输入变量的映射

本规范与 [PID原理与实现.md](</e:/2026OPC大赛/自动化交易/比赛设计文档/PID原理与实现.md>) 必须保持一致。

一致性要求补充如下：

1. `CH_rule / Q_rule / R_seed` 是交易时间 `q` 轴上的窗口级输入流。
2. PID 理论主模型默认使用二阶差分解释，即 `Delta_q P` 为速度，`Delta_q^2 P` 为加速度。
3. 若某处为了稳健性引用一阶写法，必须显式注明“工程近似”，不能改写本规范的理论口径。

## 16.1 主流程映射

| 规则层字段 | PID 输入变量 | 说明 |
| --- | --- | --- |
| `CH_rule_t` | `U_ch,t` | 游资主输入 |
| `Q_rule_t` | `U_q,t` | 量化输入；语义上仍属于规则锚点/种子流 |
| `R_seed_t` | `U_retail,t` | 散户输入；语义上仍属于规则种子流 |

## 16.2 `baseline_4d` 映射

\[
U_{ch,t} = CH^{rule}_t
\]

\[
U_{mix,t} = Q^{rule}_t + R^{seed}_t
\]

## 16.3 `full_5d` 映射

\[
U_{ch,t} = CH^{rule}_t
\]

\[
U_{q,t} = Q^{rule}_t
\]

\[
U_{retail,t} = R^{seed}_t
\]

## 17. 与模型层的边界

规则层字段不能直接等价替换最终模型输出：

1. `capital_ch`
2. `capital_q`
3. `capital_retail`

两者关系如下：

| 层级 | 字段 | 含义 |
| --- | --- | --- |
| 规则层 | `CH_rule / Q_rule / R_seed` | 行为锚点与种子流 |
| 模型层 | `c_p / c_i / c_d / eps` | 一级 PID 贡献 |
| 模型层 | `capital_ch / capital_q / capital_retail` | 由 `beta_* U_*` 计算得到的资金代理外力贡献 |

因此：

1. 规则层负责识别行为起点
2. 模型层负责闭合价格变化并给出最终外力贡献解释
3. 若下游仅做规则兜底展示，应单独输出 `capital_ch_rule_approx / capital_q_rule_approx / capital_retail_rule_approx` 或等价字段，不得覆写模型层 `capital_ch / capital_q / capital_retail`

## 18. 一致性与验收要求

## 18.1 规则一致性

必须满足：

1. 同一事件不能进入多个互斥主标签
2. 买入恒为正，卖出恒为负
3. 低置信事件不能强制输出为高置信游资/量化/散户

## 18.2 聚合一致性

必须满足：

1. 窗口级净额等于事件级净额有符号求和
2. 缩尾、补零、标准化顺序固定
3. 4维与5维模式字段语义一致，仅输入维度不同
4. 4维与5维在工程实现上必须拆分为独立处理文件；规则层只保证输入口径一致，不要求两种模式共用同一实现文件。

## 18.3 PID 映射一致性

必须满足：

1. `CH_rule` 可直接进入 `U_ch`
2. `Q_rule` 与 `R_seed` 可按约定映射进入 PID 输入
3. `Q_rule` 与 `R_seed` 不直接替代最终模型外力贡献
4. 可稳定计算 `capital_anchor_error`
5. 建议同步输出 `rule_error_q / rule_error_retail` 作为量化、散户规则流与模型外力贡献的一致性诊断

## 19. 禁止的口径漂移

工程实现中禁止：

1. 将 `Q_rule` 直接当作最终 `capital_q`
2. 将 `R_seed` 直接当作最终 `capital_retail`
3. 将 `phi / theta` 响应项反解或摊派为某一类资金身份
4. 将卖出方向先写成正数后在模型层修正
5. 同一事件重复进入多个资金池
6. 重新引入“量化/机构”混合术语
7. 为追求单日拟合效果临时修改字段语义
8. 在纯规则兜底模式下，继续复用 `capital_ch / capital_q / capital_retail` 承载规则近似值

## 20. 一句话总结

本规范将资金行为统一定义为：

1. 连续竞价中，主动塑价 -> 游资
2. 连续竞价中，主动但未达游资标准，或被动且委托时间距成交时间不超过 5 分钟 -> 量化
3. 连续竞价中，被动且委托时间距成交时间超过 5 分钟 -> 散户
4. 集合竞价中，仅统计实际成交委托；上涨/下跌方向下按委托价相对成交价关系区分游资与量化

规则层负责识别这三类行为起点，PID 模型负责完成价格闭合，并通过 `beta_* U_*` 计算最终资金代理外力贡献；`phi / theta` 负责市场惯性与阻尼诊断。
