# PID原理与实现

文档版本：V1.1  
文档状态：正式版  
适用范围：比赛系统中基于窗口级价格变化与行为代理流的解释型状态模型  

## 1. 文档目标

本文档用于说明比赛系统中 PID 解释引擎的：

1. 理论出发点
2. 数学建模方式
3. 工程实现结构
4. 输出字段定义
5. 验证与回退原则

本文档强调：

> 这里的 PID 不是传统工业控制器的直接搬运，而是一个“借用 P / I / D 作用框架来解释价格形成过程”的状态空间模型。

## 2. 核心思想

我们将单只股票在固定窗口内的价格增量 `ΔP_t` 视为一个受行为流驱动、并带有惯性与阻尼的动态系统输出。

在这个框架中：

1. **P 项**表示当期行为流对价格的直接推动作用
2. **I 项**表示上一窗口价格变化与残差记忆带来的延续作用
3. **D 项**表示价格变化率、反转速度与阻尼修正作用

因此，单窗口价格变化可以写成：

\[
\Delta P_t = C_{P,t} + C_{I,t} + C_{D,t} + \varepsilon_t
\]

其中：

1. `C_P,t`：推进贡献
2. `C_I,t`：惯性 / 积分贡献
3. `C_D,t`：阻尼 / 反转贡献
4. `ε_t`：当期未解释扰动

## 3. P / I / D 的业务含义

### 3.1 P 项

P 项反映：

> 当前窗口行为流对价格的即时推动。

典型含义：

1. 顺势推进
2. 方向确认
3. 直接冲击

### 3.2 I 项

I 项反映：

> 价格延续、情绪惯性、历史偏差吸收。

典型含义：

1. 趋势延续
2. 情绪扩散
3. 尚未被吸收的历史偏离继续传递

### 3.3 D 项

D 项反映：

> 价格变化率的变化以及对应的阻尼、反转、修正作用。

典型含义：

1. 均值回归
2. 阻尼修复
3. 尾部反转
4. 趋势加速或失控

## 4. 行为代理流与 PID 的关系

本系统不是按真实账户身份建模，而是先将逐笔行为聚合成三类行为代理流：

1. `U_ch`：游资行为代理流  
   含义：主动塑造价格路径的资金流
2. `U_q`：量化行为代理流  
   含义：主动但未达游资标准，或被动且委托时间距成交时间不超过 5 分钟的资金流
3. `U_retail`：散户行为代理流  
   含义：被动且委托时间距成交时间超过 5 分钟的资金流

其中：

1. 游资、量化、散户三类行为代理流首先作为外部输入进入 P 项
2. `beta_ch / beta_q / beta_retail` 表示三类外部资金流的即时加载强度
3. `phi / theta` 表示市场自身的惯性与阻尼响应，不再拆分为某一类资金的专属贡献

注意：

> 资金类型识别与系统响应诊断必须分层处理：三类资金贡献来自 `beta_* U_*` 外力项，`phi / theta` 用于解释市场底层状态，不作为三类资金贡献的反解来源。

## 4.1 与规则层的直接对应

资金类型规则层先对逐笔委托或逐笔成交做事件级分类，再聚合成窗口级代理流：

1. 连续竞价主动塑价事件 -> `CH_rule`
2. 连续竞价量化事件 -> `Q_rule`
3. 连续竞价散户事件 -> `R_seed`
4. 集合竞价成交事件按同样口径累计到 `CH_rule` 或 `Q_rule`

统一聚合后得到：

\[
CH_t^{rule},\quad Q_t^{rule},\quad R_t^{seed}
\]

这些量是 PID 状态模型的直接输入，不再额外引入第四类混合标签。

## 5. 工程建模框架

## 5.1 一级 PID 闭合关系

窗口级价格变化满足：

\[
\Delta P_t = C_{P,t} + C_{I,t} + C_{D,t} + \varepsilon_t
\]

进一步定义：

\[
C_{P,t} = \beta_{ch,t} U_{ch,t-1} + \beta_{q,t} U_{q,t-1} + \beta_{retail,t} U_{retail,t-1}
\]

在基线模式下，可退化为：

\[
C_{P,t} = \beta_{ch,t} U_{ch,t-1} + \beta_{mix,t} U_{mix,t-1}
\]

其中基线模式下统一定义：

\[
U_{mix,t} = Q_t^{rule} + R_t^{seed}
\]

\[
C_{I,t} = \phi_t \Delta P_{t-1} + \kappa_I \varepsilon_{t-1}
\]

\[
D\_driver_t = \Delta P_{t-1} - \Delta P_{t-2}
\]

\[
C_{D,t} = \theta_t D\_driver_t
\]

这里：

1. `β_*` 表示行为流对价格的基础加载系数
2. `φ_t` 表示惯性 / 延续系数
3. `θ_t` 表示阻尼 / 反转系数
4. `κ_I` 表示残差记忆强度

## 5.2 状态向量

### A. 4维基线模式

\[
\psi_t^{(4)} = [\phi_t,\ \beta_{ch,t},\ \beta_{mix,t},\ \theta_t]^T
\]

适用场景：

1. 已有高置信游资锚点
2. 希望保留更紧凑的输入维度
3. 需要优先保证模型可运行与可闭合

### B. 5维增强模式

\[
\psi_t^{(5)} = [\phi_t,\ \beta_{ch,t},\ \beta_{q,t},\ \beta_{retail,t},\ \theta_t]^T
\]

适用场景：

1. 量化种子流与散户种子流质量通过验证
2. D 驱动项有效
3. `theta` 可辨识性通过

## 5.3 观测方程

### A. 4维基线模式

\[
x_t^{(4)} =
[\Delta P_{t-1},\ U_{ch,t-1},\ U_{mix,t-1},\ D\_driver_t]^T
\]

\[
\Delta P_t = x_t^{(4)} \cdot \psi_t^{(4)} + \varepsilon_t
\]

### B. 5维增强模式

\[
x_t^{(5)} =
[\Delta P_{t-1},\ U_{ch,t-1},\ U_{q,t-1},\ U_{retail,t-1},\ D\_driver_t]^T
\]

\[
\Delta P_t = x_t^{(5)} \cdot \psi_t^{(5)} + \varepsilon_t
\]

## 5.4 状态转移方程

采用随机游走：

\[
\psi_t = \psi_{t-1} + \eta_t,\quad \eta_t \sim \mathcal{N}(0,Q)
\]

含义：

1. 系统结构不会在相邻窗口瞬间剧烈跳变
2. 但允许参数缓慢漂移以适应市场状态变化

## 6. 为什么采用卡尔曼滤波

本系统中的关键参数：

1. `φ_t`
2. `β_ch,t`
3. `β_q,t` / `β_mix,t`
4. `β_retail,t`
5. `θ_t`

都不是日内固定值。

卡尔曼滤波适合的原因：

1. 能在小样本窗口下递推更新
2. 能把“参数随时间缓慢变化”显式建模
3. 能同时输出状态估计与不确定性
4. 能与 RTS 平滑结合，得到更稳定的事后解释路径

## 6.1 参数辨识的工程定位

利用窗口级行为代理流 `U_ch / U_q / U_retail` 与价格响应 `ΔP`，本系统可以在既定模型结构下估计系统当前的等效加载系数 `beta`、惯性系数 `phi` 与阻尼系数 `theta`。

这里必须强调：

1. 这属于系统辨识（System Identification），不是主观猜测
2. 这里得到的是在样本、噪声假设与模型结构约束下的最优近似估计
3. 这里估计的是模型参数，不是可以被无误差观测的真实市场常数

因此，文档中所谓“还原参数”统一理解为：

> 在给定规则层输入、状态方程、观测方程与噪声约束的前提下，对 `beta / phi / theta` 做后验最优状态估计。

## 6.2 盘后与盘中的两条估计路径

### A. 盘后批处理估计

盘后可利用全天窗口数据，对参数做更稳定的事后估计。常用方式包括：

1. 静态最小二乘或岭回归：适合检验固定参数近似
2. 卡尔曼滤波 + RTS 平滑：适合估计日内缓慢漂移的参数轨迹

若使用 5 维增强模式，可写成：

\[
Y = X \Theta + \varepsilon
\]

其中：

1. `Y` 为窗口级价格变化 `Delta P_t`
2. `X` 为输入矩阵，包含 `U_ch / U_q / U_retail / Delta P_{t-1} / D_driver_t`
3. `Theta` 为待估参数向量

若仅做静态近似检验，可用：

\[
\Theta_{ols} = (X^T X)^{-1} X^T Y
\]

但在正式工程实现中，更推荐使用状态空间模型与 RTS 平滑，因为这更符合参数缓慢漂移的业务假设。

### B. 盘中在线递推估计

盘中实时模式下，使用卡尔曼滤波对参数做递推更新：

1. 上一窗口状态给出先验参数
2. 当前窗口 `Delta P_t` 与行为流提供观测约束
3. 通过预测-更新循环修正 `phi / beta_* / theta`

这样得到的不是单个固定参数，而是一条日内参数轨迹：

1. `phi_1, phi_2, ..., phi_T`
2. `beta_ch,1, beta_ch,2, ..., beta_ch,T`
3. `theta_1, theta_2, ..., theta_T`

这条轨迹本身就是市场结构变化的诊断对象。

## 6.3 可辨识性的三个工程前提

数学上“可求解”不等于工程上“可解释”。要让参数估计有业务意义，至少需要满足以下三类约束。

### A. 多重共线性约束

若 `U_ch / U_q / U_retail` 高度相关，则 `beta_ch / beta_q / beta_retail` 的独立可辨识性会快速下降。

建议：

1. 先计算三类行为流的相关系数矩阵
2. 若任意两类相关性持续过高，应启用岭回归、正则化 KF，或临时退回 `baseline_4d`
3. 不允许在严重共线条件下直接把单日参数解释为“真实独立作用强度”

### B. 有效样本与有效波动约束

窗口数大于参数数只是必要条件，不是充分条件。即使全天有 48 个窗口，若绝大多数窗口中：

1. `U_*` 接近 0
2. `Delta P` 接近 0
3. 价格长时间横盘

则设计矩阵仍可能退化。

建议：

1. 对低波动窗口降低卡尔曼增益
2. 在极端横盘阶段允许冻结参数，沿用上一窗口估计
3. 将“有效窗口数”而不是“总窗口数”作为辨识质量指标

### C. 残差白噪声约束

参数估计完成后，必须检查残差 `eps_t` 是否接近白噪声。

若残差仍存在明显自相关，通常意味着：

1. 状态方程遗漏了重要动态项
2. 规则层输入噪声较大
3. 当前模式的状态维度不足

因此：

1. 可以做 Ljung-Box 检验作为统一验收条件
2. 若检验未通过，应优先检查输入质量与模式设定
3. 不应机械地仅靠增加滞后项解决所有问题

## 6.4 “准确性”的统一表述

为避免概念误导，本文件中的准确性统一表述为：

> 在行为代理流质量可接受、模型结构成立且残差诊断通过的前提下，PID 状态模型可以对 `beta / phi / theta` 给出稳定、可解释、可回测验证的最优近似估计。

不使用以下过强表述：

1. “精确还原真实市场参数”
2. “无误差反推出刚度、惯性、阻尼”
3. “只要窗口数足够就一定能准确识别”

## 7. 参数的工程含义

| 参数 | 含义 | 业务解释 |
| --- | --- | --- |
| `beta_ch` | 游资代理流加载系数 | 游资冲击每单位净流对价格的即时推动强度 |
| `beta_q` / `beta_mix` | 量化或量化+散户合成流加载系数 | 缓慢改变走势的基础推进强度 |
| `beta_retail` | 散户代理流加载系数 | 跟随型流量对价格的加载强度 |
| `phi` | 惯性系数 | 趋势延续与情绪扩散强度 |
| `theta` | 阻尼系数 | 变化率修正、反转、阻尼或正反馈加速强度 |
| `kappa_i` | 残差记忆强度 | 历史未解释偏差被继续吸收的力度 |

## 8. D 项的特别说明

在工程上，D 项最敏感，也最容易失真。

因此这里给出统一原则：

1. D 项不再由 `ε_{t-1}` 直接驱动
2. D 项由 `D_driver_t` 获得主观测约束
3. `D_driver_t` 默认取价格变化率差分：

\[
D\_driver_t = \Delta P_{t-1} - \Delta P_{t-2}
\]

若后续 Level2 盘口特征足够好，可扩展为：

1. `book_recovery`
2. `spread_converge`
3. `cancel_replenish`
4. `pressure_reversal`

但原则上：

> 在 D 驱动项未通过有效性验证前，不能让 `theta` 主导价格闭合诊断；无论是否通过验证，`theta` 都不参与三类资金身份拆分。

## 9. 三类资金代理贡献计算

一级 PID 贡献用于解释价格变化，但比赛系统还需要输出三类资金代理贡献。正式口径不再把 `C_P / C_I / C_D` 通过三组方程反解为三类资金，因为 `C_I` 与 `C_D` 是市场系统响应项，不属于某一类资金的独立外力。

三类资金代理贡献统一由外部行为流加载项给出：

### A. `full_5d / diag_5d`

\[
capital\_ch_t = \beta_{ch,t} U_{ch,t-1}
\]

\[
capital\_q_t = \beta_{q,t} U_{q,t-1}
\]

\[
capital\_retail_t = \beta_{retail,t} U_{retail,t-1}
\]

此时：

\[
C_{P,t}=capital\_ch_t+capital\_q_t+capital\_retail_t
\]

### B. `baseline_4d`

\[
capital\_ch_t = \beta_{ch,t} U_{ch,t-1}
\]

\[
capital\_mix_t = \beta_{mix,t} U_{mix,t-1}
\]

其中 `U_mix = Q_rule + R_seed` 是量化与散户的合成外力输入。若需要展示 `capital_q / capital_retail`，只能按 `Q_rule` 与 `R_seed` 在 `U_mix` 中的绝对权重做诊断性拆分：

\[
w_{q,t}=\frac{|Q_t^{rule}|}{|Q_t^{rule}|+|R_t^{seed}|+\delta}
\]

\[
w_{retail,t}=\frac{|R_t^{seed}|}{|Q_t^{rule}|+|R_t^{seed}|+\delta}
\]

\[
capital\_q_t=w_{q,t} \cdot capital\_mix_t,\quad
capital\_retail_t=w_{retail,t} \cdot capital\_mix_t
\]

解释：

1. `capital_ch`：游资外力贡献，即游资行为流经 `beta_ch` 加载后的即时价格推动
2. `capital_q`：量化外力贡献，即量化行为流经 `beta_q` 或 `beta_mix` 诊断分摊后的即时价格推动
3. `capital_retail`：散户外力贡献，即散户行为流经 `beta_retail` 或 `beta_mix` 诊断分摊后的即时价格推动
4. `phi`：市场惯性响应，不归属于某一类资金
5. `theta`：市场阻尼/反转响应，不归属于某一类资金

注意：

> 这里得到的是三类“外部资金行为代理贡献强度”，不是账户身份证明；`C_I / C_D` 只进入市场状态诊断与价格闭合，不再反解回三类资金身份。

## 10. 规则锚点与 PID 的关系

PID 模型不是凭空运行，而是由规则层提供高置信输入。

规则层主要输出：

1. `CH_rule`
2. `Q_rule`
3. `R_seed`

统一映射为：

### A. `baseline_4d`

\[
U_{ch,t} = CH_t^{rule}
\]

\[
U_{mix,t} = Q_t^{rule} + R_t^{seed}
\]

### B. `full_5d`

\[
U_{ch,t} = CH_t^{rule}
\]

\[
U_{q,t}^{seed} = Q_t^{rule}
\]

\[
U_{retail,t}^{seed} = R_t^{seed}
\]

这也是为什么资金类型判断规范与 PID 文档必须统一维护。

## 10.1 最终算法总流程

最终算法建议按以下固定顺序执行：

1. 读取逐笔委托、逐笔成交、盘口快照与集合竞价成交结果
2. 按 [统一资金类型判断规范.md](</e:/2026OPC大赛/自动化交易/比赛设计文档/统一资金类型判断规范.md>) 完成事件级分类：
   - 主动塑价 -> `hot_money`
   - 主动但未达游资标准，或被动且 `order_age_minutes <= 5` -> `quant`
   - 被动且 `order_age_minutes > 5` -> `retail`
3. 将事件按窗口聚合成：
   - `CH_t^{rule}`
   - `Q_t^{rule}`
   - `R_t^{seed}`
4. 按固定顺序做窗口级预处理：
   - 有符号净额求和
   - 缩尾
   - 缺失补零
   - EWMA 标准化
5. 形成 PID 输入：
   - `baseline_4d`：`U_ch, U_mix`
   - `full_5d`：`U_ch, U_q, U_retail`
6. 运行卡尔曼滤波 / RTS 平滑，估计 `phi, beta_*, theta`
7. 计算：
   - `C_P,t`
   - `C_I,t`
   - `C_D,t`
   - `eps_t`
8. 由
\[
\Delta P_t = C_{P,t} + C_{I,t} + C_{D,t} + \varepsilon_t
\]
完成闭合检查
9. 依据外力加载公式求：
   - `capital_ch`
   - `capital_q`
   - `capital_retail`
10. 输出外力贡献、系统响应、噪声比、解释力、锚点误差与模式诊断结果

一句话概括：

> 规则层负责把逐笔行为压缩成三类窗口级代理流，PID 层负责估计三类外力加载系数与市场惯性/阻尼响应，并由 `beta_* U_*` 计算三类资金代理贡献。

## 11. 输出字段规范

每个窗口建议至少输出：

| 字段 | 含义 |
| --- | --- |
| `phi` | 惯性系数 |
| `theta` | 阻尼系数 |
| `beta_ch` | 游资加载系数 |
| `beta_q` / `beta_mix` | 量化或量化+散户合成流加载系数 |
| `beta_retail` | 散户加载系数 |
| `c_p` | P 项推进贡献 |
| `c_i` | I 项惯性贡献 |
| `c_d` | D 项阻尼贡献 |
| `eps` | 未解释扰动 |
| `capital_ch` | 游资外力贡献 |
| `capital_q` | 量化外力贡献 |
| `capital_retail` | 散户外力贡献 |
| `capital_ch_rule_approx` | 仅规则兜底展示时的游资近似值，不属于模型外力贡献 |
| `capital_q_rule_approx` | 仅规则兜底展示时的量化近似值，不属于模型外力贡献 |
| `capital_retail_rule_approx` | 仅规则兜底展示时的散户近似值，不属于模型外力贡献 |
| `noise_ratio` | 噪声占比 |
| `explain_ratio` | 解释力 |
| `capital_anchor_error` | 游资规则锚点与模型外力贡献误差 |
| `rule_error_q` | 量化规则流与模型外力贡献误差 |
| `rule_error_retail` | 散户规则流与模型外力贡献误差 |
| `mode_name` | 当前窗口采用的运行模式 |
| `is_structural_output` | 当前窗口是否输出了模型外力贡献结果 |

补充说明：

1. `baseline_4d` 下优先输出 `beta_mix`，此时 `beta_q / beta_retail` 可为空
2. `full_5d` 或 `diag_5d` 下优先输出 `beta_q / beta_retail`
3. `rule_base` 下状态参数与一级贡献字段可为空

## 12. 关键诊断指标

## 12.1 噪声占比

\[
noise\_ratio_t = \frac{|\varepsilon_t|}{\max(|\Delta P_t|,\delta)}
\]

解释：

1. 越低表示结构项越能解释当期价格变化
2. 越高表示该窗口越混沌

建议分层：

1. `< 0.4`：结构解释有效
2. `0.4 - 0.6`：中等可信
3. `>= 0.6`：低可信或规则兜底

## 12.2 解释力

\[
explain\_ratio_t = 1 - \min(noise\_ratio_t,1)
\]

## 12.3 游资一致性误差

\[
err_{ch,t} =
\frac{|CH_t^{anchor} - CH_t^{pid}|}
{\max(|CH_t^{anchor}|, |CH_t^{pid}|, \delta)}
\]

解释：

1. 越低表示规则游资锚点与模型解释一致
2. 越高表示规则与结构解释冲突，需降权

## 12.4 D 项状态诊断

建议使用 `theta` 作为阻尼诊断主指标：

| `theta` 范围 | 解释 |
| --- | --- |
| `< -0.3` | 强阻尼，价格易回归 |
| `[-0.3, 0.1]` | 正常阻尼 |
| `> 0.1` | 正反馈加速，趋势可能过强 |

注意：

这组阈值是默认工程阈值，正式回放后应改为分位数标定。

## 13. 预测与状态判断

本模型首先是解释引擎，其次才是短期预测引擎。

### 13.1 下一窗口预测

若当前窗口参数估计稳定，可给出下一窗口条件预测：

\[
\Delta P_{t+1}^{pred}
= \beta_{ch,t} U_{ch,t}
+ \beta_{q,t} U_{q,t}
+ \beta_{retail,t} U_{retail,t}
+ \phi_t \Delta P_t
+ \theta_t (\Delta P_t - \Delta P_{t-1})
\]

在 4 维模式下，使用 `beta_mix * U_mix` 替代 `beta_q + beta_retail` 项。

其中：

> `U_mix` 仅是 `Q_rule + R_seed` 的合成输入，属于状态维度压缩，不再代表单独的“混合池”规则分类结果。

### 13.2 趋势状态判断

建议重点关注：

1. `phi` 的方向与变化率
2. `theta` 的绝对水平与变化率
3. `noise_ratio` 是否持续升高

可解释为：

1. `phi` 上升：惯性增强，趋势延续概率提高
2. `phi` 下降：惯性衰减，趋势可能转弱
3. `theta` 由负快速靠近零：阻尼失效，突破概率上升
4. `theta` 大幅转负：均值回归和压制增强

## 14. 回退模式

为保证工程稳定性，PID 模型必须支持以下模式：

### 14.1 `rule_base`

适用场景：

1. schema 不稳定
2. 方向恢复差
3. 无法稳定形成窗口级代理流

处理：

1. 不进入状态空间模型
2. 只输出规则层结果
3. 模型贡献字段 `capital_ch / capital_q / capital_retail` 应置空或不输出
4. 若前端或提交链路需要展示近似值，应单独输出 `capital_ch_rule_approx / capital_q_rule_approx / capital_retail_rule_approx`
5. 同时设置 `is_structural_output = false`，防止将规则兜底结果误读为模型外力贡献

### 14.2 `baseline_4d`

适用场景：

1. 规则锚点稳定
2. 希望以更低状态维度运行 PID
3. `theta` 仍可用于 D 诊断

处理：

1. 使用 `U_ch` 与 `U_mix = Q_rule + R_seed`
2. 保留游资输入的独立辨识
3. 将量化与散户仅作为合成推进流进入 P 项

### 14.3 `diag_5d`

适用场景：

1. 5维输入基本可用
2. `theta` 可估但主要用于系统响应诊断

处理：

1. 保留 5 维状态
2. `theta` 主要用于诊断

### 14.4 `full_5d`

适用场景：

1. D 驱动项有效
2. 种子流质量稳定
3. 可辨识性验证通过

处理：

1. 完整启用 5 维状态
2. 对 `beta_q` 与 `beta_retail` 分别估计，增强量化与散户外力贡献的可解释性
3. `theta` 作为阻尼/反转诊断项参与价格闭合，不参与三类资金贡献拆分

## 15. 验证要求

正式实现前，至少应完成以下验证：

1. 4维 / 5维 对照实验
2. `theta` 可辨识性验证
3. `D_driver` 有效性验证
4. 规则锚点与模型外力贡献一致性验证
5. Ljung-Box 残差白噪声检验
6. 参数多起点稳定性检验

## 16. 与资金类型规范的接口要求

本文件与 [统一资金类型判断规范.md](</e:/2026OPC大赛/自动化交易/比赛设计文档/统一资金类型判断规范.md>) 必须保持一致，特别是以下几点：

1. 游资定义：主动塑价
2. 量化定义：主动但未达游资标准，或被动且委托时间距成交时间不超过 5 分钟
3. 散户定义：被动且委托时间距成交时间超过 5 分钟
4. 规则层输出的是锚点和种子流，不是最终模型外力贡献
5. 卖出方向恒为负值

如果未来修改资金类型定义，必须同步修改本文件中的：

1. 行为流解释
2. 输入变量映射
3. 外力贡献计算说明

## 17. 最终实现清单

为避免规则层与 PID 层脱节，正式实现时建议严格按以下清单落地。

### 17.1 输入清单

至少应准备以下输入：

1. 逐笔委托数据：
   - `order_time`
   - `order_price`
   - `order_amount`
   - `side`
   - `order_id`
2. 逐笔成交数据：
   - `trade_time`
   - `trade_price`
   - `trade_amount`
   - `matched_order_id`
3. 连续竞价盘口快照：
   - `best_bid`
   - `best_ask`
   - `tick_size`
4. 集合竞价结果：
   - `P_auction_trade`
   - `P_prev_close`
5. 窗口级价格序列：
   - `Delta P_t`
   - `Delta P_{t-1}`
   - `Delta P_{t-2}`

### 17.2 规则层输出清单

规则层必须输出并保存：

1. 事件级字段：
   - `capital_type_rule`
   - `confidence_score`
   - `confidence_level`
   - `fallback_reason`
   - `reason_codes`
2. 窗口级字段：
   - `CH_rule_t`
   - `Q_rule_t`
   - `R_seed_t`

### 17.3 预处理顺序

窗口级输入进入 PID 前必须按固定顺序执行：

1. 事件按主标签聚合为窗口级有符号净额
2. 对窗口级净额做缩尾
3. 缺失补零
4. EWMA 标准化
5. 形成 `U_ch / U_mix` 或 `U_ch / U_q / U_retail`

### 17.4 PID 模式开关

实现中建议显式保留以下模式开关：

1. `rule_base`
2. `baseline_4d`
3. `diag_5d`
4. `full_5d`

### 14.5 文件级实现约束

为避免 4 维与 5 维状态空间实现继续混写在同一文件中，正式工程实现必须满足以下文件级约束：

1. `pid_decomposer.py` 只作为模式选择入口和 facade。
2. `pid_decomposer_shared.py` 只存放公共骨架与公共诊断逻辑。
3. `pid_decomposer_4d.py` 只实现 `baseline_4d`。
4. `pid_decomposer_5d.py` 只实现 `diag_5d / full_5d`。
5. 上层模块统一通过选择入口决定调用哪个处理器，不直接读取 4D / 5D 内部实现细节。

并记录每个窗口最终采用的模式。

### 17.5 PID 输出清单

PID 层至少应输出：

1. 状态参数：
   - `phi`
   - `theta`
   - `beta_ch`
   - `beta_q` 或 `beta_mix`
   - `beta_retail`
2. 一级贡献：
   - `c_p`
   - `c_i`
   - `c_d`
   - `eps`
3. 三类资金外力贡献：
   - `capital_ch`
   - `capital_q`
   - `capital_retail`
   - `capital_ch_rule_approx`
   - `capital_q_rule_approx`
   - `capital_retail_rule_approx`
4. 诊断字段：
   - `noise_ratio`
   - `explain_ratio`
   - `capital_anchor_error`
   - `rule_error_q`
   - `rule_error_retail`
   - `mode_name`
   - `is_structural_output`

其中：

1. `baseline_4d` 允许只输出 `beta_mix`
2. `full_5d / diag_5d` 允许输出 `beta_q / beta_retail`
3. `rule_base` 允许状态参数与模型贡献字段为空，但近似展示字段必须单列输出

### 17.6 最低验收条件

正式提交或联调前，至少应检查：

1. 事件级三分类总数与窗口聚合总数一致
2. 卖出方向恒为负值
3. `CH_rule / Q_rule / R_seed` 与 PID 输入映射一致
4. `Delta P_t = C_{P,t} + C_{I,t} + C_{D,t} + eps_t` 可闭合
5. `capital_ch / capital_q / capital_retail` 与对应 `beta_* U_*` 计算口径一致
6. 残差白噪声检验、可辨识性检验与多起点稳定性检验通过
7. 若 `mode_name = rule_base`，则 `is_structural_output = false` 且近似值字段与模型贡献字段不混用

## 18. 一句话总结

本系统把窗口级价格变化建模为一个带有外部资金冲击、市场惯性和市场阻尼的时变状态系统；规则层负责识别高置信行为起点，PID 状态模型负责估计三类外力加载贡献、完成价格闭合并输出市场状态诊断。
