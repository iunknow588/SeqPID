| 评估项 | 状态 | 关键问题 |

**关键待确认问题（编码前必须回答）：**

### 第2层：结构系数识别（核心层）

**可行性：理论可行，但存在可辨识性风险**

#### 2.1 状态空间模型设计

| 要素 | 设计 | 评估 |
|------|------|------|
| 状态向量 | 5维 [φ, β_ch, β_q, β_retail, θ] | 合理，但 θ 维度偏高 |
| 观测方程 | ARX 结构，不含滞后残差 | ✅ 明智，避免 ARMAX 循环依赖风险 |
| 状态转移 | 随机游走 | 标准做法，但需关注 θ 的漂移约束 |
| 滤波算法 | KF + RTS 平滑 | ✅ 成熟方案，实现路径清晰 |

#### 2.2 关键风险点：θ_t 的可辨识性（最高优先级）

**问题描述：**
- θ_t 在主观测方程中通过 `D_driver_t = ΔP_{t-1} - ΔP_{t-2}` 获得直接约束
- 但 `D_driver_t` 可能与 `ΔP_{t-1}` 高度共线，导致 θ_t 估计不稳定
- 文档已提出 Q_55 约束方案（Q_55 = Q_11 × 0.05 ~ 0.5），但仍需验证

**风险等级：高**

**原因：**
1. 5维状态向量在仅4维有效观测输入的情况下，θ_t 本质上是"弱观测"状态
2. 若 D_driver 与 ΔP_{t-1} 高度共线，θ_t 估计对初值敏感
3. 若 θ_t 不稳定，将直接影响第4层 D 项拆分的可靠性

**缓解措施（文档已提出）：**
- Q_55 = Q_11 × 0.05 ~ 0.5
- 若 D_driver 失效，回退到 4 维模型
- 模拟数据验证参数恢复能力

**必须完成的验证实验：**
1. 在模拟数据上，固定真实 θ 序列，检验 KF/RTS 能否有效恢复
2. 对比 4 维模型（不含 θ）与 5 维模型的预测残差差异
3. 检查 D_driver 与 ΔP_{t-1} 的相关性、状态协方差矩阵条件数

**通过门槛：**
- θ 方向识别和强弱排序具有一致性，不出现长期单边漂移
- D_driver 有足够波动，且不与 ΔP_{t-1} 或代理流高度共线
- 5 维模型不显著恶化残差与主导判定稳定性

### 第3层：基础归因层

**可行性：可行**

| 要素 | 评估 |
|------|------|
| 权重定义 | 基于流量占比的归一化权重，数学简洁 |
| 基础贡献公式 | β·U + φ·ΔP·w + κ_I·ε·w 结构清晰，计算简单 |
| 惯性项处理 | ε_{t-1} 明确归入 I 环节，不进入 D 项，避免重复解释 |

**注意点：** κ_I = 0.5 为默认值，验证范围 0~1。若残差记忆项导致展示层与结构层偏离过大，优先降低 κ_I 或降级置信度。

---

### 第4层：D项拆分层

**可行性：条件可行，依赖 λ 参数标定**

#### 4.1 设计合理性

| 设计点 | 评估 |
|--------|------|
| D 项主要区分量化 vs 散户 | ✅ 业务逻辑合理，量化偏阻尼、散户偏恐慌反转 |
| 游资 λ 默认 0.05 | ✅ 保守合理，游资主要是冲击方而非阻尼方 |
| 显式归一化权重 s_ch, s_q, s_retail | ✅ 保证结构层闭合性 |

#### 4.2 关键风险点：λ 参数敏感性

**问题描述：**
- λ_ch=0.05, λ_q=1.00, λ_retail=0.40 为行为代理层面的拆分先验
- 文档要求验证"λ 浮动 20% 时日终主导判定变化 < 10%"

**风险等级：中等**

**原因：**
1. λ 参数直接影响量化/散户的 D 项分配比例
2. 若 λ 参数标定不当，可能导致量化/散户判定系统性偏差
3. 文档已提出敏感性扫描计划，但需历史样本支持

**缓解措施：**
- 按文档要求执行 λ 敏感性扫描
- 建立 λ 参数与人工抽检标签的吻合度评估
- 若敏感性过高，考虑缩小 λ 扰动范围或降级 D 项拆分

## 三、3 个关键风险点与闭环建议

### 风险1：θ_t 可辨识性（P0，最高优先级）

| 项目 | 内容 |
|------|------|
| 风险描述 | 5维状态向量中 θ_t 的 D_driver 可能与共线，导致估计不稳定 |
| 影响范围 | 第4层 D 项拆分、量化/散户区分度、诊断层 |
| 闭环动作 | 1. 模拟数据参数恢复实验 2. 4维/5维模型对照 3. D_driver 相关性检查 4. 条件数检查 |
| 交付物 | `model_identifiability_report.md` |
| 回退策略 | 若 θ_t 不稳定，回退到 4 维模型，D 项仅作为诊断指标不参与拆分 |

**必须回答的问题：**
- D_driver_t = ΔP_{t-1} - ΔP_{t-2} 在真实数据上是否有足够波动？
- 若 D_driver 与共线，是否加入盘口恢复速度、价差收敛速度等微观结构增强特征？
- Q_55 的默认值取 Q_11 × 0.1 是否足够保守？

### 风险2：Level2 数据 schema 不确定性（P1）

| 项目 | 内容 |
|------|------|
| 风险描述 | 逐笔数据字段、格式、关联方式尚未验证 |
| 影响范围 | 第1层分类器、整个算法输入 |
| 闭环动作 | 1. Schema 探针 2. 单股单日 48 窗口代理流生成验证 |
| 交付物 | `schema_probe_report.md` |
| 回退策略 | 若关键字段缺失，分类器降级为更简单的阈值版本 |

### 风险3：λ 参数敏感性（P1）

| 项目 | 内容 |
|------|------|
| 风险描述 | D 项拆分权重 λ 对量化/散户判定结果敏感 |
| 影响范围 | 第4层拆分结果、日终主导判定 |
| 闭环动作 | 1. λ 敏感性扫描实验 2. 与人工抽检标签吻合度评估 |
| 交付物 | `lambda_sensitivity_report.md` |
| 回退策略 | 若敏感性过高，缩小 λ 扰动范围或使用更保守默认值 |

## 六、需要讨论的问题

### 问题1：D_driver 的增强特征是否纳入 1.1 版本？

文档 14.1 节提出：若价格差分 D_driver 与共线，应加入盘口恢复速度、价差收敛速度等微观结构增强特征。

**建议讨论：**
- 是否在阶段 B 中同步验证增强特征的可行性？
- 若增强特征需要额外的盘口数据预处理，是否会增加阶段 A 的复杂度？

### 问题2：λ 参数的标定数据来源

文档 14.3 节提出三层标定：人工抽检标签 → 规则种子代理真值 → 保守默认值。

**建议讨论：**
- 是否有历史样本可供人工抽检？
- 若无人工标签，规则种子的高置信度样本如何定义？

### 问题3：午间热启动的 k_opt 标定

文档 14.4 节建议 k_opt 默认 3.0，验证范围 2.0-5.0。

**建议讨论：**
- 是否有历史数据支持"上午收盘到下午开盘"的预测误差回放？
- 若缺乏历史数据，是否先使用默认值，后续再优化？

### 问题4：Task 1 历史样本来源

算法设计主要服务于解释特征，但 Task 1 的模式聚类需要历史样本：
- 历史样本的获取方式？
- 聚类原型的训练数据量要求？

## 七、最终结论

| 维度 | 结论 |
|------|------|
| **整体可行性** | ✅ 可行，但需完成 3 个关键风险闭环 |
| **理论自洽性** | ✅ 控制论框架逻辑自洽，工程展开合理 |

**核心建议：**
1. **不要跳过 `model_identifiability_report.md`**，这是整个算法可信度的基石
2. **若 θ_t 在模拟数据上无法稳定恢复，果断回退到 4 维模型**，不要强行保留 5 维
3. **λ 敏感性扫描不要省略**，它决定了量化/散户拆分的可信度
4. **整个算法定位为"解释型辅助引擎"**，不要试图让它独立承担 Task 1/2 的全部判定责任
5. **优先构建基线版**：阶段 A 结束即可产出可运行版本，确保有提交能力，再逐步优化
