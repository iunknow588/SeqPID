### 1.3 统一字段与模型口径

本文以 `PID算法与实现.md`、`金融领域的物理学概念模型.md` 为当前模型依据，
并与 `统一资金类型判断规范.md`、`统一.md` 保持接口兼容。统一遵守以下边界：

1. 规则层字段：
   - `CH_rule / Q_rule / R_seed` 只表示行为锚点与种子流
   - 不表示最终模型外力贡献
2. PID 层字段：
   - `c_p / c_i / c_d / eps` 负责价格闭合
   - `phi / theta / beta_*` 是状态参数
3. 模型外力贡献字段：
   - `capital_ch / capital_q / capital_retail` 只能由 `beta_* U_*` 计算得到
   - 不得由 `CH_rule / Q_rule / R_seed` 直接覆写
   - 不得由 `c_i / c_d` 反解摊派得到
4. 模式边界：
   - `rule_base`：只允许规则近似展示，`is_structural_output = false`
   - `baseline_4d`：`U_ch = CH_rule`，`U_mix = Q_rule + R_seed`
   - `diag_5d / full_5d`：`U_ch / U_q / U_retail` 三路输入独立进入 PID
5. `phi / theta` 只负责市场惯性、阻尼与状态诊断，不参与三类资金身份拆分
6. 理论时间轴统一采用交易时间 `q`：窗口索引或累计成交量进度视为离散时间步，价格状态写作 `P_state(q)`。
7. `P_wv_window` 仅是成交量域窗口诊断价格；实时状态、预测和提交统一使用 `P_state`。
8. `Delta_q P_state` 是观测价格差分，`y_hat` 是模型预测值；二者不得共用字段。
9. `Delta_q^2 P_state` 仅用于离线解释或诊断，不作为默认实时高权重特征。
10. `beta_norm` 与 `m_eff` 只能在同一 `U_*_mv_ratio` 输入口径下生成；截断或不确定性过高时不得进入跨股排名。
11. 实时估计使用先验/滤波量；RTS、HP 和全样本平滑仅用于离线复盘。
12. PID 标准状态空间口径固定为 `psi_t = psi_t-1 + eta_t`、`y_t = x_t · psi_t + epsilon_t`、`y_hat_t+1|t = x_t+1|t · psi_t|t-1`；价格差分重构关系不得被实现为独立状态演化模型。
13. 5 维模式切换必须记录 `K_up / K_down / max_switch_per_day` 与 `lambda_switch / lambda_jump / lambda_error`，默认仅作为离线敏感性分析和回退诊断，不在提交期动态调参。
14. 跨股比较必须按 `q_type` 和流动性分组对齐；未通过 `cross_symbol_comparable` 时，`m_eff_slow`、`phi`、`theta` 只能做单股诊断或组内分位展示。
15. 双域映射中的 `kappa_t = Delta_t P / Delta_t V` 只在 `Delta_t V` 超过阈值、价格基差正常且分组误差通过时使用；薄成交窗口标记为 `thin_trade_window`，不得机械放大敏感度。
16. `m_slow_method` 默认优先级为 `ewma_realtime > rolling_median_realtime > kalman_filter_realtime`，这是工程稳健性排序；卡尔曼滤波必须经过离线验证后才能作为默认。

## 4. 数据接入与窗口化

### 4.1 数据接入模块

#### 4.1.1 输入契约

系统支持两种输入模式：

1. **原始重算模式**  
   直接读取逐笔成交、逐笔委托、逐笔撤单、十档盘口快照并计算参考特征。
2. **参考特征直读模式**  
   直接读取已整理好的参考特征文件，再结合原始数据补充状态解释特征。

#### 4.1.2 核心函数

```python
def load_raw_day(trade_date: str, symbol: str, input_dir: str) -> dict: ...
def load_reference_feature_day(trade_date: str, symbol: str, input_dir: str) -> "pd.DataFrame | None": ...
def probe_input_schema(input_dir: str, trade_date: str) -> dict: ...
```

#### 4.1.3 设计要求

- 所有路径使用相对路径
- 读取失败必须抛出结构化异常，不能默默返回空结果
- 必须记录每个 symbol 的原始数据覆盖率
- 第一周开发必须先产出 `schema_probe_report.md`，未经探针验证不得冻结字段契约

#### 4.1.4 schema 探针设计


### 4.2 时间窗口切分模块

#### 4.2.1 固定窗口规则

固定窗口除了承担工程聚合作用，还承担离散交易时间采样作用。A 股连续竞价共 240 分钟，按 5 分钟切分后得到 48 个基础窗口，因此窗口 `t` 到窗口 `t+1` 的推进，默认对应内部交易时间 `q` 的一步推进，窗口级价格变化、惯性和阻尼均在同一 `q` 轴上解释。

- 窗口总数：48
- 窗口长度：5 分钟
- 时间范围：09:30-11:30，13:00-15:00

窗口化必须显式记录：

```text
q_type = volume_equal / volume_quantile / window_index
delta_q
```

### 5.2 状态解释特征构建

#### 5.2.1 设计定位

该模块不是比赛唯一主模型，而是**高解释性辅助引擎**，用于：

1. 识别 `CH_rule / Q_rule / R_seed` 三类行为代理流的时间演化
2. 为 Task 2 提供判别证据
3. 为 Task 1 的模式解释增加可读性
4. 在规则层与 PID 层之间提供统一字段契约

#### 5.2.2 复用范围

复用现有 `设计文档` 中以下成熟设计：

- 三类规则流：`CH_rule / Q_rule / R_seed`
- 48 窗口状态建模
- Kalman filter；RTS 仅限离线复盘
- 稳定性检测
- 噪声占比与能量熵

#### 5.2.3 核心函数

```python
def build_capital_rule_features(windows: list[TimeWindow], runtime_config: dict) -> "pd.DataFrame": ...
def build_state_features(windows: list[TimeWindow], rule_features: "pd.DataFrame",
                         runtime_config: dict) -> list[StateFeature]: ...
def build_state_daily_summary(state_features: list[StateFeature]) -> dict: ...
```

状态解释引擎的实时计算顺序固定为：

```text
历史数据 <= t-1
    -> 构造 x_t
    -> 得到 psi_t|t-1
    -> 计算 y_hat_t+1|t
    -> 当前窗口观测到达后生成 y_t
    -> 更新 psi_t|t
```

核心字段口径为：

```text
y_t = Delta_q P_state,t
y_hat_t+1|t = x_t+1|t · psi_t|t-1
v_q,t = y_t / Delta q_t
v_hat_q,t+1|t = y_hat_t+1|t / Delta q_t+1
```

`P_state` 是实时状态和提交口径；`P_wv_window` 只用于
`price_basis_error_ratio` 和双域映射诊断。若使用 5 维或 6 维扩展，
必须通过可辨识性、特征根/Jury、残差和样本外比较，否则回退到
`baseline_4d` 或 `rule_base`。

`m_eff` 的正式生成必须满足：

```text
capital_* = beta_*_mv_ratio * U_*_mv_ratio
beta_norm_* = (capital_* / P_state) / U_*_mv_ratio
```

若 `beta_norm` 被截断、置信区间过宽或 `m_eff_uncertainty_flag = true`，
只能做单股诊断，不能进入跨股排名或正式质地画像。

补充要求：

1. 连续竞价事件必须直接归入 `hot_money / quant / retail`
2. 主动成交但攻击性细节缺失时，按 `order_amount > 100000` 回退为游资，否则回退为量化
3. 被动成交按 `order_age_minutes <= 5` 判量化、`> 5` 判散户
4. `baseline_4d` 中允许使用 `U_mix = Q_rule + R_seed` 作为状态压缩输入，但不得作为独立分类标签
5. `rule_base` 模式下，`capital_ch / capital_q / capital_retail` 模型贡献字段应置空，若需要展示近似值，必须单独输出 `capital_*_rule_approx`
6. `signed_large_active_amount / signed_mix_qr_amount` 仅作为历史兼容输入，进入规则层后必须统一转换为 `CH_rule / Q_rule / R_seed`
7. 规则层必须输出 `u_source_type / rule_confidence / rule_version / rule_latency_windows / feature_cutoff_timestamp`
8. 特征工程不得使用预测截止时点之后才可获得的当日最高价、最低价、收盘后统计或全样本统计量
9. `P_wv_window` 仅用于双域诊断；实时状态和预测输入使用 `P_state`

### 5.3 市场 PID 聚合

统一补充约束：

1. 市场 PID 聚合的主口径优先来自个股 `c_p / c_i / c_d` 的横截面统计。
2. `phi / theta / beta_*` 可进入 diagnostics、相对市场解释与后续研究分析，但不直接替代市场 P/I/D 主聚合值。
3. 若个股 PID 结果缺失，允许启用启发式 fallback，但必须在 diagnostics 中标记来源。

#### 5.3.1 设计定位

市场 PID 模块不替代单股模型，而是作为**市场先验层**，服务于：

1. 估计当日全市场状态
2. 生成市场 PID 分布快照
3. 给单股输出增加相对市场偏离指标

#### 5.3.2 输入

- 当日全市场 `DailySample`
- 单股 `PatternResult`
- 单股 `PredictResult`

```python
                        predict_results: list[PredictResult], config: dict) -> MarketPidSnapshot:
    ...

    ...
```

#### 5.3.4 核心口径

市场广度：

```text
ADR = up_count / down_count
Breadth = (up_count - down_count) / (up_count + down_count)
```

市场 PID 横截面聚合：

```text
P_market_dist = dist(P_stock for all stocks)
I_market_dist = dist(I_stock for all stocks)
D_market_dist = dist(D_stock for all stocks)
```

相对市场偏离：

```text
P_rel = (P_stock - median(P_market)) / std(P_market)
I_rel = (I_stock - median(I_market)) / std(I_market)
D_rel = (D_stock - median(D_market)) / std(D_market)
```

## 7. Task 2 参与者识别模块详细设计

### 7.1 总体流程

### 7.2 参与者类型识别

#### 7.2.1 输入特征

参与者类型模型重点使用：

- 订单规模分布
- 拆单连续性
- 订单节奏稳定性
- 撤单风格
- 盘口穿越行为
- 规则层特征：`CH_rule`、`Q_rule`、`R_seed`、`low_fallback` 占比
- 状态解释特征：`phi`、`theta`、`beta_ch`、`beta_q`、`beta_mix`、`beta_retail`、`noise_ratio`
- 外力贡献特征：`capital_ch`、`capital_q`、`capital_retail`

#### 7.2.2 建模策略

```python
def predict_capital_type(sample: DailySample, model_bundle: dict) -> tuple[str, float, dict]:
    ...
```

推荐实现：

- 首版主口径：`capital_ch / capital_q / capital_retail` 外力贡献主导项 + 规则一致性校验
- 辅助模型：`LightGBM` / `CatBoost`

#### 7.2.3 规则种子

示例高置信度种子：

- 游资种子：超大单主动买入高、尾盘冲击高、拆单一致性低、短时爆发强

#### 7.2.4 输出边界

4. `low_fallback` 事件必须参与窗口聚合，但应保留低置信诊断标记

### 11.1 数据异常

| 缺少撤单数据 | 撤单特征置缺失，降低相关模型置信度 |
| 缺少盘口快照 | 盘口压力与价格发现类特征置缺失 |
| 缺少 `order_lifetime_ms` 或等效委托存活时间 | 主动成交按 `100000` 金额阈值低置信回退；被动成交若无法恢复 `order_age_minutes`，不得输出高置信散户，需按最弱可用规则低置信归类并追加 `fallback_reason` |
| 四类文件 schema 不一致 | 直接终止批处理并输出 schema 告警 |
| 零成交窗口 | 按预先固定的 `zero_trade_policy` 处理，并保留 `zero_trade_flag`，不得读取未来窗口补值 |
| `P_wv_window` 与 `P_state` 偏差过大 | 保留双域诊断，实时状态继续使用 `P_state` |

### 11.2 模型异常

| 异常 | 处理策略 |
|---|---|
| 原型库未加载成功 | 终止 Task 1，并返回明确异常 |
| 参与者模型未加载成功 | 若规则引擎可用则降级，否则终止 Task 2 |
| 状态解释引擎失败 | 不阻塞主流程，但将相关辅助特征置缺失 |
| HDBSCAN 聚类结果不可用 | 自动回退到 KMeans/KMedoids 基线方案 |
| 市场 PID 聚合失败 | 不阻塞主提交流程，但输出缺失告警并跳过市场状态文件 |
| Jury/特征根检查失败 | 设置 `param_stability_flag = fail`，回退到 `baseline_4d` 或固定稳定参数 |
| `m_eff` 截断或不确定性过高 | 设置 `m_eff_rank_eligible = false`，仅保留单股诊断 |
| 运行时预算超限 | 设置 `runtime_warning_flag = true`，按固定策略回退，不得调用离线平滑追赶时延 |
| 任一泄漏检查失败 | 设置 `data_leakage_check = fail`，阻止正式结构化输出和提交打包 |

### 11.3 提交异常

| 异常 | 处理策略 |
|---|---|
| CSV 字段顺序错误 | 导出前校验器直接失败 |
| 行数不一致 | 禁止打包 `submit.zip` |
| 空标签 | 禁止输出提交文件 |

### 13.6.3 缺失样本与补全样本规则

2. 正式提交仍需保证股票全集完整，缺失样本统一走市场中位数补全链路。
3. 补全样本必须进入 `pid_daily_diag.csv`，不得只出现在正式提交文件中。

补全样本日级诊断固定口径：

- `mode_name = imputed_market_median`
- `sample_origin = imputed`
- `reason_code = missing_raw_data_imputed`
- `data_leakage_check = pass`
- `m_eff_rank_eligible = false`
- `submission_ready = true`
