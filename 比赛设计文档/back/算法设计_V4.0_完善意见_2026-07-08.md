## 二、V4.0 修正亮点

### 2.1 分类器去除了 order_lifetime_ms 硬依赖（✅ 重大改进）

V4.0 明确：
> "首版分类器以'成交行为 + 盘口行为 + 撤单行为'为主，不把挂单存活时间作为硬依赖"

**改进价值：**
- 消除了前序评审中的最大数据风险
- 降级路径清晰：首版用金额+盘口穿越+撤单行为，挂单时间仅作增强
- 编码前 schema 探针目标从"确认是否必须依赖"调整为"确认可使用哪些增强特征"

### 2.2 明确 θ_t 的三种稳定约束方案（✅ 重大改进）

V4.0 在 4.5 节新增：
> "若 θ_t 在主观测方程中的系数保持为 0，则应通过以下至少一种方式保持其估计稳定："
> 1. 将 Q_55 设为较小值
> 2. 引入弱观测约束或后验平滑约束
> 3. 以残差回归或阻尼诊断子模型进行二次校正

**改进价值：**
- 直接回应了前序评审中"θ_t 不进入观测方程可能导致估计不稳定"的问题
- 提供了三条可选路径，实现灵活

### 2.3 噪声协方差标定要求显性化（✅ 重要改进）

V4.0 新增 4.6 节：
> "Q_opt 与 r_opt 不应长期采用手工固定值"
> 要求：EM 算法标定 / 交叉验证 / 分层噪声基线
> 首版经验初值必须在配置中显式保留：q_process_diag, r_observation_base, q_gap_multiplier

**改进价值：**
- 消除了前序评审中"Q/r 超参数标定方法缺失"的问题
- 配置化设计便于后续调优和复现

### 2.4 伪代码 λ 参数改为 config 读取（✅ 细节改进）

```python
LAMBDA_CH = config["lambda_ch"]
LAMBDA_Q = config["lambda_q"]
LAMBDA_RETAIL = config["lambda_retail"]
```

**改进价值：**
- 消除了硬编码风险
- 便于敏感性分析和迭代调优

## 三、剩余优化建议（3项，不影响编码启动）

### 建议1：θ_t 的默认稳定策略（建议补充到文档）

**问题：** V4.0 提供了三种 θ_t 稳定方案，但未指定首版默认采用哪一种。

**建议：** 明确首版默认策略，例如：
> "首版默认采用方案 1：将 Q_55 设为 Q_11 的 1/10 到 1/100，限制 θ_t 无约束漂移。若模拟数据验证表明估计仍不稳定，则回退到方案 2（弱观测约束）。"

**理由：**
- 方案 1（Q_55 约束）实现最简单，不增加额外计算
- 方案 2（弱观测约束）需要设计辅助观测方程，复杂度较高
- 方案 3（子模型校正）需要额外模块，适合后续迭代

### 建议2：λ 参数默认值的数据支撑（建议补充验证计划）

**问题：** λ 默认值（0.05 / 1.00 / 0.40）是先验设定，文档已说明"后续应通过历史样本标定"，但未给出验证计划。

**建议：** 在文档中增加：
```markdown
### λ 参数验证计划

1. 首版按默认值实现，产出 `lambda_sensitivity_report.md`
2. 验证方法：
   - 固定 λ_ch=0.05，在 [0.5, 2.0] 范围内扫描 λ_q
   - 固定 λ_q=1.00，在 [0.1, 1.0] 范围内扫描 λ_retail
   - 评估指标：量化/散户的日内主导窗口一致性、D项诊断与人工标注的吻合度
3. 收敛标准：λ 在 ±20% 范围内变动时，日终主导判定变化 < 10%
```

**理由：**
- 明确验证计划可降低后续迭代的不确定性
- "±20% 变化 < 10%"的收敛标准可操作

### 建议3：午间热启动的 Q_gap 标定（建议补充）

**问题：** V4.0 定义了午间热启动：
```
P_{25|24}^{下午} = P_{24|24}^{上午} + Q_gap
Q_gap = k_opt · Q_opt
```

但未说明 k_opt 的默认值或标定方法。

**建议：** 补充：
> "k_opt 默认取 2.0-5.0，反映午间休市期间信息累积导致的不确定性增长。首版建议 k_opt=3.0，后续通过历史数据验证上午收盘到下午开盘的预测误差，反推最优 k_opt。"

**理由：**
- 午间 1.5 小时休市，信息累积显著，协方差膨胀是合理假设
- k_opt=3.0 是经验合理值，但应有数据验证路径

**建议4：增加闭合性断言**

```python
# 结构层闭合验证
total_struct = delta_ch_struct + delta_q_struct + delta_retail_struct + eps_smooth
assert np.allclose(total_struct, delta_P, atol=1e-10), "结构层闭合性失败"

# 展示层闭合验证
```
