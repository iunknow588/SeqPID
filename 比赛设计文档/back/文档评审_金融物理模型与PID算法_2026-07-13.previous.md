## 一、严重缺陷

### 1. 【严重】前瞻性偏差（Look-ahead Bias）—— 时间索引不一致

**位置：** `PID算法与实现.md` §4.0 主状态方程

**问题描述：**
主状态方程写作：
```
Delta_q P_t = beta_ch,t * U_ch,t-1 + ... + phi_t * Delta_q P_t-1 + ... + epsilon_t
```

但参数解释又说：
> "`beta_t`、`theta_t` 与 `x_t` 配对使用，不构成前瞻性偏差"

以及：
> "`x_t` 由截至 `t-1` 的已知输入和历史状态构成"

**逻辑矛盾：**
- 方程左侧是 `Delta_q P_t`（当前窗口价格变化）
- 右侧参数标为 `beta_ch,t`（当前窗口参数），但输入是 `U_ch,t-1`（上一窗口规则流）
- 这意味着 `beta_ch,t` 是在**已知 `Delta_q P_t` 之后**才估计的，用于**解释**而非**预测**

**后果：**
若将此方程直接用于实时预测（如比赛提交），`beta_t` 无法预先获得，必须改用 `beta_t|t-1`（滤波估计）。文档虽在§4.0末尾提到预测口径应写作 `Delta_q P_t+1|t = x_t+1|t · psi_t + epsilon_t+1`，但：
- 未明确 `psi_t` 的来源（卡尔曼滤波的一步预测？滚动OLS？）
- 未说明 `x_t+1|t` 的构造规则（`U_ch,t` 在 `t` 时刻是否已知？）
- 主方程与预测方程混用，工程实现极易误用解释口径做预测

**修复建议：**
1. 将主状态方程严格区分为**解释口径**（historical fit）和**预测口径**（forecast）
2. 预测口径必须显式使用 `psi_t|t-1`（卡尔曼滤波先验）或滚动窗口估计
3. 明确 `U_ch,t` 的可用性边界：若规则流基于窗口内逐笔数据，则 `t` 窗口结束前 `U_ch,t` 不可知，预测时只能用 `U_ch,t-1`
4. 在工程校验清单（§11）中增加前瞻性偏差扫描的具体实现检查项

### 2. 【严重】`m_eff = 1 / beta_norm` 的数值稳定性未充分保障

**位置：** `PID算法与实现.md` §6.1 等效质量估计 + `金融领域的物理学概念模型.md` §7.2

**问题描述：**
等效质量定义：
```
m_eff_* = 1 / max(abs(beta_norm_*), beta_norm_floor)
```

`beta_norm_floor` 建议：
```
beta_norm_floor = max(1e-6, p10(abs(beta_norm_*)))
```

**逻辑缺陷：**
1. `p10(abs(beta_norm_*))` 是样本10%分位，若样本量小或 `beta_norm` 分布集中，该值可能仍接近0，截断效果不足
2. 未定义 `beta_norm` 的估计误差传播到 `m_eff` 的置信区间，导致 `m_eff` 的可靠性无法量化
3. `beta_norm` 的符号翻转（如从正变负）会导致 `m_eff` 从正无穷跳变到正无穷（因取绝对值），但物理含义完全改变（从"推动上涨敏感"变为"推动下跌敏感"），文档未处理符号信息

**后果：**
- 比赛提交时 `m_eff_slow` 可能因单窗口 `beta_norm` 接近0而异常放大
- 跨日稳定性检验可能因 `m_eff` 的数值爆炸而误判为不稳定
- 符号翻转导致 `m_eff` 的物理含义丧失，但文档未要求输出 `sign(beta_norm)` 作为辅助诊断

**修复建议：**
1. 将 `beta_norm_floor` 改为固定值 + 流动性分组自适应，避免样本依赖
2. 增加 `m_eff` 的估计误差传播公式或 bootstrap 置信区间
3. 输出 `beta_norm_sign` 和 `sign_flip_flag`，在 `m_eff` 计算中保留符号信息（或至少作为诊断字段）
4. 在稳定性诊断中增加 `m_eff` 的变异系数阈值，而非仅依赖 `beta_norm` 的稳定性

## 二、中等缺陷

### 3. 【中等】5维模式可辨识性检验的阈值过于宽松

**位置：** `PID算法与实现.md` §5.3 diag_5d/full_5d

**问题描述：**
可辨识性检验阈值：
- 设计矩阵条件数 `cond(X'X) < 100`
- VIF `< 10`
- 输入两两相关系数 `abs(corr) < 0.9`

**逻辑缺陷：**
1. `cond(X'X) < 100` 对于金融高频数据过于宽松。日内窗口数通常有限（如48个5分钟窗口），5维参数在短窗口内估计，条件数100意味着矩阵接近病态，参数估计方差放大100倍
2. VIF `< 10` 是经典经济学标准，但高频数据中 `U_ch`、`U_q`、`U_retail` 往往高度相关（如大单冲击同时触发量化响应），实际 VIF 可能系统性高于10
3. 滞回规则（5窗口进入/3窗口退出）在日内切换可能过于频繁，导致模式抖动

**后果：**
- 5维模式可能在共线性严重时被强行启用，参数估计不稳定
- 频繁模式切换导致 `capital_type` 和 `capital_intention` 输出抖动，降低比赛评分稳定性

**修复建议：**
1. 将条件数阈值收紧至 `< 30` 或 `< 50`，或改用相对条件数（相对于单位矩阵）
2. VIF 阈值按流动性分组自适应：高流动性股票 `< 5`，低流动性 `< 10`
3. 增加日内模式切换次数上限（如最多切换2次），超出则锁定为4维基线
4. 输出 `mode_switch_count` 和 `mode_switch_reason` 作为比赛复盘诊断

### 4. 【中等】残差记忆扩展的启用条件存在循环论证

**位置：** `PID算法与实现.md` §4.2 残差记忆扩展

**问题描述：**
启用条件：
1. 滚动残差一阶自相关 `abs(rho_eps_1) > 0.3`
2. Ljung-Box 检验未通过 `p < 0.05`
3. 启用后验证窗口的 `closure_error`、残差方差或预测误差有稳定改善
4. 该改善不是由输入共线性或异常窗口驱动

**逻辑缺陷：**
条件3要求"启用后验证窗口有稳定改善"，但：
- 若未启用残差记忆项，如何知道启用后的改善？
- 这意味着必须先做一次**包含残差记忆项的拟合**来验证，再决定是否启用——构成循环论证
- 文档未说明这是离线诊断（用全样本试算）还是在线滚动验证

**后果：**
- 工程实现可能误用全样本拟合结果决定是否启用，导致实时预测时引入前瞻性偏差
- 或在线滚动验证时，窗口数不足导致判断不可靠

**修复建议：**
1. 明确区分：
   - **离线诊断**：用历史全样本试算残差记忆项效果，用于设定默认参数
   - **在线启用**：用滚动窗口的递推残差自相关作为触发条件，不依赖"启用后验证"
2. 将条件3改为："离线诊断显示残差记忆项在同类股票/历史样本中带来稳定改善"
3. 在线启用时仅用条件1+2，条件3和4作为离线调参依据

### 5. 【中等】`theta` 的物理含义与工程实现存在歧义

**位置：** `PID算法与实现.md` §4.0 + §2.3

**问题描述：**
文档定义：
> `theta`：阻尼、反转或助推诊断系数，具体方向由符号与稳定性诊断决定

以及：
> `theta * D_driver_t`：微分/阻尼诊断项，符号可能代表阻尼、反转或助推

**逻辑缺陷：**
1. `theta` 的符号含义未明确绑定到 `D_driver_t` 的符号组合，导致解释主观化
2. 在PID标准控制论中，D项（微分项）的作用是抑制变化率（阻尼），但文档允许 `theta` 为负（助推），这与标准PID的物理直觉冲突
3. 未定义 `theta` 与 `phi` 的联合稳定性条件（如 `|phi + theta| < 1` 或特征根在单位圆内），可能导致系统发散

**后果：**
- `theta` 的符号解释不一致，影响 `capital_intention` 的判定逻辑
- 若 `theta` 为负且绝对值较大，可能形成正反馈（助推），与"阻尼"命名矛盾
- 缺乏稳定性约束，参数估计可能越界

**修复建议：**
1. 明确 `theta` 的符号约定：
   - `theta > 0`：阻尼（抑制变化率）
   - `theta < 0`：助推（放大变化率）
   - 输出 `theta_sign_interpretation` 字段
2. 增加参数联合稳定性约束：如 `|phi| < 1` 且 `|phi + theta| < 1`，确保状态方程稳定
3. 在工程校验清单中增加特征根检验或脉冲响应检验
