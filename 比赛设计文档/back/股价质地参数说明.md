股票市场动力学参数说明与稳定性修订结论

## 1. 文档目的

本文档用于统一说明当前系统中股价动力学参数的物理定义、可解释边界、跨日期稳定性测试结论，以及后续应当保留或修改的部分。

本文与 [类比说明.md](</e:/2026OPC大赛/自动化交易/比赛设计文档/类比说明.md:1>) 保持一致，统一采用如下前提：

- 成交量可以看作“交易时间”或“市场内部时钟”
- 股价是交易时间轴上的状态量，而不是速度本身
- 股价对交易时间的一阶变化才对应速度，二阶变化才对应加速度

本次修订结合了 7 个交易日、100 只样本股票的测试结果。测试日期包括：

- 20260129
- 20260130
- 20260706
- 20260707
- 20260708
- 20260709
- 20260710

测试口径说明：

- 使用每个交易日输出目录中的 `pid_window_params.csv`、`pattern_reco.csv`、`predict_result.csv`
- 对每只股票，先将单日内多个 `window_id` 的参数取均值，再比较跨日期变化
- 稳定性同时考察数值波动、符号变化、分类标签切换

## 2. 核心物理映射

### 2.1 基础定义

本系统沿用如下物理映射，这一部分可以保留：

| 物理概念 | 数学符号 | 金融含义 | 解释 |
| --- | --- | --- | --- |
| 位置 | `P(q_t)` | 股票绝对价格 | 市场在交易时间轴上的瞬时均衡点 |
| 速度 | `Delta_q P_t = P_t - P_{t-1}` | 单位交易时间的价格变化 | 趋势强度 |
| 加速度 | `a_t = Delta_q P_t - Delta_q P_{t-1}` | 速度变化率 | 趋势强化或衰竭 |
| 外力 | `U_ch,t`, `U_q,t`, `U_retail,t` | 三类资金净作用 | 价格变化的外部驱动 |
| 交易时间 | `q_t` | 累计成交量或成交量窗口序号 | 市场内部推进轴 |

### 2.2 核心认知

在本文档中，时间轴不默认理解为自然钟表时间，而理解为由成交量推进的“交易时间”。在这个前提下，股价本身应视为位置，而不是速度。股价变化量才是速度，涨速变化才是加速度。这个逻辑与本次测试结论并不冲突，仍应保留。

它意味着：

- 无外力时，系统应趋向静止或弱波动平衡，而不是永远单边运行
- 持续资金流可以维持趋势
- 资金冲击会先表现为加速度突变，再反映为价格路径变化

## 3. 当前模型参数定义

系统目前使用以下 5 个核心参数描述市场动力学。

### 3.1 外力加载系数

| 参数 | 符号 | 当前含义 |
| --- | --- | --- |
| 游资冲击系数 | `beta_ch` | 单位游资净买卖力对价格速度的影响 |
| 量化冲击系数 | `beta_q` | 单位量化资金作用对价格速度的影响 |
| 散户冲击系数 | `beta_retail` | 单位散户资金作用对价格速度的影响 |

### 3.2 系统内生响应系数

| 参数 | 符号 | 当前含义 |
| --- | --- | --- |
| 惯性系数 | `phi` | 上一时刻速度对当前速度的留存比例 |
| 阻尼系数 | `theta` | 上一时刻加速度对当前速度的修正力度 |

## 4. 动力学方程

若把成交量窗口视为离散交易时间步，当前系统采用如下离散方程：

`Delta_q P_t = beta_ch * U_ch,t-1 + beta_q * U_q,t-1 + beta_retail * U_retail,t-1 + phi * Delta_q P_t-1 + theta * (Delta_q P_t-1 - Delta_q P_t-2) + epsilon_t`

等价地，令 `V_t = Delta_q P_t`、`a_t = V_t - V_t-1`，可写为：

`V_t = beta_ch * U_ch + beta_q * U_q + beta_retail * U_retail + phi * V_t-1 + theta * a_t-1 + epsilon_t`

这一方程的结构本身可以保留，因为它清楚地区分了：

- 外力驱动项
- 惯性延续项
- 阻尼修正项
- 噪声项

需要修订的不是方程结构，而是对参数数值稳定性和解释强度的表述。

### 4.3 理论上应当采用的公式

如果问题是“哪一个解释更符合常理”，而不是“哪一个暂时更稳定”，那么结论应当明确：

**理论上应当采用当前二阶公式。**

原因不是它更复杂，而是它与前文已经选定的物理映射保持严格一致：

- 股价 `P(q_t)` 是位置
- 股价变化 `Delta_q P_t` 是速度
- 股价变化的变化 `Delta_q P_t - Delta_q P_t-1` 才是加速度

在这个映射下，加速度不能再被重新定义为 `Delta P_t`。否则会出现概念错位：

1. 如果把 `Delta_q P_t` 叫做加速度，那么股价本身就不再是位置
2. 如果股价仍然被视为位置，那么 `Delta_q P_t` 只能是速度，不能同时又是加速度
3. 若把 `Delta_q P_t` 直接当作加速度，等于把“价格变化”与“价格变化率的变化”混成一层，物理解释会失去自洽性

因此，从理论一致性和常理出发，应当选择下式作为正式动力学方程：

#### 正式公式：二阶动力学公式

`Delta_q P_t = beta_ch * U_ch,t-1 + beta_q * U_q,t-1 + beta_retail * U_retail,t-1 + phi * Delta_q P_t-1 + theta * (Delta_q P_t-1 - Delta_q P_t-2) + epsilon_t`

特点：

- 保留了完整的“外力 + 惯性 + 阻尼”结构
- 物理解释最完整
- `theta` 直接作用在二阶差分项上，含义清晰

问题：

- 二阶差分会放大高频噪声
- 对盘口抖动、跳点和局部反向成交更敏感
- 参数更容易跨日漂移，尤其是 `theta`

理论解释：

- `beta * U` 是外力项
- `phi * Delta_q P_t-1` 是惯性项
- `theta * (Delta_q P_t-1 - Delta_q P_t-2)` 是阻尼或助推项
- `epsilon_t` 是未解释扰动

在这个公式下：

- 无持续外力时，价格变化会受到惯性与阻尼共同调节
- 市场不是“天然匀速运动”，而是“在外力、惯性和摩擦之间寻找新的平衡”

### 4.4 为什么不选一阶公式作为理论主公式

可以写出如下简化形式：

`Delta_q P_t = alpha_ch * U_ch,t-1 + alpha_q * U_q,t-1 + alpha_retail * U_retail,t-1 + rho * Delta_q P_t-1 + epsilon_t`

这个公式可以作为工程近似，但**不能作为理论主公式**。原因是：

1. 它把系统压缩成一阶响应，只保留了“价格变化延续”
2. 它不再显式区分“速度”与“速度变化”
3. 它把原本应由阻尼项解释的二阶效应，整体吸收进了经验系数

所以，一阶公式的合理定位应当是：

- 一个经验上的降阶近似
- 一个为了稳定性和可估计性而做的工程模型
- 而不是新的物理定义

### 4.5 当前明确结论

在“哪个解释更符合常理”这个问题上，不应并列保留两个答案。

**唯一应当作为理论主公式保留的是二阶公式。**

也就是说：

- 理论上：选二阶公式
- 如果以后为了稳定性做一阶化，那是工程近似，不是理论替代

因此后续文档与代码都应遵循这个原则：

1. 理论定义不改：`P(q) -> Delta_q P -> Delta_q^2 P`
2. 一阶模型若存在，只能标注为“近似模型”或“降阶模型”
3. 不再把一阶模型与二阶模型写成同等地位的两个解释体系

## 5. 跨日期测试结论

### 5.1 总体判断

测试结果表明，这组参数具有一定的状态刻画能力，但不具备强意义上的“股票固定基因”稳定性。

更准确的说法应当是：

- 它们更像“当日状态参数”
- 不适合被直接解释为“该股票长期恒定属性”
- 更适合做区间统计、状态分层、切换频率分析

### 5.2 数值稳定性

对 100 只股票、7 个日期进行统计后，得到如下特征：

1. `beta_retail` 几乎恒为 0
   - 数值上最稳定
   - 但信息量也最弱
   - 当前版本中它更像保留接口，而不是有效主变量

2. 其余参数存在中等绝对波动
   - `phi` 中位标准差约 `0.2567`
   - `theta` 中位标准差约 `0.1691`
   - `beta_ch` 中位标准差约 `0.0035`
   - `beta_q` 和 `beta_mix` 中位标准差约 `0.00186`

3. 相对稳定性较弱
   - `phi` 中位绝对变异系数约 `1.82`
   - `beta_ch` 中位绝对变异系数约 `2.72`
   - `beta_q` / `beta_mix` 中位绝对变异系数约 `2.67`
   - `theta` 中位绝对变异系数约 `2.75`

这说明很多参数的均值接近 0，导致跨日期比较时容易出现“相对变化很大”的现象。

### 5.3 符号稳定性

若将参数正负号视为方向性信号，其稳定性很弱：

- `phi` 仅有 `14/100` 只股票在 7 天内符号不变
- `beta_ch` 仅有 `4/100`
- `beta_q` 仅有 `4/100`
- `beta_mix` 仅有 `4/100`
- `theta` 仅有 `4/100`

因此，参数的正负号不能在当前版本中被视为高度稳固的个股长期属性。

### 5.4 标签稳定性

分类输出同样不稳定：

- `pattern_type` 全部 7 天不变的股票：`0/100`
- `capital_type` 全部 7 天不变的股票：`7/100`
- `capital_intention` 全部 7 天不变的股票：`0/100`

进一步看：

- 大多数股票会在 3 到 5 种 `pattern_type` 之间切换
- `capital_type` 大多数会在“游资 / 量化”之间切换
- `capital_intention` 大多数会在“买入 / 卖出 / 中性”之间切换

这意味着当前标签更像“日内状态判定”，不是“稳定人物画像”。

## 6. 哪些应当保留

### 6.1 应保留的理论部分

以下内容建议保留：

1. 股价是交易时间轴上的位置、涨跌是速度、涨速变化是加速度的物理映射
2. 外力、惯性、阻尼三部分组成统一动力学方程的建模方式
3. `phi`、`theta`、`beta_*` 作为状态变量的建模思想
4. 用窗口化估计追踪市场状态的总体方向

原因是这些部分在逻辑上成立，也与系统代码结构一致。

### 6.2 应保留但降级表述的部分

以下内容不是完全错误，但措辞应从“确定判断”改为“倾向判断”：

1. `beta_ch` 高表示游资推动效率强
2. `beta_q` 为负表示量化更偏向吸收冲击或提供流动性
3. `theta` 为负表示系统存在阻尼
4. `phi` 偏高表示趋势延续倾向较强

这些关系可以保留为解释框架，但不能再写成刚性规则。

## 7. 哪些应当修改

### 7.1 “参数缓慢演化”的表述需要修改

原文中“这五个参数共同构成股票市场的瞬时力学基因图谱，它们随时间缓慢演化”这句话需要改。

建议改为：

“这五个参数共同构成股票在特定交易日和特定窗口下的动力学状态描述。它们可能在相邻日期之间延续，也可能因市场结构、流动性和博弈主体变化而发生明显切换。”

原因：

- 从测试看，跨日期波动明显
- “缓慢演化”过强，容易误导为长期稳定

### 7.2 “典型取值范围”需要修改

原文中例如：

- `beta_ch` 通常较大（>0.3）
- `phi` 通常正
- `theta` 通常为负

这类写法都需要修订。

建议改为：

- 不再给出过强的固定阈值
- 改写成“在部分样本中常见，但跨日期存在显著漂移”
- 若要保留范围，必须使用当前实测分布重新标定，而不能用理论直觉直接给阈值

### 7.3 “顺势策略绝对安全”等表达必须删除

例如：

- “趋势极度稳固，顺势策略绝对安全”
- “变盘前夜”
- “重大突破或崩盘前兆”

这些表述过强，也不符合当前验证结果。

建议全部改成概率性语言，例如：

- “趋势延续倾向增强”
- “阻尼减弱，后续状态切换概率上升”
- “需要结合成交、盘口和市场环境进一步确认”

### 7.4 将参数解释从“长期画像”改为“状态变量”

原文容易让人以为：

- 一只股票会长期保持某种固定 `beta`
- 一只股票会长期保持某种固定 `phi` 或 `theta`
- 一只股票会长期属于某一种资金类型或形态类型

测试已经否定了这种强版本说法。

建议明确写成：

- 这些参数首先描述的是“当天该股票的交易状态”
- 对个股长期画像，只能使用跨日统计量，而不能直接使用单日估计值

## 8. 修订后的参数解释建议

### 8.1 `beta_ch`

可保留解释：

- 描述游资型资金作用对价格速度的边际影响

修订后应强调：

- 它是状态依赖的，不是固定属性
- 跨日期可能变号
- 更适合作为“当天游资推动效率”的相对指标

### 8.2 `beta_q`

可保留解释：

- 描述量化型资金对价格速度的作用方向和强度

修订后应强调：

- 负值不应被机械解释为“量化必然做市”
- 其符号和强弱对市场环境高度敏感
- 更适合结合 `theta` 和盘口结构一起解释

### 8.3 `beta_retail`

测试结论：

- 当前版本几乎恒为 0

建议：

- 暂时保留字段和模型接口
- 但在说明文档中明确写出：当前实现下该项解释力较弱，暂不建议作为主决策因子

### 8.4 `phi`

可保留解释：

- 描述速度留存，也就是趋势惯性

修订后应强调：

- 它反映的是当日状态下的惯性，而非股票长期惯性
- 单看符号和绝对阈值不够稳
- 更适合看跨窗口连续性和跨日均值区间

### 8.5 `theta`

可保留解释：

- 描述系统对加速度的修正方向和力度

修订后应强调：

- `theta < 0` 仍可解释为阻尼倾向
- 但其跨日期符号并不稳定
- 应从“强预测型指标”降级为“辅助解释型指标”

## 9. 推荐的新使用口径

结合测试结果，建议后续统一采用以下口径。

### 9.1 单日使用

单个交易日内：

- 参数可用于解释该日该股的状态
- 可用于区分当日偏惯性、偏阻尼、偏游资冲击、偏量化吸收
- 可与模式识别、成交结构、盘口状态联合使用

### 9.2 跨日使用

跨日期比较时：

- 不直接比较某一天的单点参数值
- 改比较跨日均值、标准差、变异系数、符号翻转次数
- 将“稳定性”本身作为一个新特征

推荐新增指标：

- `param_mean_7d`
- `param_std_7d`
- `param_cv_7d`
- `param_sign_flip_count`
- `pattern_switch_count`
- `capital_type_switch_count`
- `capital_intention_switch_count`

### 9.3 个股画像

若要构建“股价质地画像”，建议不再用单日标签，而用如下方式：

1. 先在多个日期上聚合参数
2. 再判断该股属于：
   - 高惯性稳定型
   - 高切换波动型
   - 高阻尼吸收型
   - 游资冲击敏感型
3. 画像基于统计区间，而不是单次估计

## 10. 本次修订后的结论

### 10.1 可以保留的核心结论

- 价格系统可以用“位置-速度-加速度”的动力学框架来建模
- `beta_*`、`phi`、`theta` 的物理解释方向总体成立
- 这组参数适合做市场状态描述

### 10.2 必须修正的核心结论

- 不能再把这些参数描述为稳定的个股长期基因
- 不能再把参数阈值写成强规则
- 不能再把单日标签解释成长期固定属性
- 不能使用“绝对安全”“变盘前夜”这类过强语言

### 10.3 当前最合理的定位

当前这组参数更适合作为：

- 日内状态特征
- 多日稳定性分析的基础变量
- 聚类、分层、切换监测的输入特征

而不是：

- 单独作为长期画像标签
- 单独作为确定性交易信号

## 12. 最终结论

本系统的动力学框架可以保留，参数定义也基本成立，但文档的解释强度必须降低，口径必须从“长期固定基因”修订为“可漂移的状态参数”。

换句话说：

- 理论框架可以保留
- 参数解释可以保留
- 强阈值、强预测、强确定性表述需要修改
- 文档应明确承认跨日期漂移和标签切换是当前模型的真实表现

这才与现有测试结论一致，也更适合作为后续模型迭代的说明基础。
